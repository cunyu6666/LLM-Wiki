# 抖音视频入库流程

> 完整流水线：从抖音收藏夹到 Wiki 知识库的端到端流程。
> 适用于后续新数据源入库时复用。

---

## 流程总览

```
抖音收藏夹 → 视频下载 → 分类筛选 → .md 生成 → 音频转文字 → GIF 预览 → Wiki 编译 → 格式规范化
```

## 1. 数据采集

### 获取收藏夹视频列表

抖音收藏夹 API：

```
GET https://www.douyin.com/aweme/v1/web/aweme/favorite/
参数：max_cursor（分页）、count=20
```

需要浏览器 Cookie（登录态）。返回 JSON 包含每个视频的：
- `aweme_id`：视频 ID
- `desc`：视频描述
- `author.nickname`：作者
- `video.play_addr.url_list[0]`：直接下载 URL

**注意**：
- yt-dlp 的抖音提取器已失效（报 "Fresh cookies needed"）
- 详情 API (`/aweme/v1/web/aweme/detail/`) 返回空响应（反爬）
- Playwright 会触发验证码页面
- 唯一可靠方式：收藏夹 API + 浏览器 Cookie

### 下载视频

```bash
# 脚本：/tmp/douyin_batch_dl.py
# 从收藏夹 API 获取视频 URL，用 curl 下载
# 支持断点续传（跳过已存在的文件）
# 每 100 个视频刷新一次 Cookie
python3 /tmp/douyin_batch_dl.py
```

### 分类筛选

```bash
# 脚本：/tmp/douyin_classify.py
# 按关键词匹配将视频分到 8 个类别
# 输出：/tmp/douyin_classification.json
python3 /tmp/douyin_classify.py
```

分类：品牌与视觉、动效与动画、UIUX设计、设计师成长、AI设计工具、前端与编程、AI与大模型、灵感与案例

### 挑选设计相关视频

```bash
# 脚本：/tmp/douyin_filter.py
# 从分类结果中筛选设计/编程/AI 相关视频
# 复制 .mp4 到 raw/来自抖音/
python3 /tmp/douyin_filter.py
```

## 2. 生成 .md 文件

```bash
# 脚本：/tmp/douyin_to_md.py
# 为每个 .mp4 生成同名 .md 文件
# 包含：frontmatter（id, source, author, url, tags）、描述、视频嵌入
python3 /tmp/douyin_to_md.py
```

生成的 .md 文件格式：

```markdown
---
id: "7078606345435467042"
source: 抖音
author: "作者名"
url: https://www.douyin.com/video/7078606345435467042
tags: ["标签1", "标签2"]
---

# 视频标题

视频描述文字

![[视频文件名.mp4]]

[原始视频](视频文件名.mp4) | [抖音链接](https://www.douyin.com/video/xxx)
```

## 3. 音频转文字

```bash
# 脚本：/tmp/douyin_transcribe.py
# 使用 Groq Whisper API（免费）
# 模型：whisper-large-v3-turbo
# 流程：ffmpeg 提取 mp3 → Groq API 转写 → 写入 .md 的 ## 文字稿 区
python3 /tmp/douyin_transcribe.py
```

**关键依赖**：
- `static-ffmpeg`（pip 包，自带 ffmpeg 二进制，不依赖系统 brew）
- Groq API Key
- ffmpeg 路径：`/Library/Frameworks/Python.framework/Versions/3.11/lib/python3.11/site-packages/static_ffmpeg/bin/darwin/ffmpeg`

**注意**：
- Groq 有速率限制（429），遇到限流需等待后重试
- 无人声视频（纯音乐）会生成极短文字稿，可跳过
- 转写完后删除 .mp3 临时文件节省空间

## 4. GIF 预览生成

```bash
# 脚本：/tmp/make_gifs.py
# 从每个视频的前 8 秒抽取帧，生成 GIF 预览
# 参数：240px 宽，1.5fps，约 12 帧
# 输出：raw/来自抖音/gifs/ 目录
python3 /tmp/make_gifs.py
```

**ffmpeg 命令**：

```bash
ffmpeg -y -v warning -i input.mp4 -t 8 \
  -vf "fps=1.5,scale=240:-1:flags=lanczos" \
  -loop 0 output.gif
```

**参数说明**：
- `-t 8`：只取前 8 秒
- `fps=1.5`：每秒抽取 1.5 帧（8 秒 ≈ 12 帧）
- `scale=240:-1`：宽度 240px，高度等比缩放
- `-loop 0`：无限循环

**结果**：559 个视频 → 521 个 GIF（38 个失败，主要是无声/纯黑视频），总计 74MB

## 5. Wiki 编译

使用 wiki skill 的并行 Agent 模式，将 raw/ 中的文字稿编译到 wiki/ 目录：

```
/wiki compile raw/来自抖音
```

每个 Agent 负责一个分类，读取文字稿，提取关键信息，编译到对应的 wiki 文件中。

**编译后结构**：
- 正文叙述：综合多个视频的要点
- 引用块：来自周刊/文章的补充信息
- `**相关视频**：` 区：GIF 预览 + 链接

## 6. 链接规范化

### 6.1 修复断链

```bash
# 脚本：/tmp/fix_wiki_format.py（包含断链修复）
# 修复文件名中的 %20 编码问题
# 确保所有 raw/来自抖音/ 链接指向正确文件
```

### 6.2 转换为 Obsidian wikilink

所有链接使用 Obsidian wikilink 格式：

```
# 视频链接（显式 .md 扩展名，避免解析歧义）
[[文件名.md|显示文本]]

# GIF 预览嵌入
![[文件名.gif]]

# 外部文章链接（保持 markdown 格式）
[显示文本](../../raw/design/filename.md)
```

**为什么用 wikilink**：
- Obsidian 对 `../../` 相对路径支持不好（"folder exist" 错误）
- wikilink 原生支持文件名中的空格，无需 `%20` 编码
- 显式 `.md` 扩展名避免与同名 .gif 文件冲突

### 6.3 GIF + 链接组合格式

```markdown
**相关视频**：
- ![[视频名称_xxx.gif]]
- [[视频名称_xxx.md|视频标题]]
```

每个视频两行：GIF 预览 + 可点击链接。

## 7. 格式规范化

```bash
# 脚本：/tmp/fix_wiki_format.py
# 批量修复 wiki 文件格式
python3 /tmp/fix_wiki_format.py
```

修复项：
- 移除 Raw Sources 区（过长的文章摘要）
- 清理 Sources 区（只保留统计行）
- 删除空的 "相关视频：" 占位符
- `####` 标题降级为 `###`
- 移除 `&valid=false` 的失效图片
- 修复 `||` 表格样式的 blockquote
- 确保 See Also 在文件末尾
- 清理多余空行

## 注意事项

1. **ffmpeg 必须用 static-ffmpeg**：系统 brew 安装的 ffmpeg 可能不可用，用 `pip install static-ffmpeg` 安装
2. **Groq API 限流**：批量转写时会遇到 429，需要间隔请求或等待重试
3. **文件名空格**：抖音视频文件名包含大量空格，所有路径引用必须用 wikilink 格式
4. **GIF 大小控制**：240px 宽 + 1.5fps 可以将 GIF 控制在 50-400KB
5. **Obsidian 链接格式**：用 `[[file.md|text]]` 不用 `[text](path)`，避免路径解析问题
6. **同名文件歧义**：.gif 和 .md 文件同名，链接必须带扩展名 `[[file.md|text]]` 而非 `[[file|text]]`
