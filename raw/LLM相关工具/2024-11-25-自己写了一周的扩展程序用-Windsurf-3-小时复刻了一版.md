# 自己写了一周的扩展程序，用 Windsurf 3 小时复刻了一版

[mp.weixin.qq.com](https://mp.weixin.qq.com/s?__biz=MzU1MzAyODM5Nw==&mid=2247484348&idx=1&sn=a5cdffd1504d3bf63c1215a27513a4fe&chksm=fa956904ce877b85c4b9e1ff21d17671e3f7361f925f0323fd04f0c07f6baa4e06c02576c30d&mpshare=1&scene=1&srcid=1125NNxUNghw7uXgNQ0GeGrJ&sharer_shareinfo=b5df3b35e4a0870340dd32971b554b50&sharer_shareinfo_first=b5df3b35e4a0870340dd32971b554b50)PlayerYK PlayerYK


为了测试 Windsurf Editor 的能力。把去年调试 ChatGPT API 时做的小工具重构了一版。

这个工具功能很简单：**右键翻译**。选中网页上的一段文字，右键菜单中会出现一个"翻译"选项，点击后即可看到翻译结果。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FXJbBccialjy1YTc86vhDjFYWiaLnZcMIyFMGRyicTQAZO5KBKLnwTefBDlw867OT7jnA7NSH43OD9oUZibrSzyXVIw%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)Right Translator 右键菜单

这个扩展自从去年 4 月上线后，一直放着没管，也没更新任何功能。现在只在谷歌商店已经积累了 700 多用户了，Edge 那边也有几百。我这个扩展没有用什么特殊的权限，功能也简单，一句代码不用改，同一个压缩包就可以去 Edge 商店发布。

## 使用 Windsurf 从头开发浏览器扩展

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FXJbBccialjy1YTc86vhDjFYWiaLnZcMIyF4xXgdJhvRwbWcDVIGhldQJ7W3N5blYy4eiaqAuhlnIT61SsSKBuo7rQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)Windsurf IDE

整个过程从零开始。首先创建一个空文件夹，用 IDE 打开，然后建立一个 readme.md 文件，把现有功能用自己的语言描述清楚。

虽然功能简单，但1.0.0版本是我亲自写的代码，亲历过程，感触颇深，忍不住写得稍微详细了些。以下是完整的描述：

    # Chrome Extension: Right Translator - GPT-powered Translation

    ## 功能简介

    轻松实现右键翻译，支持多语言互译。

    ## 使用方法
     1. 选中文本后，右键点击。
     2. 选择"Translate to {language}"。
     3. 弹出的浮层将展示原文和译文。

    ## 功能亮点
    - 使用 Manifest V3。
    - 流式传输：翻译结果实时更新，无需等待。
    - 一键复制：轻松复制翻译文本。
    - 界面优化：
     • 使用了 Shadow DOM，避免样式污染。
     • 初始计划用 TailwindCSS，但样式简单，最终没用上。
    - 目标语言：用户可在 Popup 页面设置目标语言，支持55种语言，包括：
      •  English (English)
      •  中文（简体）
      •  中文（繁体）
      •  日本語 (Japanese)
      •  한국어 (Korean)
      •   ...以及其他50多种语言。

    ## 技术实现

    ### 翻译接口

    为了防止滥用，我对接口进行了重新包装，限定其功能仅支持翻译，其他请求将无响应。
    \`apiHost = {我自己包装的接口}\`

    ### 翻译接口的 Prompt

    \\\`\\\`\\\`
    You will translate the text to {targetLang}:

    {sourceText}
    \\\`\\\`\\\`

    返回数据结构
    翻译结果通过连续的 EventStream 数据返回，每条格式如下：

    \\\`\\\`\\\`
    {"time":"2024-11-20 22:39:03","content":"开始"}
    {"time":"2024-11-20 22:39:03","content":"使用"}
    {"time":"2024-11-20 22:39:03","content":" -"}
    {"time":"2024-11-20 22:39:03","content":" 免费"}
    {"time":"2024-11-20 22:39:03","content":" !"}
    \\\`\\\`\\\`
    最终拼接得到完整的翻译结果：

    \`"开始使用 - 免费！"\`

    实时显示在浮层中，流畅自然。

    ## 发布与打包

    打包命令如下：

    \\\`\\\`\\\`
    zip -r Right-translator-1.0.1.zip . -x \*.git\* \*.DS_Store readme.md
    \\\`\\\`\\\`

是不是感觉上面的描述过于工整了，用词也很书面？因为我原本的文字更凌乱，让 AI 帮我重写了一下，看起来他准确的领会了我的意思。

在侧边栏打开聊天，告诉他帮帮我完成整个扩展，然后就去 Chrome Webstore 发布......是不可能的，第一版代码能运行都已经很幸运了。即使有了 AI 的帮助，也有许多细节需要调整和优化，最终我花了大概 3 个小时进行完善。

## AI 助力开发的体会

Windsurf 的优势在于逻辑性和细致性。比如，我明确指出某个问题后，它可以快速定位并修改相关代码，无一遗漏。更强的是，它可以自动在多个文件中同步改动，省去了不少人工操作。整个流程让我只需在浏览器中重新加载扩展、确认效果即可。

**与 Cursor 相比，Windsurf 仍有些不足：**

1.
   缺少视觉反馈：不能粘贴图片调试，排查样式问题主要靠描述和 log。
2.
   开发风格不同：习惯了 Cursor 的即时编程风格后，Windsurf 的讨论模式可能需要适应。不过好处是，经过充分讨论后，AI 能自主完成所有改动，而无需一步步确认。

**最终，这个扩展实现了以下功能：**

*
  支持55种语言互相翻译。
*
  支持暗黑模式，UI 可拖动。
*
  最重要的是：完全免费（我自己掏钱购买的 ChatGPT API）。
