# 仅 11MB 开源小工具，斩获 1.4 万 GitHub Star！

[mp.weixin.qq.com](https://mp.weixin.qq.com/s?__biz=MzAxOTcxNTIwNQ==&mid=2457992272&idx=1&sn=1c49b17c818f4d389b277183fe668ce2&chksm=8d5cee32d6edc36675755c7c1b659a10a3e63781b3ebea5f454772fa8d352776d14655356e70&mpshare=1&scene=1&srcid=0131Z3f5hnb6gK9hajz6CCqV&sharer_shareinfo=dd712ee71cc19834ae12b6ce615b5f74&sharer_shareinfo_first=dd712ee71cc19834ae12b6ce615b5f74)小 G GitHubDaily

很多朋友从 Windows 转到 macOS 后，最难适应的可能是系统原生的 Cmd + Tab 窗口切换逻辑。

比如，我们同时开了三个 Chrome 窗口，想快速切到其中某一个，系统却只能笨拙地定位到一个窗口，无法直接锁定具体窗口。

为了找到对应窗口，我们还得动鼠标一顿找，这种明明一步能到位，却非要被强行打断的割裂感，真的很搞心态。

今天来跟大家分享一个叫 **Alt-Tab** 的开源小工具，正好把这个问题给解决了，目前还在持续更新中，累积已斩获 14000+ GitHub Star。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRqZy1ibjWvtz3lajbxuAaU7KQic57icJZ8qhvtHXbRy3QHNp0MetsLgDuQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D0)

它能把 Windows 上那个高效、直观的 Alt + Tab 切换体验，原汁原味地带到 macOS 上，甚至做得更好。

这款工具最大的特点，就是**所见即所得** 。

当我们按下快捷键时，它展示的不再是光秃秃的应用图标，而是每个窗口的实时缩略图。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_jpg%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRUWMoDuzibrf73oMDun5P9RTTiaicRDER9QSGV0ErOysP8tW0oiaGMo7pAA%2F640%3Fwx_fmt%3Djpeg%26from%3Dappmsg%23imgIndex%3D1)

我们可以清晰地看到每个 Chrome 窗口里打开的是什么网页，每个 Word 文档里写的是哪部分内容。

这种以"窗口"而不是"应用"为维度的切换方式，对于平时习惯开一堆窗口干活的朋友来说，效率提升是非常明显的。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_jpg%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRJr3YofibI3NibTibxEuatC2aqeqKWeyic2g8N6CShdoOnKJpf0rj8gEFeg%2F640%3Fwx_fmt%3Djpeg%26from%3Dappmsg%23imgIndex%3D2)

除了基础的切换功能，它在交互细节上也打磨得很顺手。

在切换界面中，如果看到某个不想用的窗口，我们可以直接把鼠标悬停在上面，点左上角的红叉把它关掉。

或者，直接按一下快捷键，将其最小化。

根本不需要切进窗口，再去找关闭按钮，整个流程一气呵成。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRJJicI7I9QLh6caRaL9zjULNzXjffWviasx6yEiaToAw0wBLzvibdmQ7UuA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D3)

它的自定义程度也相当高。

如果不希望某些 App 出现在切换列表中，可以在设置里把它们拉进黑名单。

甚至连缩略图的大小、标题文字的字号、快捷键的触发方式，都能根据我们自己的习惯来调整。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRTGrXpQnMf47FFvOMrn6qzJz0vWu7LDBmIvunJJRgXf8SzJZsfBy0Ew%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D4)

虽然它的功能逻辑源自 Windows，但 UI 设计却完全遵循了 macOS 的设计规范。

装上之后，看起来就像是系统自带的功能一样，毫无违和感。

安装也非常简单，作者在官网提供了安装包，直接下载安装就行。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRIb6Us7QRzRSdUL5iapFLp6GQJ1lM2cmGD8ZxAdqrkSTjeTicd9v45WAQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D5)

对于习惯用命令行的朋友，也可以通过 Homebrew 一键搞定：

    brew install --cask alt-tab


需要注意的是，因为要读取窗口的缩略图，首次运行时它会请求屏幕录制权限。

这是 macOS 的系统机制限制，给它授权就好，不用担心隐私问题。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FuDRkMWLia28ia1U2Mtv1vuW6xk0RNphsjRynFKVuqCb4oHwpPaVpmyHvKpIqDz8psqciara5RwtNdK6SGKEpBFVbQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D6)

总的来说，如果你也经常被 macOS 原生的窗口切换逻辑搞得晕头转向，那这款工具值得一试。

GitHub 项目地址：https://github.com/lwouis/alt-tab-macos
