**1**


**为什么需要可视化Claude Code工作流？**


如果你用过 Claude Code 或类似的 Agent 框架，大概会遇到这些问题：


* 工作流一复杂，配置文件就开始失控

* 多个 Agent / 子任务之间的关系不直观

* 新同事接手时，光理解结构就要半天

* 改一个逻辑，要在好几个文件里来回跳


ClaudeCode Workflow Studio 的思路很简单：  
既然逻辑本质是流程，那就让人"看得见"。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FVU6jnz0kaQE7tGDFdva8eFiayJ5IoPIn2RJdYHdUdMAiaibhp99EYxpFdYORTftRJnJmOwqiaFl4VVoUMnCEMo0aew%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D1)


**2**


**项目速览**


简单一句话概括：


它是一个为 Claude Code 设计的 VS Code 可视化工作流编辑器。


你可以在画布里：


* 拖拽节点

* 连接任务流

* 配置 Prompt、条件分支、子 Agent

* 然后一键导出 Claude Code 可直接运行的文件


整个过程不需要你手写复杂结构文件。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FVU6jnz0kaQE7tGDFdva8eFiayJ5IoPIn2IpFDdqQnbz3fLjk4ba0EdicibPZqRHYwM8D1xqlVJjCXEZgoM3ibGlnwA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D2)


**3**


**一个经典使用场景**


举个非常常见的例子：**自动化代码审查 Agent**

你可以搭出这样的流程：


1. 调用 GitHub MCP 拉取 PR 信息

2. 根据 PR 大小分流到不同审查策略

3. 发现问题 → 生成报告并通知提交者

4. 没问题 → 请求合并权限


整个流程在画布里完成，逻辑清晰，维护成本极低。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FVU6jnz0kaQE7tGDFdva8eFiayJ5IoPIn23W8iaPbFnX9lTgd7P5TKz0LVHqztpBNUwO9AHYMEPEpk7MLyCTUPReQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D3)
