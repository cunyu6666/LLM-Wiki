# Notion发布面向团队的Custom agents，Ivan Zhao：不能被Agent用的产品没有未来

Notion 发布了 Agent 的新产品------Custom agents。

从 2023 年接入大模型做 Notion AI 开始，Notion 几乎每一步都踩在了节奏上：AI 写作助手、AI 自动填充数据库、AI 会议纪要，一路迭代到最近的 3.3 大版本------正式上线了 Custom Agents（自定义智能体）。用户可以自己创建全天候运行的 AI Agent，自动回答重复问题、分类工单、生成状态报告，并拥有企业级的权限和安全管控。

Notion 用自己的方式加入了主动式 Agent 的赛道。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FUrL1kkHON7IFhEibhIYKcunboOS31z8D5nn0ZiaVEAyib5D3W7k2fT81ElhbYw20ZojdgJiaAll3yCjzAPorKBfsM13LvQKnDXL2l6uibfMuL6mA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D1)

在 Custom Agents 公开发布前一周，Notion 创始人兼 CEO Ivan Zhao 接受了 Access Podcast 的采访，聊了很多超出产品发布本身的东西：他本人已经不再以传统方式查看邮箱，而是通过 Agent 管理收件箱；Notion 超过 50% 的数据库现在由 Agent 而非人工创建；公司招了一名 16 岁的高中生工程师参与下一代产品开发；他还透露 Notion 即将推出独立的 Agent Chat 应用和更多面向开发者的功能------Notion 历史上不是一个开发者导向的公司，但这即将改变。

更值得关注的是 Ivan 对整个 SaaS 行业的判断：**不能被 Agent 使用的软件产品将失去市场。** 而能够拥抱 Agent 的公司，有机会从「卖席位」转向「卖工作成果」，进入一个 10 倍于传统科技行业的知识工作经济市场。Notion 正在用自己的转型路径来验证这个判断。

以下是采访中值得重点关注的核心观点：

* 如果你的产品不能被 Agent 使用，我认为未来不会乐观。但如果你的产品能被重新架构、被 Agent 使用，你就能进入科技行业从未触及的全新市场。

* 卖工具、卖座位，你还是在科技行业里。当你能卖工作、卖使用量、卖成果的时候，你就进入了知识工作经济------那是 10 倍大的市场。

* AI 带来的时间节省目前是「水平型」而非「垂直型」的------每个人每天省 5 到 10 分钟，但还没有整个岗位被 Agent 取代。

* 很多人说一个人就能做十亿美元公司，但我们相信未来仍然是一小群人一起创建公司------多人协作场景比单人更重要。

* 目前还没有一家软件公司在开发者工具之外真正完成了向「卖工作成果」的商业模式转型。我们要在上市之前把这件事做好。

* 我们历史上的弱点是上手难，因为我们给用户的是乐高积木。但语言模型的出现把这个弱点变成了优势------Agent 可以帮你搭乐高。

* 我们在招人上转向更年轻、更 AI native 的人才。很多经验已经不重要了，你只需要会问对的问题。

## **01**

## **Custom Agents 主要解决 busy work**

**主持人：Notion Custom Agents 马上就要公开发布了，这个产品筹备了多久？核心要解决什么问题？**

**Ivan Zhao：** 我们做这个产品快一年了，去年底在 Make with Notion 大会上做了预览，一些公司已经在跑了，下周是正式公开。

我们这次主要瞄准三类使用场景。

第一类是**回答重复性问题** 。大量知识工作，甚至可以说四分之一的知识工作，本质上就是回答那些在某个知识库里能找到答案的问题。我们要把这件事做到极致。

第二类是**分类工单和待办** 。你们可能看过演示------有人在一个 Slack 频道里提问题，Agent 判断应该分给另一个频道的哪个团队。很多 PM 和工程经理的日常工作就是干这个。

第三类是**写状态报告和状态更新** 。那么多团队每周都在汇报「上周做了什么」「昨天做了什么」，这些不需要人来做了。

这三类用例，就是我们说的「忙活」（busy work）。大部分知识工作者的工作里，有很大一块就是这种忙活。Agent 现在不仅能做好，有时甚至能做到超越人类的水平。把这些委派给 Agent，人类就可以往上走一层，去做自己更想做的事情了。

**主持人：你们内部是怎么用 Custom Agents 的？**

**Ivan Zhao：** 在 Notion 内部，工程和产品设计团队是最先用起来的。我们所有的 daily standup（每日站会）现在都由 Custom Agent 完成了------不用自己总结会议纪要，不用说上周做了什么。这些每次省 5 到 10 分钟。另外，Bug backlog 的分类也不再由任何 PM 手动做了。

我们还在 Slack 上部署了一个 Agent 来监控行政事务频道------有人问充电器在哪、办公用品库存怎么样之类的基本问题，Agent 可以直接回复，提醒大家库存情况，如果解决不了就自动创建一个工单让人去处理。不是什么革命性的事情，但确实减轻了很多日常负担。

**主持人：能不能分享一个客户案例？**

**Ivan Zhao：** Ramp 是一个很好的例子。他们的路径其实是两步走。

**第一步是合并。** Ramp 大概 3000 人的公司，之前用了六个不同的协作工具。去年全面迁移到 Notion 之后，首先是省了成本，所有信息都在一个地方了。

**第二步才是 Agent 化。** 一旦你把所有信息放在一个地方，就有了一个统一的上下文给 AI Agent 使用。所以他们现在在全公司部署 Notion，一个地方做 AI 会议纪要，一个地方跑 Custom Agents。

他们最受欢迎的一个 Agent 是给销售团队用的。Ramp 发布功能特别快，左一个右一个，销售团队根本跟不上------最新功能是什么？客户问了这个问题怎么回答？现在他们设了一个 Custom Agent，从 Slack 频道、内部笔记、文档、邮件里抓取信息来回答这些问题。**过去两周，这个 Agent 回答了 4000 个问题。如果每个问题算 30 分钟，那就是大约 2000 个人时。**

**主持人：节省了 2000 个人时之后，人们实际上在做什么不同的事情？团队效率真的提升了？**

**Ivan Zhao：** 说实话，目前**AI 带来的时间节省是「水平型」的，不是「垂直型」的** 。什么意思呢？就是每个人每天省了 5 到 10 分钟，但不是某个人整个岗位的工作被 Agent 替代了。正因为如此，我们还没有看到公司里出现结构性变化------每个人的日子都好过了一点，但这些节省目前还无法精确对应到某个人的全部职责。

但如果问我们团队「如果把 Custom Agent 从你手里拿走」，他们会造反的。模型能力还需要更强，权限和护栏也需要更完善，但趋势已经很清楚了。

02

## **如果你的产品不能被 Agent 使用，**

## **那就很危险了**

**主持人：很多人在讨论「** **SaaS** **是否已死」。Open Claw 的创始人 Peter Steinberger 在 Lex Friedman 的播客上说，未来大部分 App 都会变成** **API** **调用。你怎么看 Notion 在这个趋势下的生存问题？**

**Ivan Zhao：** 我觉得这是一个「双城记」式的故事。

一方面，**软件被使用、被消费、被构建的方式正在发生根本性变化** 。你不再只是为人类构建产品，你也在为其他语言模型和 Agent 构建。如果你的产品不能被 Agent 使用，我认为未来不会乐观。但如果你的产品能被重新架构、被 Agent 使用，你就能进入一个科技行业从未触及过的全新市场。

过去 15 年，科技行业卖的主要是工具------SaaS 订阅、按座位收费。但更大的市场不是卖工具，而是卖工作和服务，那个市场大 10 倍。如果你能让你的产品和技术栈去服务那个市场，那才是真正的机会。问题只是：你能不能足够快地重新架构你的产品来搭上这波浪潮？这就是为什么我们一直在不断重构 Notion。

**主持人：那些不为 Agent 重新设计的软件公司，你觉得会怎样？**

**Ivan Zhao：** 很简单。假设一个产品是 90 年代到 2000 年代初的风格，屏幕上有一千个按钮------Agent 根本搞不懂怎么用它。如果 Agent 不会用你的产品，而未来大部分知识工作将由 Agent 完成，那你就会丢市场份额。

其次是商业模式的转型。你不能只卖座位了，你得想怎么卖使用量、卖工作、卖成果。但说实话，还没有一家公司在开发者工具之外真正做到了这一点。大家都在互相观望，但在未来一两年内我们会找到答案。

**主持人：所以一些公司不愿意开放** **API** **和数据端口，是因为害怕被「掏空」？**

**Ivan Zhao：** 未来确实不确定，用即时战略游戏的比喻来说，前方有很多战争迷雾。不确定的时候，人们倾向于保守。不想一不小心把自己的护城河抽干了。

但至少在 Notion 这边，我们的信念是：**当其他人在关门的时候，恰恰是开门的时机。** 不只是欢迎人类来读写你的产品，也要欢迎 Agent 来读写。你可以通过提供真正好的、深思熟虑的用户界面来赢。你可以通过做「瑞士」、跟最好的模型合作来赢。除了下周的 Custom Agents，我们接下来几周和几个月还会发布更多面向开发者的功能。**Notion 历史上不是一个很开发者导向的公司，但这即将改变。**

03

## **上手难是 Notion 多年的弱点，**

## **AI 把它变成了优势**

**主持人：Notion 一直有个问题------功能太强大，上手太难。你觉得 AI 改变了这个局面吗？**

**Ivan Zhao：** 是的。这在某种意义上一直是我们多年来的弱点------**我们给用户的是乐高积木。** 但这个时刻反而变成了一种恩赐。

你想想传统 SaaS 是怎么做的：把工作流硬编码到软件逻辑里，这就是你卖的东西。所有东西都是刚性的，但它强大、自动化、在云端。大多数 SaaS 都是这样------工作流被固化了。Notion 的乐高积木恰恰相反，非常灵活，但正因为灵活，历史上一直难以上手。

**现在有了大模型，模型可以替你搭那些乐高。** 这就把我们的弱点变成了优势------现在你可以在 Notion 里做各种开放式的事情、各种不同风格的知识管理。

最典型的例子就是数据库。过去你得从零开始搭行、列、关系结构，现在 Agent 直接帮你搭。目前 Notion 里超过 50% 的数据库是由 Agent 创建的，而不是人工创建的。

**主持人：《华尔街日报》的 Joanna Stern 也是因为 AI 才终于「理解」了 Notion？**

**Ivan Zhao：** 是的，她发推说「我终于因为 AI 而理解了 Notion」。我之后私信了她，准备一对一帮她真正上手。这件事说明两点------一方面我们的新用户引导确实可以做得更好，但另一方面也说明我们的 AI 确实很有魔力，你让它做什么它都能做到。Agent 帮你搭数据库这件事，可能就是让她突然「理解 Notion」的关键。

**主持人：说到 Notion 的协作功能------你们到现在还看不到别人实时打字，Google Docs 还有这个优势吧？**

**Ivan Zhao：** 我们在架构上能做这个。问题只是：是把更多时间花在 AI 上，还是回头去做 Google Docs 竞争那一套？光标和高亮在实现复杂度上是一样的。加一个 to-do 吧。

**主持人：作为** **CEO** **，你每天怎么用 Agent？**

**Ivan Zhao：** 每天早上我会收到一个 Agent 给我的简报。我念一下今天的------第一条，确认一个域名，因为我们在为新产品买域名；第二条，为今天下午的播客录制做准备，简报里附了日历和邮件里的相关链接；第三条，下班前审核招聘数据库里的候选人。我们内部有一个招聘数据库，Custom Agent 每天会通知我哪些人有 offer 要发出，我每天审核这些 offer。

另外，**我基本上不再以传统方式查看邮箱了。我主要通过一个 Custom Agent 来管理收件箱。** 它了解我在做什么，了解我关心什么样的事情，帮我筛选和归档。我可以直接告诉它「今天不重要的邮件帮我归档掉」，而不需要自己点 50 个按钮去归档。

还有一个超级强大的 Agent------连接了我们的 Snowflake 后端。这意味着我可以查询公司内部几乎任何数据，而且带有 Slack、Notion 文档、邮件的上下文评注。感觉就像口袋里装了一整个数据科学团队**。** 这种能力是从编程 Agent 革命以来才有的，非常新。把各种不同工具接进来，感觉几乎什么都能做。

04

## **卖座位是科技行业，**

## **卖工作成果是知识经济**

**主持人：Notion 目前还是按座位收费的模式，对吧？你们真的要转向 usage-based（按使用量计费）吗？**

**Ivan Zhao：** 是的，我们正在做大量实验。**Custom Agents 是我们第一个按使用量计费的产品** ，客户对此非常接受。跟 Ramp、Vercel、Clay、Cursor 这些公司聊，你问他们从 Custom Agents 中获得了什么价值------他们在节省实实在在的人时，这些可以转化为可观的金额，远超座位费。

人们接受这种模式，因为它创造的价值是看得见的。

**主持人：Brett Taylor 在 Sierra 也在讲同样的事情------为什么不按成果收费呢？但按座位收费有点像健身房会员------你大概能从很多不使用的座位上赚钱，对吧？而 AI 服务又很贵，你们已经是模型厂商的大客户了。这里面取舍应该不少。**

**Ivan Zhao：** 卖工具、卖座位，你还是在科技行业里。当你能卖工作、卖使用量、卖成果的时候，你进入的是知识工作经济------那是 10 倍大的市场。

这才是更多软件公司应该思考的方向------不要只想着「有人能用 Vibe Coding 自己做一个产品然后压缩你的利润」，那是向内看的。**向外看的话，整个知识经济里有那么多忙活在等着被自动化，那才是更大、更有意思的市场。**

**主持人：你们做了一轮不小的 tender offer（股票回购），还取消了新员工的一年 cliff（股票锁定期）。Figma 已经上市了，大家都在问 Notion 什么时候上市。你怎么想的？**

**Ivan Zhao：** 我们内部说得很清楚：**最重要的事情是在 IPO 之前完成向 usage-based 商业模式的转型。** 在知识工作领域，这是最大的市场之一，但目前还没有一家产品在开发者工具之外真正完成了向「卖工作」的转型。我们趁着还是私有公司的时候可以冒更大的风险、动得更快，也可以观察公开市场的走向。这可能在接下来两三个季度就会搞定。等 Custom Agents 全面上线、新定价模型跑通之后，我们再谈 IPO 的时间。

05

## **Open Claw 是单人版，Notion 要做企业版**

**主持人：在 AI Agent 的世界里，Notion 到底占据什么位置？怎么跟 Open Claw、Claude Cowork 这些玩家区分开？**

**Ivan Zhao：** 如果你看整个市场------Open Claw 在 Mac Mini 上跑，面向极客，单人使用，很强大但复杂。Claude 的 Claude Cowork 面向知识工作，处理文件很厉害，律师和财务团队喜欢，Anthropic 确实在文件处理上 all in 了，但因为以文件为核心，还是偏单人使用的。

**我们想占据的位置是：从一开始就为多人协作设计、跑在云端、为商业和团队协作而生。** 这个定位目前没有太多人在做，而我们有先发优势------协作产品做了五年多，文档和项目管理的所有积木都在，AI 产品也做了两三年。

这也涉及到一个关键区别。语言模型的优点是自由------任何人只要会说英语或日语就能修改东西。语言模型的缺点也是自由------任何人都能修改任何东西。在企业环境里，你不能什么都自由。有些人应该看到某些内容，有些人不应该。某些工作流应该由专家定义，别人不应该随便改。你需要权限的护栏、安全的护栏------这些东西无聊但极其重要。所以 Open Claw 你可以在 Mac Mini 上跑，管理你自己的生活，那是一个人的事。**但如果是一家公司呢？那就是 Open Claw 的精神------异步的、几乎能做你在电脑上做的一切------加上多人协作、企业级权限和安全。**

另外，做模型的「瑞士」也有巨大价值。现在有三四家模型公司在互相超越，没有人想被锁定在任何一家。OpenAI、Anthropic 发布新模型，当天或者第二天我们就上线------客户需要这个。所以我们的定位是：**多人协作界面、模型中立、真正面向** **B2B** **。** 这个位置在市场上是全新的、大开的，也是我们正在全力进攻的方向。

**主持人：那为什么要收购邮件和日历产品？**

**Ivan Zhao：** 我们想建自己的管道（pipes）。两年前收购了邮件和日历产品，长远来看是要打通自有管道。但现在有一个更快创造价值的方式------把邮件和日历做成最好的 Agent 工具调用能力。

模型能力已经够了，差不多一年了。缺的是基础设施和管道------高质量的、彼此衔接的工具调用能力，权限管道如何打通。这就是我们在建的东西。我们已经把邮件和日历团队重新定位，专门去做这些管道。近期你会先看到 computer use 和 browser use 的能力，更往后才是完全打通的管道------两条路径，两个节奏，最终都会到位。

06

## **能不能同时管一千个 workspace？**

## **限制因素是人类界面**

**主持人：「workspace」这个词 Microsoft 和 Google 都在用。它还是正确的概念吗？在 Agent 时代，人们会怎么谈论这个东西？**

**Ivan Zhao：** 我不太喜欢这个词，但它已经变成了约定俗成的说法。不过它背后的意义仍然很重要------你有一个集中的地方，一个记录系统，记录工作流、驱动你的项目和业务，一群人看着同一个东西、一起工作。这件事不会消失，在 Agent 时代只会变得更重要。

因为你可以邀请不同的 Agent 进入你的 workspace------Claude Cowork Agent、Cursor Agent 进来理解你的业务在做什么，在你的 workspace 里工作，然后去 GitHub 上做点什么。

**过去，大部分知识工作其实就是人在不同的 workspace 之间搬运信息。** 现在我们有了这种「无限心智」（infinite minds）的神奇材料来帮我们做这件事。问题变成了：我们能不能不只管一个 workspace，而是管 10 个？能不能同时玩一千个 workspace 的 Sim City？也许可以。**但限制因素还是人类的界面。**

**主持人：说到界面------大家在试各种管理 Agent 的形态，你觉得什么样的界面才是对的？**

**Ivan Zhao：** 有人说 90 年代和 2000 年代的即时战略游戏可能是管理 Agent 的好形态。看板也是------过去一两年大家在用 IDE 来管理 Agent，现在又回到了管理多个 Agent 的场景，结果发现最好的方式还是看板。我们还在寻找最佳形态。

核心原则是：**人类想留在 loop（循环）里，但你得提升 loop 的层级。** 你不想在做忙活的那一层循环里，你要往上走。从管理具体工作，到管理一座城市。我们仍然在摸索什么样的界面能让我们真正提升到那个层级。

07

## **一个小团队加编程 Agent，几周做出一个新 App**

**主持人：你怎么想 Notion 的产品矩阵------是把文档、日历、邮件全塞进一个主应用，还是保持独立 App？**

**Ivan Zhao：** 我最近其实在改变自己的想法。

因为现在用编程 Agent 做东西实在太快了------**我们很快会发布一个独立的 Agent Chat 产品，把所有商务聊天需求整合到一个地方。** 以对话为核心的 Notion，拥有 Notion 里所有的上下文，可以用来管理日历和邮件。这是一个独立的 App，一个小团队加编程 Agent，几周就能做出来。

**主持人：那 Slack 呢？Notion 是不是就差一个消息应用了？**

**Ivan Zhao：** 呃......是的。

**主持人：日历和邮件在未来还需要独立的 App 吗？**

**Ivan Zhao：** 日历我觉得会一直存在。这种信息密度很高的时间展示方式已经存在了一百多年，很难被取代。

邮件的话，取决于你要什么样的信息密度------如果你是专业销售人员，整天泡在收件箱里，同时看五到十封邮件，那界面仍然很重要。但对我来说，我每次只挑三四封重要的邮件，所以界面就不那么重要了。

我的预测是，**未来两年内，邮件会变成一个被 Agent 垃圾邮件严重淹没的媒介。** 有 Agent 管收件箱，有 Agent 写邮件，来回的垃圾信息会让邮箱变得过于饱和。到时候会怎样，我们走着看。

08

## **招了个 16 岁的工程师，在写下一代产品**

**主持人：像 Shopify** **CEO** **Tobi Lutke 说「除非能证明 AI 做不了，否则不允许招人」。你们在招聘标准上有类似的变化吗？**

**Ivan Zhao：** 没有那么严格，但我们确实在找非常 AI native 的人。整体上，**我们在向更年轻、更早期职业阶段的人倾斜。**

去年年中我们招了一个 18 岁的，觉得已经是最年轻的了，刚高中毕业要上大学。结果过了一个月，我们在 YouTube 上发现了一个人------不是在 Twitter 上出名的那种，就是在做一些很酷的关于设计和 AI 的视频。我们邀请他来办公室，我在电梯里碰到他，问「你在哪儿上的大学？」他说「我没上大学」。我说「哦你退学了？」他说「不是，我还在上高中，10 年级。」

但这个孩子太有才华了，他现在是我们下一代产品的工程师，在写其中很大一部分代码。我们计划五月份发布，届时更多人可以用到。

**主持人：为什么觉得 AI 时代要倾向于招更年轻的人？**

**Ivan Zhao：** 他们拥抱新工具更快，更少预设假设------很多事情你「假设」它做不到，但这个假设每两个月、每个月就在被推翻。他们更乐观、更饥渴。而且**很多经验已经不重要了，你只需要善于提出正确的问题。**

另外，这些孩子完全是在互联网里长大的，他们的学习速率和我们这一代是不同的复利曲线。再加上过去几年有 ChatGPT 陪着长大，那又是另一个量级的复利。

09

## **AI 时代，Notion 仍站在「工具」这一边**

**主持人：Satya Nadella 在达沃斯把你和 Steve Jobs、Bill Gates 放在一起引用了------他特别喜欢你[那篇文章里的「Manager of Infinite Minds」（无限心智的管理者）](https://mp.weixin.qq.com/s?__biz=Mzg5NTc0MjgwMw==&mid=2247521881&idx=1&sn=bc5439c4fce07ed3ae42c9dfaf2d6224&scene=21#wechat_redirect)** **这个概念。什么感觉？**

**Ivan Zhao：** 很多人实时发消息给我，我非常受宠若惊。微软是我们敬仰的公司------它把个人电脑放到了每张桌子上、每个家庭里，创造了这个行业。Steve Jobs 是我进入科技行业时的偶像。而我觉得 Satya 是真正懂行的人------据我所知他是一路从 Excel 做起来的，做过微软 90 年代的核心产品，他知道自己在说什么。看到他引用我们假期发表的东西，是一种巨大的荣幸。

**主持人：这个短语------「Manager of Infinite Minds」------你是怎么想出来的？**

**Ivan Zhao：** 说实话我也不确定。**可能就是 Notion AI 帮我改了一千遍之后某些词浮现出来了，我大概更多是扮演编辑的角色。** 读到的时候觉得「嗯，感觉对了」，就留下了。

我喜欢用隐喻来思考。隐喻能帮我从不同角度看问题，也能让其他人产生共鸣。比如 Steve Jobs 说「自行车为心智而造」（bicycle for the mind），但你想想------我们有互联网已经 30 多年了，到现在还在踩自行车在信息高速公路上跑。接下来的变化是：我们要在信息高速公路上开汽车了------通过 AI**。** 当我把这个隐喻告诉团队和其他人的时候，他们马上就懂了。

**主持人：如果你当年有这些 AI 工具，会不会走一条完全不同的路？**

**Ivan Zhao：** 好问题，从来没人问过我这个。

计算机科学历史上一直有两个学派。东海岸是 AI 学派------从 MIT 的 John McCarthy 开始，教计算机下国际象棋，那是 1950 年代的事。西海岸是 HCI（人机交互）学派------Douglas Engelbart、Alan Kay，他们讲的是「增强人类智能」。

这两个学派一直是计算领域的两极。现在 AI 这一侧的力量确实增强了很多，但吸引我当初做 Notion、吸引我接触计算机的，是界面、工具、craft（手艺）那一面------创造某种你可以握在手里的、能放大人之所以为人的那些东西的形状。

如果让我现在重新进入科技行业，我想我还是会被那个方向吸引。来过我们办公室的人知道，我们非常在意工具和 craft。但现在有意思的地方是：AI 变得如此强大，如何用这种力量去创造全新形态的工具------这个问题还没有人真正回答过。
