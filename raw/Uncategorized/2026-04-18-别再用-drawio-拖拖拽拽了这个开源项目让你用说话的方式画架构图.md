用 Mermaid 要学语法，写几十行 DSL 还不一定对齐；打开 draw.io 拖拖拽拽半小时，换个颜色又要重来；截图粘贴到文档，分辨率一放大全是锯齿。最后交出去的图，连自己都觉得拿不出手。

今天要介绍的这个开源项目，就是专门干这件事的。

*** ** * ** ***

## 简介

fireworks-tech-graph 是一个 Claude Code Skill，装上之后，在 Claude Code 里用自然语言描述系统，它自动生成 SVG 格式的技术图表，同时导出 1920px 宽的 PNG 文件。

一句话定位：**用说话的方式画图，得到的是可以直接放进文档和 PPT 的高清技术图。**

支持 7 种视觉风格，覆盖 14 种 UML 图类型，还内置了一批 AI/Agent 领域的常见架构模式（RAG、多 Agent、Tool Call 流程等）。

## 支持哪些图

### AI/Agent 领域的内置模式

这是这个项目比较有意思的地方。对于 AI 工程师常画的那几类图，项目里已经把结构规则都写进去了，不用从零描述。说一句「画一个 RAG 流程图」，它就知道应该有哪些节点、箭头该怎么走。

内置的模式包括：

*
  • RAG Pipeline（查询→嵌入→向量检索→召回→LLM→返回）
*
  • Agentic RAG（在 RAG 基础上加 Agent 循环和工具调用）
*
  • Agentic Search（查询→规划器→并行检索/计算/代码→合成器）
*
  • Mem0 内存层（输入→内存管理器→向量库+图数据库→上下文）
*
  • 五类 Agent 记忆（感知/工作/事件/语义/程序性）
*
  • Multi-Agent（协调者→多个子 Agent→聚合→输出）
*
  • Tool Call 流程（LLM→工具选择→执行→解析→回 LLM，循环）

### 14 种 UML 图

完整支持 UML 标准里的 14 种图类型：类图、组件图、部署图、包图、复合结构图、对象图、用例图、活动图、状态机图、时序图、通信图、时序图、交互概览图、ER 图。

### 形状和箭头有固定含义

画出来的图不只是「好看」，每种形状代表固定的语义：

*
  • LLM/模型：圆角矩形，双边框，带 ⚡ 图标
*
  • Agent/编排者：六边形
*
  • 向量库：带内圈的圆柱体
*
  • 图数据库：三圆簇
*
  • 工具/函数：矩形带 ⚙ 图标
*
  • 决策节点：菱形

箭头颜色和线型也有规则：实线是主数据流，虚线表示写入操作，弯曲箭头是反馈循环。这样，图本身就在传递信息，不需要额外文字解释。

## 安装和使用

### 安装

先装依赖（用于 SVG 转 PNG）：

    # macOS
    brew install librsvg

    # Ubuntu/Debian
    sudo apt install librsvg2-bin

然后安装 Skill：

    npx skills add yizhiyanhua-ai/fireworks-tech-graph

### 基本用法

装好之后，直接在 Claude Code 里说：
> 画一张 Agentic RAG 架构图，Agent 核心层包含 Planner 和 Tool Selector，Memory 层包含 VectorDB 和 GraphDB，Dark Terminal 风格
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2F38IO1Mnhnaz9r4p3O5xxPWicfcDYdjgy1gfn3jWgFpu0MV9ZFDmPFKEqbBX8hDW2rmMico0hEs97PP0M2QX85xwUZaEsUcKzaJXCHPB0W2vS4%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D8)

输出的文件是 SVG（可以用向量编辑器继续改）和 1920px 宽的 PNG（直接放文档或 PPT）。

### 更新

    npx skills add yizhiyanhua-ai/fireworks-tech-graph --force -g -y

## 适合谁用

如果你经常要写技术文档，每次画图都这么麻烦，这个工具就是为你准备的。在用 Claude Code 的同学更直接，装好之后用自然语言说说要画什么，连图一起就出来了。做 AI 应用开发的同学也会视为实用，RAG、Agent、记忆架构这类图都能直接论上。

当然，如果你只是偶尔在 Markdown 里插一个简单示意图，Mermaid 就够用了。如果你需要精细拖拽每一个元素的位置，draw.io 更合适。工具没有天下第一，选适合自己场景的就好。
