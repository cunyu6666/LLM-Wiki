## 🕳️ 先捅痛点：VS Code 为啥这么肥？

VS Code 这玩意儿，看着是个编辑器，实际上是个**披着编辑器皮的双系统** ：

Electron 本质上 = Chromium + Node.js 打包在一起的超级 bundle。每个 Electron 应用都要自带一套完整的浏览器内核，不管你电脑里是不是已经装了 Chrome。

VS Code 光安装包就 775MB，运行时内存 500MB 起，稍微装点插件，1GB 内存直接蒸发。

更骚的是------如果你电脑上同时跑 VS Code、Cursor、Windsurf、Figma（也是 Electron）......恭喜，每一款都自带一套 Chromium。内存占用直接原地起飞。

Tauri vs Electron 架构对比 --- 用系统原生 webview 替代内置 Chromium

## ⚡ SideX 出世：一刀砍掉 96% 体积

SideX 干的事情说白了就一件：**把 Electron 换成 Tauri** ，其他所有东西照搬。

等等，这里有个误解------这不是什么"VS Code 风格"的克隆，而是真正的 VS Code 源码树，5600+ 个 TypeScript 文件，完整移植。

作者把 Electron 的 main process 整体用 Rust 重写了：

|      Electron 替换方案      |         Tauri / Rust 实现          |
|-------------------------|----------------------------------|
| ipcMain / ipcRenderer   | invoke() + events                |
| Node.js APIs (fs, path) | Rust std::fs / std::path         |
| node-pty (终端)           | portable-pty                     |
| 内置 Chromium             | 系统原生 Webview (WebKit / WebView2) |
| @vscode/sqlite3         | rusqlite                         |

Tauri 的骚操作是------**不内置 Chromium，直接用系统自带的 webview** 。macOS 用 WebKit，Windows 用 WebView2（Edge 内核），Linux 用 GTK WebKit。

这些 webview 系统里本来就有，压根不用打包进安装包，体积自然就下来了。

## 🧠 Rust 后端到底干了啥？49个命令拆解

别以为 Tauri 那头是啥薄薄的一层 wrapper，SideX 塞进去了 49 个 Rust 命令，分布在 9 个模块里：

|       模块       | 命令数 |                        功能说明                        |
|----------------|-----|----------------------------------------------------|
| Git            | 17  | status, diff, log, branch, stash, push/pull, clone |
| Terminal       | -   | 真实 PTY，portable-pty 驱动                             |
| File System    | -   | read, write, stat, watch (notify crate)            |
| Storage        | -   | SQLite via rusqlite                                |
| Search         | -   | 递归文本搜索 + 智能过滤                                      |
| Extension Host | -   | Node.js sidecar，VS Code 扩展兼容                       |
| HTTP Proxy     | -   | CORS bypass，接 Open VSX 市场                          |

Git 功能尤其猛，17 条命令覆盖了日常开发 90% 的操作------status、diff、log、branch、stash、push/pull、clone，全给你焊好了。

## 🚀 部署教程：手把手把 SideX 跑起来

### 🔧 第一步：装环境

SideX 依赖 Tauri，Tauri 依赖 Rust 和 Node.js。先把这些伺候好：

Tauri 开发环境先决条件 --- Windows 需要 Visual Studio Build Tools

|   平台    |                         依赖                         |
|---------|----------------------------------------------------|
| Windows | Rust + Visual Studio Build Tools (勾选"C++ 桌面开发")    |
| macOS   | Rust + Xcode Command Line Tools                    |
| Linux   | Rust + libwebkit2gtk-devel, libssl-dev, pkg-config |

**老司机排雷** ：Windows 下装 Visual Studio Build Tools 别瞎点，左侧选"使用 C++ 的桌面开发"，右侧把"Windows 11 SDK"和"MSVC v143"勾上，不然 Tauri 编译铁定报错。

### 📥 第二步：克隆代码


git clone https://github.com/Sidenai/sidex.git  
cd sidex  
npm install

**老司机排雷** ：这个仓库 5600+ 个 TypeScript 文件，npm install 会慢到怀疑人生。有条件的开个代理，或者耐心等着，别以为卡死了。

### ⚡ 第三步：开发模式跑起来


npm run tauri dev

**Linux 用户注意** ：如果遇到白屏/空白窗口，试试这个：

WEBKIT_DISABLE_DMABUF_RENDERER=1 npm run tauri dev

### 🏗️ 第四步：生产构建

前端编译需要大内存，这玩意儿源码太大：

NODE_OPTIONS="--max-old-space-size=12288" npm run build  
npm run tauri build

**老司机排雷** ：12GB 内存是底线，不够的话前端编译直接 OOM。换台机器或者加内存，别硬怼。

## 💥 踩坑与局限性：这玩意儿能用于生产吗？

先泼盆冷水------SideX 目前是 Early Release，很多功能在 stub 或 partial 状态：

|       功能       |                状态                |
|----------------|----------------------------------|
| 核心编辑器 (Monaco) | ✅ 可用：语法高亮、基础 IntelliSense        |
| 文件管理器          | ✅ 可用：打开文件夹、创建/编辑/删除文件            |
| 扩展 Host        | ⚠️ 早期：部分扩展可能不工作                  |
| 终端             | ⚠️ 开发中：shell 集成、profile 检测待完善    |
| 调试器            | ❌ 未实现：Debug Adapter Protocol 待开发 |

还有几个具体的坑：

🔴 WebView 兼容性：Linux 下部分发行版 WebKit 版本太老，可能白屏。macOS 和 Windows 10/11 基本 OK。

🟡 文件监控：notify crate 在大文件场景下稳定性一般，跑大项目可能丢事件。

🟡 大文件处理：文件系统这块还没针对超大文件做优化，别拿它开 10GB 的日志。

结论：**目前适合尝鲜、内网开发、轻量化场景。拿来当主力 IDE？再等等。**

## 🤖 对 AI 编程工具的影响

这是 SideX 最骚的地方------**它证明了这个架构可以跑通** 。

现在市面上的 AI 编程工具，Cursor、Copilot、Windsurf、Cline，全都是基于 VS Code 的 Electron 栈。这意味着每装一个 AI 编程工具，都要附带一份 Chromium。

SideX 把这条路走通之后，未来可能出现：

• Cursor Tauri 版：内存占用砍一半，续航飙升

• Copilot Desktop：本地 AI + 轻量编辑器，老机器也能跑

• Windsurf Nano：嵌入式 AI 编程，小设备专属

不管 SideX 本身能不能成，它验证的这条路劲------**AI 编辑器 + Rust 后端 + 原生 webview** ------才是真正的未来。

项目结构也很清晰：

src/  
├── workbench/ # IDE 外壳、功能、服务  
└── main.ts # 前端入口  
src-tauri/src/  
├── commands/ # Rust 命令：fs, terminal, search, git, window  
├── lib.rs # App 配置、菜单、命令注册  
└── main.rs # 入口  

**开发 tip** ：TypeScript 导入带 .js 扩展名（ES modules 规范），Services 用 VS Code 的 @inject 装饰器做依赖注入。Rust 新命令往 src-tauri/src/commands/ 里怼，然后去 lib.rs 注册。
