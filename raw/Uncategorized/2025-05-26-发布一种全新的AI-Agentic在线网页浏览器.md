🎬 场景  
想象一个类似Cursor+Chrome/Safari的体验，在用户阅览一个网页的同时，AI就可以帮用户把整个网页目录按级分类准备好了，同时也是一个用户的网页知识库。  

⛳️ 目标  
在各种强力AI模型加持下，不断创造一种全新、高效、纯净的阅读、学习、科学研究、Vibe Coding、工程开发的方式。  

🌟 特性 - 专为AI设计与打造的Agentic Web Browser （可直接网页浏览，无下载非APP）  

1️⃣ 毫秒级的极速+免费提取任意网页内容为Markdown各式，可以一键Copy内容喂给任意AI  

2️⃣ 快速抓取当前网页下的所有子页，并将他们分级整理去重后，返回一个可供大模型导航的 sitemap 🔗链接地图树🌲，类似llms.txt但更精简，可免去做RAG的必要  

3️⃣ 可以自由手动切换实时浏览当前页面的实际界面，几乎囊括原网页的大部分功能  

4️⃣ 内置浏览器的操作也可以完全由AI模型控制  

5️⃣ AI可对当前用户浏览的网页内容有感知和Context，用户可以直接提问或者提要求  

6️⃣ 包含免费基础主流大模型以及付费订阅的高级大模型  

目前产品刚刚发布，产品会持续迭代开发，长期目标是打造世界上最好用，功能最强大的免费浏览器内浏览器，包括但不限于，实时在线搜索、可直接根据当前网页或者用户收集的所有网页、生成图片、代码、页面或可部署的网站，或者其他一系列Agent可代替的工作。也可在任意网页前加上 "llml.ai/" 将当前网页打造成一个图文并茂的知识库，类似deepwiki那样。  

📞 API \& MCP  
后续会把所有后端核心业务打包成独立API服务供开发者整合进现有或者打造全新的AI Agent或应用服务。  

🔩 技术栈  
核心：自主研发开发打造，不依赖🔥Firecrawl等现成的API（我的API发布后，就是冲着争夺Firecrawl市场去的，性能速度可达他的3-10倍，价格却可以做到1/3甚至更低！）
