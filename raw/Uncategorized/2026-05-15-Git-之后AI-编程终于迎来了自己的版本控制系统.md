用 Claude Code、Cursor 这些工具写代码，效率确实高了，但也多了些烦心事。

AI 改完代码突然崩了，想回滚却不知道改了哪；看着一行陌生的代码，想不起来是自己写的还是 AI 生成的，更不知道当时给了什么提示词；同时开几个会话改不同功能，最后搞混哪个会话改了哪个文件。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FSic6125iczk8qneF7EJgP4oXbcsnRpWgia4r9RlHLWontYFMlTyldKlbCzSuIaYmgmTPsahXQxnDcicibWiaXlv0f0BPYKsxagKwxjj4NF7tczSGY%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D0 "null")

传统版本控制只能跟踪代码变化，记录不了AI的活动。re_gent做的就是这件事！


### re_gent 是什么

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FSic6125iczk8oxHeicbejtaBbHkRR0Sed2dwQSYsicoUPw8LDoaMl2u7uybTJmXDiaibIWotrwLbeeLflul09CTBo1kfB60nibNIicicWLmT6Veuep4Q%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D1 "null")

它是给 AI 编程代理设计的版本控制系统，把 Git 的版本控制思想延伸到 AI 活动层面。不只是记录代码变成了什么样，还记录代码**为什么** 变成了这样。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FSic6125iczk8pjK6zGxosD317ciaiceZaGSvqXtYuFDVLVVF4KTUQvAq2LcA9hJzaTMHmHhiaDnwXicQs3XaLXeHMeZaoQgGPm7GGiaqIL6mgfOVTU%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D2 "null")

最省心的地方是全自动捕获。不用改工作习惯，也不用手动提交，正常使用 AI 工具就行------每次编辑文件、写入代码、执行终端命令都会被自动记下来。对开发者和 AI 都是透明的，没有额外负担。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FSic6125iczk8r5ibpvFFeiacVBavDbnTksXROd803QOJWwXFwpic8L25pPzI4Ck5cqsRm3CUIKiaxECxLJms4p0jnN0KuwxBQUyElfLGxiaGa08Rvo%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D3 "null")

有了完整记录，追溯问题就简单多了。一条命令查看某行代码的来源：哪个步骤改的、用了什么工具、当时输入的提示词是什么、AI 的回复内容也完整保留。哪怕在 Claude 里用了 /compact 或 /clear 清理了聊天记录，re_gent 里的历史也不会丢。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FSic6125iczk8pbvuZsr0mLsfnAyqpuVYjia9xUj0SB7MvwKIrWicet5IictesxXAMp2qrRoUQAibLsZI1u8ywdzYX7shNvicz77t0QvfzBD2KPVLvE%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D4 "null")

同时处理多个任务时，re_gent 会给每个 AI 会话创建独立分支，多个会话可以同时工作互不干扰。能随时查看每个会话的操作轨迹，也能单独过滤某个会话的历史，不会再出现几个会话改乱同一个文件的情况。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FSic6125iczk8p8ic2Pp3zDVF0yKjrACP23FHo3Dics94piczE0icx75frXkicldXa0BDqT5CI5JXOl5u3eu8icIatBibYsZXmCIxuSFnYS07yCF1VcJI%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D5 "null")
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FSic6125iczk8qor68sbJA1lRsIVnaxf2AKkNtOxzk1DhIFEeOJxd9XTyxAibZJCOrWn7ru4h7vDfwsa47MoPouHIovqO0YJd3537tMBhGt32h0%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D6)

re_gent 不是要替代 Git，而是作为补充。Git 跟踪开发者的手动提交，re_gent 跟踪 AI 的自动操作，两者共存互不影响。

### 快速上手

#### 安装

**Homebrew（macOS/Linux）**

    brew tap regent-vcs/tap
    brew install regent

**Go 安装（跨平台）**

    go install github.com/regent-vcs/regent/cmd/rgt@latest

安装后得到 rgt 和 regent 两个命令，用法一样，看习惯用哪个。

#### 基本使用

1.
   1. 进入项目目录初始化

    cd your-project
    rgt init

1.
   2. 正常使用 Claude Code 等 AI 编程工具  
   AI 的工具调用会自动跟踪，不用额外操作
2.
   3. 查看 AI 的活动历史

    # 查看所有操作记录
    rgt log

    # 查看某一行代码的来源
    rgt blame src/file.go:42

    # 查看所有活跃的 AI 会话
    rgt sessions

    # 查看特定会话的历史记录
    rgt log --session claude-20260502-143021

    # 查看某个步骤的完整上下文，包括对话内容
    rgt show <step-hash>

#### VSCode 扩展

习惯用 VSCode 的话可以装官方扩展，从 GitHub Releases 下载 .vsix 文件，然后在 VSCode 里通过 "Extensions \> ... \> Install from VSIX..." 安装。

装完后编辑器里直接能看到每行代码的修改来源，鼠标悬停查看步骤信息，侧边栏显示会话时间线。

### 项目现状

官方路线图显示，团队正在开发第三阶段的高级功能：从任意步骤创建新会话的 rgt fork，以及非破坏性时间回溯的 rgt rewind。  
这两个功能完成后，可以放心让 AI 做各种尝试，不满意随时回到过去的状态。

如果你经常用 AI 编程工具，且偶尔会为"这段代码谁写的、为什么这么写"困惑，可以试试看。
