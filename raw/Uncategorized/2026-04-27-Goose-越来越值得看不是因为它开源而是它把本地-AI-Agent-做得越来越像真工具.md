# Goose 越来越值得看，不是因为它开源，而是它把本地 AI Agent 做得越来越像真工具

如果最近只能选一个值得认真看看的一线开源 Agent 项目，我会把 Goose 放得很靠前。

不是因为它只是"又一个开源替代品"。

而是因为它越来越像一个能真的被拿来干活的本地 AI Agent。

Goose 现在最有意思的地方，在于它不是只做一个终端里的玩具助手。

GitHub 仓库现在把它定义得很清楚：

桌面端、CLI、API 都有，而且不只服务代码场景，还覆盖研究、写作、自动化、数据分析这些更通用的工作。

这其实很重要。

因为很多 Agent 项目的问题，不是概念不对。

而是最终只停留在"帮你写点代码"。

但真实世界里，大家想要的不是一个局部脚本精灵。

而是一个能在自己机器上工作、能接不同模型、能调不同工具、还能被真正嵌进日常流程的系统。

Goose 现在走的就是这条路。

仓库页面显示，截至 2026 年 4 月 25 日，它已经有 4.32 万 star，最新 release 是 2026 年 4 月 23 日的 v1.32.0。

官方 README 还写得很直接：

它支持 15 以上模型提供方，也能通过 MCP 连接 70 多个扩展。

这说明 Goose 的价值不只是"它能跑"。

而是它在往一个开放的本地 Agent 平台走。

还有一个特别值得注意的变化。

Goose 这个项目已经从 Block 迁移到 Linux Foundation 旗下的 Agentic AI Foundation。

这背后的信号很强。

说明开源 Agent 这件事，正在从单一公司项目，往更中立、更标准化的生态基础设施靠。

你可以把它理解成：

大家开始不只是比谁家的 Agent 更聪明。

也开始比谁能把 Agent 生态做成真正可互通、可扩展、可治理的公共底座。

这也是为什么我觉得 Goose 很适合写公众号。

因为它不是那种只有开发者才会兴奋的小众项目。

它代表的是一个更大的方向：

本地 Agent 要想真有生命力，必须同时解决三件事。

第一，用户真的能装、能跑、能上手。

第二，模型和工具不能被单一平台锁死。

第三，它得足够开放，能长出生态。

Goose 现在还远没到终局。

但它已经明显不是"一个会写代码的开源壳子"这么简单了。

它越来越像本地 Agent 赛道里，真正开始有产品味道的一批玩家。
