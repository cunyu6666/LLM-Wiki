**和大部分AI产品一样，如果只是简单描述，AI通常会给你一个很标准的页面** ：上面是一个大标题，下面是几块功能介绍。结构没有问题，但问题就是这太普通了，不会吸引到任何用户。

3. **使用方式**

4. **边界和原则**

**Prompt模板**

**先描述产品核心：** 这是一个面向（用户）的产品，他们的核心问题是（要解决什么问题）；

**接下来描述使用方式：** 这是一个（一步完成 / 分步骤 / 引导式）的流程，重点是让用户在过程中感受到（参与感 / 效率 / 探索 / ... ）；

**同时需要明确边界：** 比如不要（不希望出现的方向，例如模仿某种风格 / 过于工具化 / 像电商）、生成内容应该围绕（用户输入 / 场景 / 数据） 展开；

**最后再补充设计风格：** 比如整体氛围是（温暖 / 轻松 / 冷静 / 游戏感），并避免（过强科技感 / 信息过多 / 模板化布局）。


**💡 和AI沟通的本质，其实和工程师、设计师是一样的：把你的想法、判断和取舍讲清楚。所以一个有效的方式，是把prompt当成需求文档来写。**

**03**


**不同类型的产品**

**如何修改Prompt**


在实际使用中，不同类型的项目，修改prompt的重点其实也不一样。你可以根据产品的目标，去调整描述的方向。


**效率型产品（比如工具、SaaS、管理类）**

重点放在信息清晰和操作路径上。强调减少步骤、突出关键操作、让用户快速完成任务，比如优化信息优先级、简化流程、强化CTA的明确性。

**内容浏览型产品（比如资讯、社区、展示类）**

重点是阅读节奏和信息层级。可以调整内容排列顺序，让用户有自然的浏览路径，而不是大量信息的堆叠。

**表达型产品（比如角色创作、故事、个人内容）**

重点在语气和情绪。让界面更像在讲述一个故事，而不是在解释一个功能。视觉上也可以强调氛围，而不是结构本身。

**游戏或体验型产品**

重点在参与感和反馈。可以强调分步骤引导、即时反馈、以及"过程感"。

**交易或平台型产品**

重点在信任和信息组织。需要明确区分核心信息和辅助信息，让用户快速判断，同时避免过度复杂的结构。

**05**


**AI工具到底有多大影响？**


最后想说，工具确实越来越好用。但不管是Google Stitch也好，其他AI工具也好，本质上都只是把想法做出来------什么该做、什么不该做、哪里不对、为什么不对...这些事情还是要自己想清楚。**而这些一直都是比使用工具本身难上好几倍的事情。**

过去，Figma之类的软件，是整个流程的起点。我们之前也说，《[未来5年，设计师可能不用再学软件](https://mp.weixin.qq.com/s?__biz=MjM5MjIzMjU3Ng==&mid=2650200141&idx=1&sn=5f699845e1b1e1937712fd7e4b429053&scene=21#wechat_redirect)》，现在看起来还是保守了。现在，设计不再是从空白页面开始，大家都将在AI生成的完整页面上进行修改。


[![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FsJfckLux8WV4sYwpLUBTx8B4UZ3Do07l5ia8Ktt6JkJIFicmiajLTYIUanXSzlct6kiaG2UD4XRlCEBELM61aW9l8yF67efIe4SFp7iaFjD0B0icQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D23)](https://mp.weixin.qq.com/s?__biz=MjM5MjIzMjU3Ng==&mid=2650201924&idx=1&sn=0ee33d706929729b5c547623688b6ba0&scene=21#wechat_redirect)

GPT-Image-2生成的作品集排版

看起来是效率的提升，实际上是设计逻辑的改变。当UI/UX的基础工作减少了90%，我们从设计者转换为了决策者。**说得再通俗点，现在设计师一上来，就干了过去资深设计师的活儿。**

AI把设计水平的下限提高了，如果今天还把大量时间放在把界面画出来，那确实很容易被AI取代。但如果你能把AI当作工具，**去更快地生成版本、验证想法、调整逻辑，那么设计的上限反而会被拉高。**

**![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FsJfckLux8WUFVBbeicCnx79SzkicvTgHkgicbGibvNWTseeMC0vOj1gppLvqdgVqMR7KF82xCqibwxnrrB1mpnehqYlYMkLmPwYdI7SbwwJvKtuE%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26tp%3Dwebp%26wxfrom%3D5%26wx_lazy%3D1%23imgIndex%3D31)**
