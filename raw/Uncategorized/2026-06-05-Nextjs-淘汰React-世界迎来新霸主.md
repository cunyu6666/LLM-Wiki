一时间，整个 React 社区都在讨论同一个问题：
> Next.js 的时代，要结束了吗？

## TanStack Start 正在快速崛起

很多开发者对 **TanStack** 的印象还停留在 React Query。

但实际上，今天的 **TanStack** 已经发展成了一套完整生态：

*
  TanStack Query
*
  TanStack Router
*
  TanStack Form
*
  TanStack Table
*
  TanStack DB
*
  TanStack Start
*
  TanStack AI

其中最值得关注的，无疑是 TanStack Start。

简单来说，它是 **TanStack** 推出的全栈 React 框架!

支持 SSR、SSG、CSR、Streaming、Server Functions 和 Type Safe Routing，底层构建在 Vite + TanStack Router 之上，而不是 **Next.js** 那套体系。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2F7YalpHdlCYicI6W6OSKWwRibKLJiaKxdnz4bibpF85MSicxKkwBvjKKcSQHtTKf6JxYrhRS2pwIic4a6bq6ew7NMoI94RqfWgVHmCmIHbxbHtJ5h8%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D2)

很多开发者第一次体验 **TanStack Start** 后都会有一种共同感受：
> React 全栈框架终于被重新设计了一遍。

没有大量隐藏的运行机制，没有越来越复杂的框架魔法，所有行为都尽可能显式。

这种设计理念不仅让开发者更容易理解项目结构，也让 **AI** 更容易生成和维护代码。

## AI 公司为什么开始集体选择 TanStack？

过去几年，**React** 世界几乎只有一个标准答案：

    React = Next.js

但 **AI** 时代到来之后，框架的评价标准开始发生变化。

以前大家关注的是社区规模、招聘市场和生态成熟度；现在越来越多团队开始关注：

*
  AI 是否容易理解项目结构
*
  AI 是否容易生成代码
*
  AI 是否容易维护大型代码库

而这恰恰是 **TanStack** 最擅长的领域。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2F7YalpHdlCYictCqgBQXIs84nUWiawkbpEZeeLsQzX43rAgDx1QHZJxMUvlCJRQhew64baic0xIAvREDuA5lqtokEibibK7RpxTvcFeDMjicIvjwgU%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D3)

TanStack 一直坚持一个非常鲜明的理念：

    Explicit > Magic

显式优于隐式，类型安全优于运行时猜测。

对于人类开发者友好，对于 **AI Agent** 同样友好。

这也是为什么越来越多 AI 创业公司开始把 TanStack 当成新的默认选项。

## TanStack AI 才是真正的大招

如果说 **Lovable** 迁移只是一个信号，那么最近发布的 TanStack AI 才是真正值得关注的事情。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_gif%2F7YalpHdlCY9JDLmiaGWuUL2JOofY2bKZmvInmJLNBibeQ1e8iatY847coLdyEunrciagu5Tzyk0D1gnPqxwGfnaFbofzGnAq3KX2cUsKyc7DoOg%2F640%3Fwx_fmt%3Dgif%26from%3Dappmsg%23imgIndex%3D4)

很多人以为它只是另一个 **AI SDK** ，但实际上它更像是一套 **AI** 应用基础设施。

目前已经支持：

*
  Agent
*
  Tool Calling
*
  Middleware
*
  Workflow
*
  Orchestrator
*
  Streaming
*
  React Hooks
*
  React Native

尤其是官方重点推出的 **Workflow** 与 **Orchestrator** 。

**Workflow** 本质上是固定流程：

    A → B → C → D

而 **Orchestrator** 则允许 **Agent** 根据当前状态动态决定下一步执行什么：

    A
    ↓
    AI 决策
    ↓
    B 或 C 或 D

这也是未来 **Agent** 系统最核心的能力之一。

## TanStack 的野心已经变了

如果把最近半年的动作全部串起来看，你会发现 **TanStack** 已经不再满足于做几个优秀的 **React** 工具库。

它正在构建一套完整的 **AI** 原生开发生态：

    TanStack Query
           ↓
    TanStack Router
           ↓
    TanStack Start
           ↓
    TanStack DB
           ↓
    TanStack AI

从前端状态管理，到路由、全栈框架、数据库，再到 Agent Runtime，几乎覆盖了 **AI** 应用开发的整个链路。

这也是为什么越来越多人开始把 **TanStack** 称为：**AI 时代的 React 基础设施。**

## 写在最后

**Next.js** 会被淘汰吗？

我的答案是：**不会。**

至少未来几年，**Next.js** 依然拥有最大的 **React** 社区、最成熟的企业生态，以及 **Vercel** 强大的平台支持。

但另一件事同样正在发生：

越来越多 **AI** 公司和创业团队，已经开始把 **TanStack** 当作新的技术栈选择。

过去：

    React = Next.js

未来或许会变成：

    React =
    TanStack Query
    +
    TanStack Router
    +
    TanStack Start
    +
    TanStack AI

**React** 世界的下一场大战，可能才刚刚开始。
