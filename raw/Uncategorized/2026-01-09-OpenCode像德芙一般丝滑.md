## 安装一行命令

    npm i -g opencode-ai

进入项目目录，输入 opencode 就能用。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FKN0Z2XZibPdYYbY8JibFw6kfz2CAmKYaibI1ewzE77dq0FJYgzzpklyniaWuNdfMQGnvQCv5IoLwSJCxWxMiauA0COQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D3)

## 插件 oh-my-opencode

单独用 opencode 已经不错了。但配上社区开发的 oh-my-opencode 插件，体验直接起飞。

这个插件的核心是**多 Agent 协作** ：

*
  • **Oracle** ：架构审查，帮你把关代码设计
*
  • **Librarian** ：文档检索，快速找需要的信息
*
  • **Frontend Engineer** ：前端专项代理

这些后台代理并行工作，主 Agent 只接收结果。**上下文保持精简，效率拉满** 。

还有几个实用功能：

*
  • **AST 感知的代码操作** ：基于语法树重构，不会误伤字符串里的同名文本
*
  • **Comment Checker** ：自动防止 AI 过度注释（这个救命）
*
  • **ultrawork 模式** ：输入关键词激活最大性能，类似 Claude Code 的 Ultrathink
*
  • **自动会话恢复** ：断网重连能接着之前的上下文

安装：

    bunx oh-my-opencode install

## 和 Claude Code 怎么选

说实话，底层模型能力没有本质差距。差异在于**上下文工程** （Context Engineering）。

opencode + oh-my-opencode 的优势：

1.
   1. **灵活性** ：可以随时切换模型，日常用便宜的，复杂任务再切 Claude
2.
   2. **稳定性** ：配合国产模型，国内网络基本没断连问题
3.
   3. **社区活力** ：更新频繁，几乎每天都有新 commit

如果你追求开箱即用、不想折腾，Claude Code 仍然是最省心的选择。

但如果你想**控制成本** ，或者**网络环境受限** ，这套开源组合值得一试。

opencode 是开源的，对想学 Agent 开发的人来说，直接看源码就是最好的教程，而且完美兼容 Claude Code，它兼容 Claude Code 的命令、代理、技能、MCP、钩子（PreToolUse、PostToolUse、UserPromptSubmit、Stop）机制，迁移成本几乎为零。

你甚至可以让 opencode 结合自身给你制定学习计划------它会告诉你该读哪些文件、核心架构是什么、各模块怎么设计。

废话少说，赶紧去试试吧，免费，还要啥自行车，趁机体验体验CLI快乐编程。
