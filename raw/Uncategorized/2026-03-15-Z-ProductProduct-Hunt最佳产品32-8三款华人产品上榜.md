**TOP1：Anything API**

**一句话描述：** Notte 的 Anything API 是一个"把任何网页操作压扁成一个可调用 API 端点"的服务，专门为没有公开 API 的网站补上接口。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FxJ0xcBAKz2DKDlh7Hp9oxKcajiadCLjiaqIibfNiahUCGTsTHP6lJOcvibyeKN96Vq1RWhpQZyYB74gxTQtlszPzxaibEiboftYL5JQoh11PmUbeEU%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D3)

*图片来源：Product Hunt*

**简介：Anything API 定位为"构建在 Notte 浏览器代理平台之上的网页→API 工厂"，你只用自然语言描述要在某站上完成的任务，它就帮你生成一段可部署的 Notte Function。**

目标用户 \& 痛点：主要面向需要从各网站抓数、自动化操作、或集成内部系统的开发者和增长 / 运营团队，这些人长期被"官网没 API 或 API 太贵 / 太有限"卡住，只能自己敲爬虫或半手工点网页。Anything API 解决的是"业务需要稳定、可复用 API，但网站只有前端"的落地难题。

**核心功能与差异化优势：**

**（1）从任务描述到浏览器工作流：你用自然语言描述浏览器要完成的操作（如"给任意商品 URL 返回价格和前 10 条评论"），Notte 的代理会在真实浏览器里跑一遍，自动摸清流程。**

**（2）自动产出可复用函数 / 端点：首轮浏览器会话结束后，Anything API 把流程固化成一个 Notte Function，今后每次调用都直接走 HTTP 请求而不是再开浏览器，认证、Headers、请求格式都封装好，返回结构化 JSON。**

**（3）一键部署、Cron、API 调用：生成的 Notte Function 可以在 Notte 基础设施上无服务器部署、挂 Cron 定时跑，或作为普通 HTTP API 被你自己的服务调用，完全复用 Notte 现有的会话、代理和 Vault 安全体系。**

用户体验类比：体验像"给没有 API 的网站请了一个工程团队，帮你把那套浏览器点点点封装成一个正规接口"，之后你只需像调用其他后端服务一样打一行 HTTP 请求。

**数据表现：** Anything API获得了 584 个 Upvote，116 条 comment。

**网站链接：** https://www.notte.cc/?ref=producthunt

**TOP2：Claude Marketplace**

**一句话描述：** Claude Marketplace 是一个"给企业买 Claude 生态应用的应用商店"，可以直接用你已有的 Anthropic 预付额度来付钱。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FxJ0xcBAKz2Bw7mxeWGyRYFFePEr11KWKoseMgF8v4d8Ue8nw1J7riaU4xPHIRicOop8gUq0MA6UkQU09DDHpw8wtVqAGJqgYkpib3aC5DgZh3c%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D4)

*图片来源：Product Hunt*

**简介：Claude Marketplace 定位为"面向企业采购的 Claude 应用集市"，聚合了基于 Claude 模型构建的第三方工具（如 GitLab、Harvey、Replit、Snowflake 等），由 Anthropic 统一结算和管理访问。**

目标用户 \& 痛点：主要面向已经和 Anthropic 签了年度承诺消费的中大企业，这些公司一方面想用更多垂直 AI 工具（编码、法务、数据分析等），另一方面又被漫长的多家供应商采购流程拖慢。Claude Marketplace 要解决的，是"同一笔 Claude 预算怎么顺手买周边应用、且只要一份合同一张账单"的问题。

**核心功能与差异化优势：**

**（1）用既有 Claude 承诺额度买应用：企业可以把原本承诺给 Anthropic 的部分预算，挪一部分用于购买 Marketplace 里的第三方 Claude 应用，相当于给"超买的额度"多一个出口。**

**（2）一份合同、一张发票：由 Anthropic 代为管理合作伙伴的计费和开票，企业不需要再和每家工具商单独谈价、签约、走法务审查，极大缩短采购周期。**

**（3）Claude 生态专属渠道：只有基于 Claude 打造的应用才能进 Marketplace，这既给工具开发者一个触达企业客户的新渠道，也反过来激励他们优先深度集成 Claude。**

用户体验类比：体验像"企业版的 Claude App Store + 统一结算中心"，采购团队只和 Anthropic 打交道，就能把多个 Claude 驱动的专业工具一起买进来，用的都是同一桶 Claude 预算。

**数据表现：** Claude Marketplace获得了 483 个 Upvote，12 条 comment。

**网站链接：** https://claude.com/platform/marketplace?ref=producthunt

**TOP3：TestSprite 2.1（华人Builder）**

**一句话描述：** TestSprite 是"给 AI 原生团队用的自动化测试代理层"，专门负责把 AI 写的代码测干净。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FxJ0xcBAKz2DqCxVsGbicjsJoa6ZVoaziczQyAsf5q2OTmPNHwHfBGp93pr9IXL7n2HEOCTMjPOvh8o2ogvv35zaJXa1NOhf5MCbjI4EXOXgZc%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D5)

**简介：TestSprite 定位为"嵌进 IDE 的 agentic 测试工具"，通过 MCP 直接连到 Cursor、Windsurf、Claude Code 等，把从测试计划到测试代码生成、执行、诊断全部自动化。**

目标用户 \& 痛点：主要面向已经大量用 AI 写代码、但不想测试质量失控的研发团队，他们常常有"代理写得飞快、人修 bug 修到吐"的痛点。TestSprite 想解决的是"AI 写得快，但测试和质量跟不上"的缺失环节。

**核心功能与差异化优势：**

**（1）零提示生成完整测试套件：接入 IDE 后，基于代码库和需求文档自动规划、生成并运行 UI、API 和端到端测试，不需要你一条条写用例或手动敲 Playwright / Selenium。**

**（2）2.1 的更快引擎 + 可视化编辑：新版测试引擎提速约 4--5 倍，几分钟跑完整套测试，还提供可视化编辑器，你可以点任意步骤看当时页面快照，直接改定位符、输入、断言，后续步骤会在保留自定义的前提下自动重生。**

**（3）深度 GitHub 集成与"自动挡"质量闸：每一个 PR 都会自动在预览环境（Vercel、Netlify、Render、Fly.io 等）上跑全套测试，结果回写到 PR，并可开启"失败即阻止合并"，把测试变成默认开启的质量闸门。**

用户体验类比：体验像给你的 AI coding agent 配了一个"全自动 QA 室友"，它负责理解需求、写和跑测试、标出问题并阻止有坑的 PR 被合并，让你可以放心让 AI 多写代码。

**团队信息：** 据领英公开信息，Yunhao Jiao 现任 TestSprite 联合创始人兼 CEO 此前，他在亚马逊任职近五年，先后担任 AWS 与 Amazon Business 的软件工程师与 SDE II。教育方面，Yunhao 拥有耶鲁大学计算机科学硕士学位，本科毕业于浙江大学计算机科学专业。

**TOP4：Codex Security**

**一句话描述：** Codex Security是一款"用 AI 当应用安全工程师"的安全代理，专门帮你在代码库里找洞、验证、再给出修补建议。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FxJ0xcBAKz2Dxo77tkPF9UKQicXBIa9fL1AT7caSQQsRgl80gwo6LTLgibF6m2OqJuTOLlFzexxGrhy70X8Scb7Cx087SXXOnHo0CLh6R5PXOI%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D7)

*图片来源：Product Hunt*

**简介：Codex Security 定位为"面向工程和安全团队的应用安全代理"，直接连到你的 Git 仓库，基于项目级威胁模型和代码上下文，持续分析、校验并协助修复漏洞。**

目标用户 \& 痛点：主要面向被"AI 提速写代码，但安全审查跟不上"困扰的开发和 AppSec 团队，他们要么被传统 SAST 工具的海量误报淹没，要么人为 Code Review 顶不住越来越多的改动。Codex Security 想解决的是"噪音高、真漏洞漏掉、人手不够"的安全瓶颈。

**核心功能与差异化优势：**

**（1）仓库级威胁建模：不是按文件扫 pattern，而是先吃下整个代码库，自动构建可编辑的威胁模型，了解系统边界、信任域、数据流和暴露面，再在此基础上找真正有利用价值的漏洞。**

**（2）自动验证，显著降噪：对疑似漏洞在沙箱或项目专用运行环境中"压力测试"，尝试实际利用并生成 POC，只有确认可利用才报出来，从私测数据看，告警噪音能降 80%+，误报减半以上。**

**（3）上下文感知补丁建议：结合系统架构和周边代码风格自动生成修复补丁，以 PR 或建议形式提交，由工程师审核后合入，既减少重复体力活，又保持人对最终改动的掌控。**

用户体验类比：体验像给团队多招了一位"永远在仓库巡逻、会自己验证和起草补丁的资深安全工程师"，你只需要审一审它挑出来的高置信漏洞和修复建议，就能在不拖慢发版的前提下提高安全基线。
