# 1.1万 Star，又一款笔记工具火出圈了！

我最近在整理自己的笔记系统，Obsidian 用了三年，插件装了七十多个，配置文件快赶上一个小项目了。每次换台电脑，光是把那套环境弄回来就要大半天。后来看见有群友推荐，试了试开源的 Tolaria，这玩意儿是新出的，用过之后感觉相当哇塞，赶紧推荐给你们。

## 简介

Tolaria 的定位是一个跨平台（macOS、Windows、Linux）的桌面应用，用来管理以 Markdown 为核心的知识库。听起来就和 Obsidian 差不多，但不完全是。它的每一处设计决定，出发点和Obsidian都不太一样。

Tolaria 很受追捧，发布后短短几周就冲到了 10,000 Star 以上，截止到现在，已经超过1.1万 Star 了。

## zero lines of code

开源者 Luca 在 GitHub 页面里直接写了一句话：**整个代码库超过 10 万行，而他自己一行都没有亲手写。全部是用 Claude Code 搞的。**

评论有人说这不过是花哨提法，实际改一个非常具体的 bug 的时候还是得自己上。我的看法有点不一样，这不是「开发者消失了」的证据，而是一种非常扎实的开发证明：**人跨得过去，工具才能跨得过去。**

Luca 拿这个工具去管理自己的超 10,000+ 条笔记。他不是先有软件就来写了，而是先将自己的工作流程运转了多年，摸了一套真实有效的知识管理方式，再把它用代码实现出来。这个项目的特别之处是：**它是从真实使用场景里长出来的，而不是为了做一个工具而做工具。**

## 先进的设计原则

**Files-first** ：你的每一条笔记就是磁盘上的一个 .md 文件。Tolaria 永远不拥有你的数据，它只是一个读写工具。就算哪天停止更新了，或者你决定换别的软件，文件和文件里的内容一分不少地都还在那里。

**Git-first** ：每个 vault（仓库）都是一个 git repository。这意味着你天然就有完整的修改历史、可以用任意 git remote 同步，当 AI 工具编辑过你笔记之后，你可以直接看到 diff、随时回滚。这点对我来说相当关键：我现在让 Claude Code 拿我的笔记做不少事，没有 git 的话根本不敢放手让它动文件。

**Offline-first** ：没有账号，没有订阅，没有云依赖。vault 完全离线也能正常工作。

**AI-first but not AI-only** ：Tolaria 内置支持 Claude Code、Codex CLI、Gemini CLI 的接入配置，同时也有一个 MCP server，让任何支持 MCP 协议的 AI 客户端直接操作你的知识库。但它不强制你一定要用 AI------它就是一个普通的 Markdown 知识库管理工具，过没有 AI 的日子也照用不误。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FRropZVGGmsBic572sXdHBGTHDjuZaQ3XHQTA0JsF4rNZJ8ttxDN8fqiamEfOBeul0Fwr8A87CfwkBDzGt1V6X8sLMuiczic3QDar3GJh9sTMdNA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D1)

**Keyboard-first** ：Command Palette（Cmd+K）是所有操作的入口。上手一天之后你基本可以不摸鼠标了。

## AI 集成

Tolaria 内置了一个 MCP server，就在 mcp-server/ 目录里。它提供了一套 tool：search_notes、read_note、create_note、edit_note_frontmatter、append_to_note 等等。Claude Code 或者 Cursor 加上这些 tool，就可以在你的知识库里索引、读取、创建、修改笔记。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FRropZVGGmsCofQvQe6fGc37bAg2ruLOOicwXM9Pr8fIic4KaZIFW5QqzLpjWY7oZq82HIYy3B149khFk2oJGlSVkEKwl1HqfrsyjujSayOTRo%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%26watermark%3D1%23imgIndex%3D2)

使用方式不复杂。在 Tolaria 的 Status Bar 或者 Command Palette 里运行一下 MCP 注册，它会自动把自己写入 ~/.claude.json、~/.cursor/mcp.json、~/.config/mcp/mcp.json 等配置文件里。不需要手动拉配置，在 GUI 里点几下就搞定。

除了 MCP，应用内部还直接集成了 AI 面板（AiPanel）。你可以在右侧栏切换到 AI 面板，直接对当前打开的笔记进行对话。支持的 Agent 有 Claude Code、Codex、OpenCode、Pi、Gemini、Kiro，这个列表跟目前主流的 CLI Agent 基本平齐了。

另外还有一点设计我连续看了好几遍才理清楚：当 Claude Code 通过 MCP 对你的笔记做了修改，它会自动检测文件操作，一旦有 file modified 事件就会触发 vault reload。你在 UI 里就能看到哪些笔记被改动了，不需要手动刷新。

## 技术栈撸一层

Tolaria 是用 Tauri v2 + React 19 + TypeScript 写的，后端是 Rust。这个组合是目前做跨平台桌面应用的主流方案之一，比 Electron 轻很多。

编辑器用的是 BlockNote，支持块式富文本编辑和原始 Markdown 模式（CodeMirror 6）。图表用 Mermaid，白板用 tldraw。都是当前工具链里比较靠谱的开源选择。

搜索目前是基于关键词的 walkdir 文件扫描，没有语义搜索和向量索引。这对大部分人足够用，但如果你在意 10,000 条笔记里语义相近的内容搜索，现在这块确实要靠上面提到的那些 AI 工具来补。

vault 的缓存策略设计得比较精细------基于 git HEAD 进行增量更新，不是每次全量重扫。缓存文件放在 ~/.laputa/cache/，跟 vault 目录分开，没有乱七八糟的临时文件。

## 和 Obsidian 做个对比

这俩没有好坏之分，只看合不合适自己吧。

Obsidian 最大的优势是东西少------它的插件生态是能打造出一套完全自定义的工具链的。但这也是它的问题：上手门槛高，你得自己搞明白哪些插件值得装、怎么配，为了第一次用就能顺手往往要花好几个下午。

Tolaria 是反面思路：它是 opinionated 的。vault 就是一个 git repo，笔记类型用 YAML frontmatter 里的 type: 字段定义，左侧导航栏按类型和集合分组显示。开箱就用，不需要配。

对于喜欢自己一个一个搬砖头搬起来的人，Obsidian 更适合。对于想要一个「拿出来就能用」的、AI 友好、带 git 历史的笔记环境，Tolaria 要简单很多。

## 怎么安装

macOS 用 Homebrew 最简单：

    brew install --cask tolaria

Windows 和 Linux 可以直接到 releases 页面 下载安装包。Linux 的话需要安装一些系统依赖（WebKit2GTK 4.1 + GTK 3），Debian / Ubuntu 上：

    sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget file \
      libxdo-dev libssl-dev libayatana-appindicator3-dev librsvg2-dev \
      libsoup-3.0-dev patchelf

第一次开启时会提示你选择 vault 或者克隆官方的 getting started vault，这个入门仓库本身就是一个配好了示例类型和工作流程的模板，跑一遍大概就明白它在做什么。

## 写在最后

Tolaria 用下来也有一些问题，简单提一下，搜索目前是纯关键词，移动端不支持，如果你笔记量大、组织方式跟它预设的类型不大对得上，就有点难受。

但这个项目的出发点就是拿来解决一个具体的痛点：**如何让一个自己用、AI 用得顺畅的 Markdown 知识库变得更容易继续存在和扩大。** 它把 Files、Git、AI Agents 这三个东西串起来的方式，我目前还没看到有别的工具做得这么自然。

开源这个决策本身也很关键。AGPL-3.0 许可证，代码全开放，有问题可以直接看源码、开 issue、提 PR。发布后几周社区里已经有小几百个 PR 在测了，活跃度还不错。
