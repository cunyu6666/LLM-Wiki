# 2026年，写给所有人的Vibe Coding工具路径图

昨天，我们号达成了一个小成就，创号四个月以来，图文原创文章达到了100篇。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FMHVsz9lwuhVutyG7Tt90RrDlX9wWLyesaGRjibEGENyn9RB9Pueu2KVciaf1Qz48tkIvC4TWCgsiaWXszA795IZdNkaknvYicP1eslHpQu0jQTM%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D0)

在这个快节奏的AI时代，我甚至感觉到，似乎我们号日更原创文章，都难以追上这快速发展的脚步。

每到深夜，我一边看着Claude Code在终端疯狂吐token，一边刷着X上的最新的AI动态，FOMO和焦虑情绪也偶尔敲打着疲惫的心态。

从去年2月份Andrej Karpathy提出Vibe Coding的概念以来，到现在才1年时间，整个AI编程赛道已经出现了天翻地覆的变化。

过去一年，我几乎尝试过市面上所有的AI编程工具，有的每天深度使用，有的只是下载下来浅尝辄止。但作为一个研发出身的技术博主，我觉得我应该有点资格来谈一谈AI编程赛道整体的工具竞争格局。

按照我的理解（不保证绝对合理），目前Vibe Coding的工具基本分成四大赛道：AI编程CLI、AI编程IDE、AI编程App、AI编程SaaS。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FMHVsz9lwuhWs3Wte00Z52zGX8sARZQrAS2ualjXQoqicMUA6pz1bxIVUhr8ibDsGLQcwvSEUdMV50OTtBk3OfeAASriaKaNX2Cdhm9xgvic5Olk%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D1)

下面一个一个聊。

### AI编程CLI：今后的绝对主流

CLI Coding Agent是我个人花时间研究最多的工具类型，也是我日常的生产力主力。

而Claude Code就是其中的最大代表。从前段时间默认模型升级到Opus 4.6之后，3月份以来的更是重磅更新频出：push-to-talk语音模式（按住空格说话就行）、/loop循环任务、100万token上下文窗口、Code Review、新增类似OpenClaw的Channels功能等。

从去年8月份以来，我开始逐步将Claude Code融入到日常的各种工作中，截止到目前，我日常的80%左右的工作几乎都是由Claude Code独立或者协助完成。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FMHVsz9lwuhUUwULbH5mly4ibLEpnT28OYwmP2WQ11RicxfSibOC1jeXcLRa2vvtLqOo7ibCOhQDLarAz9FnfS4e0SAFOvlEOAVfzibf90v4NialAA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D2)

OpenAI的Codex CLI近期进展也很猛，Rust写的，轻量快速，并且越来越好用，整体性价比可能已经超过了Claude Code。

Google的Gemini CLI则是免费开源，个人Google账号可以免费用Gemini 3，同样100万token上下文，白嫖党首选，但缺点是想要作为日常生产力，不如Claude Code和Codex好用。GitHub Copilot CLI今年2月正式GA，会自动把任务委派给专门的Agent（比如代码审查、实现规划），主持国外御三家的大部分主力模型。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FMHVsz9lwuhVUo9kNcLF82g608tnGBISNuQSqoxicIof8sgpKMd0bib944S9c3yyZAZDld4eheQOt3ESib54Qctrgr5Z499dDo66bu631mjajGU%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D3)

国内CLI也在快速跟进。光阿里一家就推出了三款不同的CLI，包括Qwen Code、iFlow和Qoder CLI。月之暗面的Kimi Code现在搭配Kimi 2.5模型，也有很多人在用。腾讯的CodeBuddy也出了CLI版本。

CLI工具怎么选，其实主要看你用什么模型生态。偏好Anthropic选Claude Code，偏好OpenAI选Codex CLI，想白嫖选Gemini CLI。国内用户如果网络受限，Kimi Code和Qwen Code都是可用的选项。

需要提醒大家的时，今年作为AI Agent元年，所有的软件未来都会CLI化，CLI这种看起来专属于开发者的工作入口，在接下来会成为每个人拥抱未来世界的主入口。所以，不管怎么样，选择一款适合你的AI CLI Coding工具，也是大势所趋。

### AI编程IDE：大部分人的主战场

如果说CLI是未来主流，那么IDE绝对是当下主流，这是长期积累绑定的结果，短时间内不会很快改变。

Cursor还是这个赛道的标杆。今年3月刚发布2.6版本，最大的变化是Cloud Agents，Agent可以跑在云端，从浏览器甚至手机上就能启动任务。另一个值得说的是MCP Apps，把Figma、Amplitude这些工具的交互界面直接嵌进了聊天窗口里，对于习惯使用IDE的用户而言，体验会更加丝滑。

但我个人而言，已经有好几个月没打开个Cursor了（我是CLI坚定派）。

Google去年11月份推出了Antigravity，虽然主打免费，Agent功能也非常强大，但跑起来时常中断，很大程度上影响了其易用性。以至于现在都拿反代工具来使用Antigravity的模型，建议谷歌加快将Gemini CLI和Antigravity改造成收费产品。

国内这边，AI IDE也是各大厂必争之地。字节的Trae，个人版免费，SOLO模式让AI从写代码到跑测试全流程自动化，月活已经过了百万。阿里的Qoder，主打上下文工程，一次能检索10万个代码文件，Quest Mode支持从自然语言直接走到代码交付，价格也便宜。腾讯的CodeBuddy IDE版本，据官方数据内部90%工程师在用，编码时间平均减少40%。

还有Windsurf，去年底被Devin母公司Cognition以约2.5亿美元收购，今年2月在LogRocket的AI开发工具排名中超过了Cursor排到第一。它的Cascade系统会自动迭代代码直到真正能跑，对新手相当友好。

老牌IDE，VSCode自然也是很多人的选项，VS上个月做了大更新，结合Copilot新增Multi-Agent驱动和任务指派，用户再配合各类Coding Agent插件（Claude Code for VSCode、Codex），易用性上也非常能打。

简单来说，有编程基础的话，Cursor和VSCode二选一基本够用。想省钱或者偏好国内生态，Trae和Qoder都值得试。

### AI编程App和SaaS：零基础的入口

这两个赛道是真正面向所有人的。

App这边，OpenAI的Codex App今年3月刚上了Windows，可以同时跑多个Agent并行处理任务，GPT-5.4模型加持。它的Automations功能能自动处理issue分类和CI/CD这些日常杂活，适合已经有项目在跑的团队。Claude Desktop里面的Code模块也能做编程辅助，和Claude Code共享底层能力。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FMHVsz9lwuhVlmNk2iaAXMsW25TeIwtUQZgfMIia4465PP7F2k2ibByLibMXPGp6EQJpkjKEuhG2PlLV1zMka494xibodic0MI4nlibvzAaOXbjDaIk%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D4)

SaaS赛道门槛最低。Google AI Studio的Build模式完全免费，自然语言描述需求就能生成React应用，适合快速验证想法。Trickle是专门做Web项目的Vibe Coding的SaaS平台，内置图片视频生成和数据库。Devin云端可以并行运行多个AI工程师，根据官方数据效率比1.0提升了83%。Lovable通用是一款Agent驱动的全栈Web应用开发平台，支持可视化编辑、和一键部署等功能。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FMHVsz9lwuhVM5dRqUVJlwqvknjpuDbJG1c9RSXvsWOHmNthbv0PySlhVx7V718MS9luLuULvBI2UyS5AGrkS7LgTOAIm76Z0GS0ZDJossRc%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg%23imgIndex%3D5)

如果你完全不会写代码，从Google AI Studio Build开始是成本最低的选择，免费且不需要配环境。

### 工具路径综合建议

聊了这么多，按不同背景给个综合路径建议。

有经验的程序员，CLI加IDE组合效率最高。我自己的搭配是Claude Code做主力CLI，VSCode做复杂项目的可视化编辑和代码Review，基本覆盖了日常所有场景。

初级开发者或者刚转行的，先从一个IDE入手就够了。Cursor或VSCode都行，国内用户可以先从Trae或者Qoder开始，免费而且中文支持好。

完全没有编程基础的，又不想折腾各种环境配置，以及只是想做个网页啥的，SaaS赛道是你的入口。Google AI Studio Build零成本起步，等对代码有了基本感觉，再升级到IDE也不迟。

我做医疗AI算法这么多年，写Python还行，前后端方面差远了，相关基础也是从前两年才开始学习补齐的。靠这些工具折腾了一年多，从零做了几个全栈项目，有些还真的跑在了生产环境上。上手门槛比想象中低，你花点时间肯定也没问题。
