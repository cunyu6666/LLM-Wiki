# Cursor 新版本提前预览， UI 大变更

> 来源: [mp.weixin.qq.com](https://mp.weixin.qq.com/s?__biz=MzIyMzU5Mzg2NQ==&mid=2247484824&idx=1) · gchr 读书不多


提前拿到了 0.46 版本，我这里只是简单体验了不到一天。更多变动可能没注意到，或者和最终的版本还有差别，难免有遗漏和错误的地方。

## 右侧 Chat 面板大变动

Cursor 即将发布的大版本更新是 0.46，这次提前拿到了新版本，也简单体验了一下，简单记录下新版本都有什么变化。

最大的变化就是 UI 的改版, 右侧的 AI Pane 变为一个统一的 Chat 面板。不再有 Composer 和 Chat 两个窗口。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FhGFJYrPZQhI3icR0khV1BK7IHU0a6ZP6wZd7ibc3Sib5jQE96GxjhuASBfG1GMRuLtnDTyAUgicicE4rohUkicYRlceQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)UI 变更

而在 Chat 面板中，输入框下面有一个新的选项下拉，可以选择的范围分别是: Agent, Ask, Edit。 可以通过快捷键 Cmd + . 来快速切换。

这里大概解释下这几种模式和之前版本的对应关系：

- Agent 对应之前的 Composer 下的 Agent 模式
- Edit 对应之前的 Composer 中的 Normal 模式
- Ask 对应之前的 Chat 模式

当选择不同的模式的时候，可选择的模型会自动高亮，比如当我选择 Agent 的时候，gpt-4o 和 claude-3.5-sonnet 就自动高亮了。 DeepSeek-R1 类似的模型就无法使用，如果你选择了类似的在 Agent 里面无法使用的模型会提示:


    Model deepseek-r1 doesn't have good agent support yet. Start a new thread using edit or ask mode for better results.

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FhGFJYrPZQhI3icR0khV1BK7IHU0a6ZP6wbibx5Z93iartZkaC1HbsibwEwDq3aAv9jiahf04m3icvRN9XHLOBiahEd89Q%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)Agent 模型提示

但是你还是可以继续使用，只是在 Agent 里面可能无法触发一些 Tools 的调用。

当我不是在yolo 模式下运行的时候，现在会有一个友好的选择，可以选择 yolo 模式执行 ，后续就可以自动执行命令。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FhGFJYrPZQhI3icR0khV1BK7IHU0a6ZP6wtFdQ3vsSv4WoguaW2eiaNIL78JdbibJcNKyXCaeEDowd2sJibq422gJWQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)yolo 提示

## Chat 里面的 @ 功能

@Files 和 @Folders 合并为 @Files & Folders 一个指令 增加 @Code 指令 之前介绍个 @/ 可以呼出文件筛选，但是这个版本这里失效了，不知是 feature 还是 bug。

输入 / 这里多了个 Disable Iterate on Lints / enable Iterate on Lints 选项，可以快速选择开或者关。这里的操作对应在设置 Features 里面 Iterate on Lints 选项。

在这里你可以 @Terminal，把终端中的上下文直接带入大模型中。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FhGFJYrPZQhI3icR0khV1BK7IHU0a6ZP6wPFYWibLfoeZyqEBZzSK16OcU1DtEMUwxWlcernpjoK8BibGAyBEFVePA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)  

## Cmd/Ctrl + K 快捷键

这里的弹出的输入框，可以输入 @ 来呼出 Code 和 Web 的指令。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FhGFJYrPZQhI3icR0khV1BK7IHU0a6ZP6wtuiaMVZoXPo21iaMKSlkab73f2KO6b5ibe2mYRST0HtrCYoMia8W5iaE5Dw%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)CleanShot 2025-02-22 at 02.24.43@2x.png

## 设置

明显的几个变化:

1.
   单独增加了 Rules 和 MCP 菜单，可见后面对这两部分的重要性吧，当然这两部分的可玩性也很高。 现在 MCP 也可以在 yolo 模式下执行了。不需要再手动点击确认，但是目前还是限制 25 次，和之前一样的限制。

2.
   在 Features 里面也去掉了一些设置，比如 Auto context 。

3.
   也增加了一些选项，比如 Iterate on Lints 选项。还有 Web Search Tool 选项（大意是 Agent 中自动识别需要调用网络搜索的能力）。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fsz_mmbiz_png%2FhGFJYrPZQhI3icR0khV1BK7IHU0a6ZP6wc0RaZV0VIJSWzg3icpTGHtIYbDJkRcF0ExeaQktUicU1DSKIEiaO2gCzQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)相关设置

## 总结

一个很大的好处是， 在一个 Chat 里面，我可以先选择 Ask模式 下的 DeepSeek-R1 模型来设计方案，然后再切换到Agent 中的 Claude 3.5 Sonnet 来写代码。

## 一些小问题

已经没了 Composer ，当时还有它的痕迹。设置的 Beta 菜单下面。按下 Shift+Cmd+A 搜索 Composer ，发现还是有很多Composer 的痕迹。
