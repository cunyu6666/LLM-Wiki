# 我的设计工作流程已经被 AI 所改变

> 来源: [mp.weixin.qq.com](https://mp.weixin.qq.com/s?__biz=MzIyMjU0NTQ3Mg==&mid=2247735787&idx=1) · 有言编辑部 三分设

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FemyVgtwLlkBcibe7eeaT6JxAvwKIaXTqjlK7wT2icFugzcwuSNoHmyxaYsqW8VKdDzGTK8kxP2IPsowpLibSXOnOQ%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)

AI 工具不仅加快了我的工作进程，还改变了我对传统设计流程本身的看法。如今，AI 工具不断推陈出新，然而，其中最大的转变在于我的思考方式、协作模式和解决问题的思路。接下来，我分享在过去 6 个月里我的工作流程发生的变化，以及我在这个过程中的体会。

...

1. 绘制问题空间

我从调研和收集信息入手，Perplexity 能够帮助我收集外部背景信息，比如：用户需求、社交媒体舆情以及竞品功能等。我借助高级推理模型来勾勒问题空间，探讨那些已知和未知的内容。在这个阶段，我会把找到的每一份文档都上传到与模型的聊天对话中，包括：谈话记录、用户调研结果、视频号和公众号上的文字记录，以及其他任何相关内容。

在梳理问题的过程中，我会在 FigJam 白板上整理所有的见解。这块白板就像是"无限画布"或者"第二大脑"，帮我构建对问题空间的认知模型，并与同事分享我的思路。更新起来也很快，其他人添加他们自己的见解也很容易。在白板上协作感觉没那么正式，更像是共同展开调查，而且有了模型辅助，绘制问题空间的地图似乎比以往任何时候都要更快。

...

2. 打造共享知识库

研究背景信息只有在易于获取的情况下才有用。一旦积累了足够多的信息，我就会使用 NotebookLM 将这些背景信息转移到一个共享且可搜索的工作空间中。任何人都可以添加研究结果、提出问题，或者查看已经探索过的内容。白板帮助我们把握全局，而工作空间则让我们在需要时能够整理信息并深入研究。

知识常常会在微信群或被遗忘的文档中流失。为团队打造一个共享知识库，意味着我们不太可能重复造轮子，而更有可能在彼此的工作和思路基础上继续推进。

...

3. 明确问题

一旦我弄清楚要解决的问题，就会借助 GPT 起草清晰的问题陈述，优化表述，并与团队一同审核定义。每个人都应当能用相同的方式描述问题。这些陈述是项目简报、启动会议以及后续讨论的良好基础。当大家对问题达成一致时，决策速度会更快，解决方案也会更有力。

...

4. 构思与快速草图绘制

在创意设计阶段，我会同时使用 Cursor 和 Figma。我越来越倾向于直接在 Cursor 中开展工作，但 Figma 的无限画布仍便于我进行复制、试验，并对尝试过的内容做可视化记录，这非常实用。

先从 Cursor 入手...

最近，我常常跳过 Figma，直接在 Cursor 中借助模型开展工作。我利用 AI 起草 UML 对象图、需求文档或其他资料，然后导入 Cursor，这样就能借助模型搭建初始概念的框架。初稿通常不怎么样，但它就像一套代码积木，我可以开始重新组合。

经过几轮尝试后，可能会出现一个有趣的想法。我可能会在 Cursor 中继续完善它；或者，如果在无限画布上思考更轻松的话，我会以此为起点，转到 Figma 中借助无限画布进行迭代。

先从 Figma 入手...

在 Figma 中：早期草图特意画得粗略且宽泛，以探索各种可能性。我发现允许自己先在画布上大胆构思一些夸张的高保真想法，这是一种激发创意灵感的有趣方式。生成一大堆想法，能很好地帮助我开始设想如何解决问题。如果产品已有现成界面，若它们是在 Figma 中构建的，我会重新创建或重新组合这些界面。

这个做法有两个目的：其一，我越早将解决方案可视化，就能越快朝着解决方案迈进；其二，随着项目推进，构建一套初始界面能进一步提升速度，因为它为我提供了更多可复用的组件和图层，便于在达成共识后绘制更多解决方案。

在 Cursor 中：早期我会设置一个代码仓库或分支。一旦在 Figma 中有了有趣的想法，我就切换到 Cursor 构建一个初始版本。这使我能够生成想法并创建代码模块，以便快速迭代。我模拟真实组件越多，并将它们连接到原型中，就能在版本控制中保留越多版本，从而使迭代速度越来越快。我越早发布真实界面，就能越早了解哪些方案可行。这样做的好处是，我的想法能迅速在代码中实现，更贴合设计系统的实际情况，也更接近终端用户的真实体验。

...

5. 原型制作、迭代与共享

原型制作是在 Figma 和 Cursor 之间循环往复的过程。我默认选择可点击原型，因为相较于静态图片，它总能引发更深入的思考。不过，如果静态图片能更好地阐述内容，我就会用 Figma。当交互、动效以及精确数据比较关键的时候，Cursor 更具优势。如果原型具有交互性，关于设计的讨论质量往往更高。

在 Cursor 中迭代时，了解目录设置、文件名，并让模型重构代码且保持文档更新，这些都是在 Cursor 中提高效率的技巧。若缺少这些，获得理想结果的耗时会大幅增加，还会让人受挫。能够说出特定设计系统的标记和组件名称也很有帮助。这属于上下文工程和提示设计的范畴，作为非技术人员，通过大量实践就能更擅长此道。

对于某些方面，比如：精确的视觉优化，在 Figma 中迭代还是会快一些，因为模型需要非常具体的上下文才能实现精确优化。我发现，有时把 Figma 中的绘图上传到 Cursor 的聊天窗口，就能得到我想要的结果。目前，模型对视觉示例的识别和反馈越来越好。

在共享方面，当有了不错的方案后，根据团队的同步（或异步）协作方式，我要么用腾讯会议或飞书进行通话，语音讨论设计；要么用 Screen Studio 快速录制内容，然后上传到飞书，利用其 AI 进行编辑，快速发布并分享。我争取每周至少向关键相关人员和同事开展线上视频会议，让团队了解项目进展。

...

总结思考

随着新模型和新工具不断涌现，我的设计工作流程习惯每天都在变化。当下对于设计领域而言，是一个前所未有的黄金时代，我们面前有众多新工具可供探索。就在这短短几个月，我的工作流程变得更快、协作性更强，也不再那么拘泥于形式。我现在花在揣测上的时间变少了，而更多地去测试真实的想法。AI 帮助我快速从调研过渡到制作原型，并且不会丢失上下文信息或工作势头。这些工具让我能同时进行代码编写与设计工作，这样随着问题及解决方案的演变，我也能迅速做出调整。

现在，我的创意想法与执行之间的差距比以往任何时候都要小，这意味着通过展示真实产品比语言描述的成本比以往更低。我相信，未来的设计流程应该是：更快推出更好的产品，减少走弯路的情况。
