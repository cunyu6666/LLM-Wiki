# 2025年，【MCP】会是智能交互与AI产品设计必知的技术知识点

> 来源: [mp.weixin.qq.com](https://mp.weixin.qq.com/s?__biz=MzkyNzIwNzczMw==&mid=2247752160&idx=1) · 智将菌 朝菌学园ZhaoDesign


![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_gif%2FET1CK99sg2CjHrg7hgkR9waq27sbfT6ibD9iaTUuDNMmSmYdOALCrKjI0uGVhaB6schUMzHMibk6FsslfM8OoTA1A%2F640%3Fwx_fmt%3Dgif%26wxfrom%3D5%26wx_lazy%3D1%26tp%3Dwebp)

AI技术在设计中的创新应用，是目前考研/保研的大热门话题，也是顶尖高校重点关注的前沿研究领域。

了解目前AI应用产品的发展趋势，就不得不提到前段时间火爆的【Manus】。它是由中国初创公司 Monica 开发的自主 AI 代理。Manus AI 于 2025 年 3 月 6 日推出，旨在通过**自动化复杂任务和增强决策过程来改变我们与技术的互动方式** 。  


![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_jpg%2FET1CK99sg2AokGhnMfou2zbqEH64UiaawKvcKux4kpRMiaWJ3kppM3F1mI8yAsqxcI9zBHyILTFLMstw9IErwn7A%2F640%3Fwx_fmt%3Dwebp%26from%3Dappmsg)

通俗来说，GPT、DeepSeek模型是AI的"大脑"，过去同学们尝试的AI设计概念，大多是"AI提供文字版的解决方案参考或建议指南"---比如"AI教育助手"、"AI情感陪伴"等等，AI的产出仅仅是一段文本，或者图片视频等较为单薄的内容。

而Manus AI则是给AI产品拓展了能交付完整项目产出的"手"，能根据人类的指令像人一样进行具体任务的操作行为，独立完成工作，产出一个"完整的项目"，或者提供多维度的服务，而不仅仅是"纸上谈兵"与"内容生产机器"。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FET1CK99sg2AokGhnMfou2zbqEH64Uiaaw0Zh4ENV8ic3WlP0yI1Sz3KC1cYTsjvWT4ttqaGHIWSwkLfr2XxoLzfw%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)

像 Manus AI 这类能根据人类的指令独立（或人机协同）完成任务，产出完整作品的工具称为【AI Agent】，类似的产品还有Flowith、Coze、Claude、Cursor，Agent功能侧重点各有不同。

**如果我们想突破"对话机器人"的设计局限思维** ，AI Agent产品可实现功能的边界，是非常值得我们学习讨论的。

当然，AI Agent并不是什么魔法般的技术。让AI拥有"干活的手"，有很多技术解决方案。【MCP -Model Context Protocol （模型上下文协议）】是当前实现AI Agent功能的热点技术，也是作为设计师进行相关原型设计时有必要掌握的技术知识点。

MCP 是一种开放式协议，它规范了应用程序向 LLM提供上下文的方式。我们可以把 MCP 想象成**人工智能应用的 USB-C 接口** ，就像 USB-C 提供了将设备连接到各种外设和配件的标准化方式一样，**MCP 也提供了将人工智能模型连接到不同数据源和工具的标准化方式。**

Manus 独立执行调研任务

如果你设计了一个【AI 助手】，希望它能帮助用户完成各种任务，那么这个助手就需要知道用户的信息，比如文件、代码，甚至用户的日程安排。传统技术实现上，我们需要为每一个数据源和工具编写专门的接口，这不仅麻烦，还容易出错。

MCP 就是为了解决这个麻烦问题而诞生的。它可以理解为 AI 应用和各种数据源之间的"通用接口"或者"万能插头"。通过 MCP，你的 AI 助手可以安全、便捷地访问各种外部信息，从而获得进行各种工作的工具使用权限。

MCP 采用客户端-服务器架构，简单来说，可以分为三个部分：

**![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_gif%2FET1CK99sg2AokGhnMfou2zbqEH64UiaawoTNKNE9nNd1QLDDuibR776PUzlkwN6ujQsAJahzfiaVmq4qSAEJicb2OQ%2F640%3Fwx_fmt%3Dgif%26from%3Dappmsg)**

**1/ MCP 主机 (Host)：** 就是用户界面，比如你的 App 或者网页。

**2/ MCP 客户端 (Client)：** 负责管理主机和服务器之间的安全连接，相当于一个"中间人"。

**3/ MCP 服务器 (Server)：** 提供特定功能的外部程序，比如代码仓库、数据库或者 API。

当用户在主机上发出指令时，客户端会把这个指令传递给服务器，服务器执行完指令后，再把结果返回给主机。整个过程就像一个餐厅点餐：

- 你 (用户) = MCP 主机，你告诉服务员你想吃什么。

- 服务员 = MCP 客户端，负责把你的订单传递给厨房，并把菜端给你。

- 厨房 = MCP 服务器，负责制作你点的菜。

MCP技术也为设计创新带来了丰富的灵感和可能性，下面我们也为大家挑选了五个应用场景------

**增强聊天界面**

MCP使得聊天界面的功能得到极大拓展。以旅游规划为例，用户可以通过一个集成MCP的聊天机器人，轻松规划整个旅行行程。用户只需告诉聊天机器人旅行的目的地、时间和预算等基本信息，机器人就能自动调用多个服务的MCP接口，如航班预订、酒店预订、交通安排等，为用户制定详细的旅行计划。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FET1CK99sg2AokGhnMfou2zbqEH64Uiaaw4aP9XnwicsVegQBzsyUdVXZic2HrogfUib23P121ZJBF6zfBbgaOQdChA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)

在旅行过程中，聊天机器人还能根据实时情况调整行程，比如天气变化时提醒用户更改活动安排，或者根据用户的饮食偏好推荐餐厅。这种聊天界面的设计，不仅提供了丰富的功能，还注重了用户与AI之间的自然交互。我们可以在聊天界面中融入更多实用功能，同时保持界面的简洁和易用性，提升用户的参与感和满意度。

**自定义AI工作流**

MCP为打造个性化、智能化的AI助手提供了可能。例如，一个自定义的AI工作流可以记录用户的日常习惯和偏好，如会议时间、常用的应用程序、文件存储位置等，然后根据这些信息自动为用户规划日程、管理任务。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FET1CK99sg2AokGhnMfou2zbqEH64Uiaawqz9DCXjecaMJkf1uhLxgMbRFokRH5hibTicmXIiaHa4qiasvzQdkNTe75A%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)

Coze 支持用户自定义 AI工作流模板

www.coze.cn  

当用户需要安排一个会议时，AI助手可以自动查询参与者的日程，找到合适的时间，并发送会议邀请。同时，AI助手还能根据用户的习惯，提前准备好会议所需的文件和资料。在真实的设计项目中，我们可以思考如何通过MCP连接不同的数据源和工具，让用户能够根据自己的需求定制AI助手，提高工作和生活的效率。

**AI模型治理和上下文控制**

在需要高度安全和合规的场景中，MCP可以帮助监控和控制AI系统的数据访问。例如，在金融行业，AI模型需要处理大量的敏感数据，如用户的账户信息、交易记录等。通过MCP，可以对AI模型的数据访问进行严格的权限管理，确保只有授权的模型和在特定的上下文中才能访问敏感数据。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FET1CK99sg2AokGhnMfou2zbqEH64UiaawVxHupg1OdXCSmw1RLJ88L3IH3G0d4T6104vb8F8kM1fIXmfia0nQJicw%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)

在设计数据隐私要求较高的应用时

关注铭感数据的权限提示与引导设计

同时，MCP还可以记录AI模型的数据访问日志，便于审计和追踪。对于设计背景的同学来说，需要我们在设计涉及敏感数据的应用时，不仅要关注功能的实现，还要考虑如何通过界面设计体现数据安全和隐私保护的理念，比如在用户授权数据访问时，提供清晰的提示和操作指引，增强用户对数据安全的信任感。

**企业环境中的智能客服系统**

在企业环境中，MCP可以用于构建更加智能和高效的客服系统。例如，一个基于MCP的AI客服机器人可以安全地访问多个内部系统，如员工的HR记录、项目管理工具、客户关系管理系统等。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_jpg%2FET1CK99sg2AokGhnMfou2zbqEH64Uiaawt9U9zQ1MxMUTxQw1aaNgMfWicRibkYZmSbamSiclEqQchHgvibjNhs3EbA%2F640%3Fwx_fmt%3Djpeg%26from%3Dappmsg)

当客户提出问题时，AI客服能够实时查询相关信息，并根据上下文提供准确且个性化的回答。这种跨系统的数据访问能力，使得AI客服能够更好地理解和响应用户需求，提供更加智能化和个性化的服务，提升用户体验和企业效率。如何让用户在AI客服功能的使用过程中，感受到自然流畅，获得舒适且具有信任感的使用体验，则是设计工作关注的重点。

**多模态工具集成的创新应用**

MCP支持与多模态工具的集成，为交互设计带来更多的可能性。例如，在教育领域，一个MCP驱动的AI教育平台可以结合文本、图像、音频等多种数据源，为学生提供个性化的学习体验。

![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmmbiz.qpic.cn%2Fmmbiz_png%2FET1CK99sg2AokGhnMfou2zbqEH64UiaawTMKdc8TOPo0vWBQTWCcXOqFcX5wejCQbNlpNMYVWApwT1lwljI4QxA%2F640%3Fwx_fmt%3Dpng%26from%3Dappmsg)

学生可以通过语音提问，AI助手不仅能理解问题，还能根据教材、网络资源等多模态数据生成详细的解答，并以图文并茂的方式呈现。在医疗领域，医生可以利用MCP连接患者的病史记录、医学影像和实验室检测系统，AI助手能够综合分析这些多模态数据，辅助医生进行更准确的诊断和治疗方案制定。
