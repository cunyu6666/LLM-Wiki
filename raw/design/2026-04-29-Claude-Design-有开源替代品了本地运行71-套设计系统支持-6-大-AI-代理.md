# Claude Design 有开源替代品了：本地运行、71 套设计系统、支持 6 大 AI 代理

> 来源: [mp.weixin.qq.com](https://mp.weixin.qq.com/s?__biz=MzkyMTQ0NzY4MQ==&mid=2247483996&idx=1) · brother7 小陈的第二大脑


你有没有想过一个问题：当你用 Claude 或其他 AI 工具生成网页设计稿时，你的 prompt、你的品牌信息、你的业务数据，去了哪里？

答案是：云端。它们被发送到服务商的服务器上，经过大模型的推理，再返回结果。对大多数场景来说这没问题，但对一些涉及商业敏感信息的设计需求------比如内部数据看板、未发布产品的落地页、带真实用户数据的演示文稿------"数据不出本地"是一道硬门槛。

Anthropic 今年推出了 Claude Design，一个集成在 Claude 里的设计生成工具，体验很惊艳。但它有个前提：你得用 Claude 的云端服务。最近 GitHub 上出现了一套叫 **open-design** 的开源项目，走了一条截然不同的路------它不做自己的 AI 模型，而是直接调用你电脑上已经装好的 Claude Code、Codex、Cursor 等本地代理，把所有设计和生成都留在你的机器上完成。

📌 **一句话总结** ：open-design 是一套本地优先的开源设计工具，通过调用你电脑上已有的 AI 代理（Claude Code、Codex、Cursor 等），在浏览器里完成网页原型、PPT、文档等设计内容的生成、预览和导出，所有数据和 API 密钥都留在本地。

## 它不是又一个 AI 生图工具

市面上的 AI 设计工具大致分两类：一类是 Midjourney、DALL-E 这样的"出图工具"，生成的是静态图片；另一类是 v0.dev、Claude Design 这样的"代码生成工具"，输出的是可以直接运行和部署的网页代码。

open-design 属于后者，但它有一个关键差异：**它不绑定任何特定的 AI 服务商** 。它支持 Claude Code、OpenAI Codex、Cursor、Gemini CLI、OpenCode、Qwen 一共 6 种本地代理，自动检测你 PATH 里装了哪个就用哪个。你用自己的 API key，推理过程在你本地或服务商的 API 端点完成，但所有的文件读写、项目存储、预览渲染，全部发生在你的电脑上。

这种"BYOK + 本地运行"的模式，对企业内网环境、对数据敏感型行业、对不想被某个平台锁定的开发者来说，是一个很有吸引力的选项。

## 71 套设计系统意味着什么？

很多人用 AI 生成网页设计时，最大的痛点不是"生成不了"，而是"生成的太像 AI 生成的"。统一的圆角、安全的配色、毫无个性的排版------业内有人戏称这种风格叫"AI slop"（AI 馊饭）。

open-design 应对这个问题的方式很直接：它内置了 **71 套品牌级设计系统** 。不是模糊的"科技感"或"简约风"，而是有名字、有来源的真实设计规范：

🔹 **产品型** ：Linear（干净利落的 SaaS 风格）、Notion（知识管理风）、Stripe（金融科技风）、Figma（协作工具风）

🔹 **平台型** ：Vercel（开发者工具风）、Supabase（开源数据库风）、Anthropic（AI 实验室风）、Cursor（编辑器风）

🔹 **品牌型** ：Airbnb（旅行体验风）、Tesla（硬核工业风）、Apple（极简主义风）、Spotify（流媒体风）

这意味着什么？当你说"生成一个类似 Linear 风格的定价页"时，AI 不是凭空想象 Linear 长什么样，而是直接按照 Linear 真实的设计规范------特定的字体栈、特定的配色方案、特定的间距节奏、特定的按钮圆角------来生成。出来的东西，第一眼就不会让你感到"这是 AI 做的"。

除此之外，它还有 **5 个预设的视觉方向** ，每个方向都有确定性的 OKLch 调色板和字体组合：Editorial Monocle（编辑杂志风）、Modern Minimal（现代极简风）、Tech Utility（技术实用风）、Brutalist（粗野主义风）、Soft Warm（柔和温暖风）。你可以把它理解为"设计系统的元模板"------先选大方向，再选具体品牌，两层筛选确保输出不会跑偏。

## 19 个技能覆盖了哪些场景？

open-design 把不同类型的设计任务封装成了 19 个"技能"（Skill），每个技能本质上是一份告诉 AI 怎么完成特定任务的指令集。这些技能覆盖了从网页到文档的多种输出形式：

🔹 **网页类** ：Web 原型、SaaS 落地页、仪表盘、定价页、博客文章、文档站、404 页面

🔹 **应用类** ：移动 App 界面、组件库

🔹 **演示类** ：PPT 演示文稿（PPTX 格式）

🔹 **管理类** ：产品需求文档（PRD）、运维手册、周报

🔹 **其他** ：简历、邮件模板

每个技能对应不同的输出格式和约束条件。比如"PPT 技能"会生成符合演示文稿比例和阅读节奏的内容，"仪表盘技能"会自动考虑数据可视化的布局逻辑。这种"任务特化"的设计，比让 AI 自由发挥要靠谱得多。

## 五维自我批判：怎么避免"AI 馊饭"？

如果上面提到的 71 套设计系统是从"输入端"控制风格，那 open-design 还有一个从"输出端"控制质量的机制------**五维自我批判门控** 。

简单来说，在 AI 把最终产物输出给你之前，它要先过一道"自检关卡"。这个自检从五个维度审查生成内容的质量：视觉一致性、品牌契合度、排版合理性、交互逻辑、代码规范。如果发现某个维度不达标，AI 会重新生成或修改，直到通过全部检查。

这种设计借鉴了软件开发中的"代码审查"理念------只不过审查者是 AI 自己。它的目的是在内容到达用户之前，拦截掉最明显的低质量输出。同时系统还有一份明确的"AI slop 黑名单"，把常见的设计反模式（比如过度圆角、无意义渐变、模板化布局）列出来，让 AI 在生成阶段就主动避开。

## 技术架构：前后端分离，但全在本地

open-design 的架构很清晰，分为三层：

**前端** ：Vite + React + TypeScript 的单页应用，你在浏览器里看到的所有界面。它通过 API 代理与后端通信。

**守护进程** ：Node + Express + SQLite，运行在本地 7456 端口。它是整个系统的"调度中心"------接收前端的请求，管理项目状态，把任务分发给 AI 代理，并把结果流式返回给前端。

**代理运行时** ：真正的"大脑"。守护进程会在每个项目的独立工作目录里启动你选择的 AI 代理 CLI（如 Claude Code），代理用 Read、Write、Bash、WebFetch 等工具操作真实文件系统。

一个很有意思的细节是**沙盒预览** 。AI 生成的网页代码不会直接在你浏览器的主环境里运行，而是被放进一个隔离的 iframe 里渲染。iframe 里预装了 React 18 和 Babel，确保即使 AI 输出的是 JSX 代码，也能即时预览。这种隔离设计既安全又灵活------AI 写错代码不会搞崩你的界面，你也能实时看到修改效果。

## 上手难吗？

如果你已经装好了 Node.js 和 pnpm，启动只需要几步：

git clone https://github.com/nexu-io/open-design.git  
cd open-design  
pnpm install  
pnpm dev:all

# 然后打开浏览器访问 http://localhost:5173

首次使用时，系统会让你填一个"发现表单"------确定你要做什么类型的内容、面向什么受众、用什么语气、选哪个设计系统。这个表单的存在很聪明：它在 AI 动笔之前就锁定了大方向，避免后续反复修改。

生成过程中，你可以在界面上实时看到 AI 的"待办列表"------每个子任务从"进行中"变成"已完成"的状态更新。配合流式输出，整个过程的透明度很高，你不会干等着猜它到底在干嘛。

## 适合谁？有什么局限？

🎯 **适合的人群：**

🔹 对数据隐私有要求的团队------所有内容本地生成，设计稿不会经过第三方平台

🔹 不想被单一 AI 平台锁定的开发者------6 种代理自由切换，今天是 Claude，明天可以换 Cursor

🔹 需要快速出原型的产品经理和设计师------从想法到可点击原型，分钟级而不是天级

🔹 需要统一设计规范的团队------71 套真实品牌设计系统提供了现成的视觉基准

⚠️ **需要注意的局限：**

🔹 项目目前处于早期实现阶段，虽然核心流程已经跑通，但稳定性和功能完整性还在快速迭代中

🔹 输出质量高度依赖你选择的底层代理------不同代理的代码生成能力差异明显

🔹 复杂交互和动态数据绑定的场景仍然需要人工介入调整

## 写在最后

open-design 的出现，代表了一种值得关注的趋势：**AI 设计工具正在从"云端 SaaS"走向"本地软件"** 。它的价值不仅在于"能做设计"，更在于它给用户提供了选择权和控制权------选哪个 AI、用什么设计系统、数据存在哪里，这些关键决策权回到了用户手中。

对开发者来说，这个项目还有另一层意义：它是 Apache-2.0 开源的。这意味着你不仅可以用它，还可以 fork 它、改它、把它集成到自己的产品里。如果 Claude Design 的闭源模式让你有所顾虑，open-design 提供了一个完全可以自己掌控的替代方案。
