## What is HAX
什么是HAX

As Nielsen quoted [\[2\]](https://www.nngroup.com/articles/ai-paradigm/), UX until now was based on Human-Computer interaction (HCI). In HCI, a set of tasks and instructions are performed by the user to reach the expected output.  
正如尼尔森引用的[\[2\]](https://www.nngroup.com/articles/ai-paradigm/) ，到目前为止，用户体验是基于人机交互（ HCI ）的。在 HCI 中，用户执行一组任务和指令以达到预期输出。

For example, *filling out a form for personal details → identity verification → bank account creation* .  
例如，*填写个人信息表格→身份验证→创建银行账户*。

However, the new paradigm is intent-based outcome specification. Meaning, that the user expects the final output they intend. Like prompting ChatGPT to generate an image of a cat with cute unicorn horns. We have an image output within seconds. This is HAX (Human AI Interaction).  
然而，新的范式是基于意图的结果规范。这意味着用户期望他们想要的最终输出。就像提示ChatGPT生成一张长着可爱独角兽角的猫的图像一样。我们在几秒钟内就可以输出图像。这就是HAX （人类人工智能交互）。

## Why HAX 为什么选择HAX

Our very talented AI assistant hallucinates a lot if not guided properly. lol.  
如果没有正确引导，我们非常有才华的人工智能助手会产生很多幻觉。哈哈。

We define AI by its ability to complete a task. The AI we have now can complete instruction-based tasks, like winning a chess game or identifying a similar character in a series of photos. This kind of AI is categorized as weak or Narrow AI. General/Strong AI is self-conscious and only possible in science fiction movies. [\[3\]](https://www.ibm.com/blog/ai-vs-machine-learning-vs-deep-learning-vs-neural-networks/)  
我们根据人工智能完成任务的能力来定义人工智能。我们现在拥有的人工智能可以完成基于指令的任务，例如赢得一场国际象棋比赛或识别一系列照片中的相似角色。这种人工智能被归类为弱人工智能或狭义人工智能。通用/强人工智能具有自我意识，只有科幻电影中才有可能。 [\[3\]](https://www.ibm.com/blog/ai-vs-machine-learning-vs-deep-learning-vs-neural-networks/)

So if not given proper instructions, the model hallucinates, which leads to wrong or undesired solutions.  
因此，如果没有给出正确的指示，模型就会产生幻觉，从而导致错误或不需要的解决方案。

If we're dealing with a chatbot like ChatGPT, we can say, "eh.. maybe a few more prompts or a prompt engineer can get me the desired result." Meaning, we could try multiple-shot prompting. [\[4\]](https://www.forbes.com/sites/jodiecook/2023/06/26/how-to-write-effective-prompts-for-chatgpt-7-essential-steps-for-best-results/?sh=343884f12a18)  
如果我们正在处理像ChatGPT这样的聊天机器人，我们可以说，"呃..也许更多的提示或提示工程师可以让我得到想要的结果。"这意味着，我们可以尝试多次提示。 [\[4\]](https://www.forbes.com/sites/jodiecook/2023/06/26/how-to-write-effective-prompts-for-chatgpt-7-essential-steps-for-best-results/?sh=343884f12a18)

But in applications that are driven towards unique solutions, we need the output to be specific/ precise. Especially in sensitive industries like medicine.  
但在追求独特解决方案的应用中，我们需要输出具体/精确。尤其是在医药等敏感行业。

HAX Design aims to make the 2-step user experience (intent and output generation) much more meaningful. It aims to avoid possible model errors, direct the users towards the goal, and humanize it. We still need human direction and perception to inform these designs as the Narrow AI we have today is not capable of performing and dissecting things like a human brain can. At the end of the day, we're the ones who need to empathize. 😂  
HAX Design 旨在使两步用户体验（意图和输出生成）变得更有意义。它的目的是避免可能的模型错误，引导用户实现目标，并将其人性化。我们仍然需要人类的指导和感知来为这些设计提供信息，因为我们今天拥有的狭义人工智能无法像人脑一样执行和解剖事物。归根结底，我们才是需要同情的人。 😂

## Important steps of HAX
HAX的重要步骤

The clichéd steps of HCI still apply to any experience we design i.e., putting user needs at the top.  
HCI的陈词滥调步骤仍然适用于我们设计的任何体验，即将用户需求放在首位。

Here are a few steps to follow to break an AI solution into understandable bits to make the user experience better and error-free.  
以下是将人工智能解决方案分解为可理解的部分的几个步骤，以使用户体验更好且无错误。

## Step 1: 步骤一：

Understand the AI solution. Begin by defining the tasks the model is designed to perform. Don't forget to compare it with the user journey that existed before the AI era. (p.s --- there's still a user journey to the experience but the task journey that AI goes through is slightly bigger)  
了解人工智能解决方案。首先定义模型要执行的任务。不要忘记将其与人工智能时代之前存在的用户旅程进行比较。 （ps------体验仍然有一个用户旅程，但人工智能经历的任务旅程稍微大一些）

## Step 2: 步骤2：

Formulating a decoding statement. Like constructing a problem statement.  
制定解码语句。就像构建问题陈述一样。

***AI needs _______ in order to complete the task ________ with _________ limitations.
AI 需要 _______ 才能在 _________ 限制下完成 ________ 任务。***

For example, the app is about generating custom filters and edits on the photos users upload.  
例如，该应用程序用于生成自定义过滤器并对用户上传的照片进行编辑。
![](https://cubox.pro/c/filters:no_upscale()?imageUrl=https%3A%2F%2Fmiro.medium.com%2Fv2%2Fresize%3Afit%3A1400%2F1%2AHRszHaGhceueAlcH06W7jQ.png&valid=false)

## Step 3: 步骤3：

User interface design is backed by 2 things:  
用户界面设计有两点支持：

1. Basic HCI: user research, ideation, brainstorming, lo-fi  
   基础人机交互：用户研究、构思、头脑风暴、低保真
2. **Task Transparency 任务透明度**

Understand what tasks of the model need to be presented as steps of the experience, and what tasks need to be made transparent for ethical reasons.  
了解模型的哪些任务需要作为体验步骤呈现，以及出于道德原因哪些任务需要变得透明。

For our previous example of filter generation application,  
对于我们之前的过滤器生成应用程序示例，

Tasks presented as steps would be:  
作为步骤提出的任务是：

*Tooltips and error messages that help users upload the right kind of image with the proper dimensions and resolution
工具提示和错误消息可帮助用户上传具有适当尺寸和分辨率的正确类型的图像*

Tasks that need to be transparent would be:  
需要透明的任务是：

*Asking for permission such as for using the user data in an "x" way
请求许可，例如以"x"方式使用用户数据*

Because AI is relatively uncharted territory, users tend to be even more cautious about their information. Make sure to include the usage of their data in the permissions. Also, it's critical to make sure no hallucinations are happening as part of structuring our envisioned user journey.  
由于人工智能是相对未知的领域，用户往往对自己的信息更加谨慎。确保在权限中包含其数据的使用情况。此外，在构建我们设想的用户旅程时，确保不会出现幻觉也至关重要。

## Step 4: 第4步：

**User control and customization:
用户控制和定制：**

With AI, it has become possible that a solution can be customized in many ways. This makes applications more diverse and robust.  
借助人工智能，可以通过多种方式定制解决方案。这使得应用程序更加多样化和强大。

With that said, not all users might want to give up all controls for full customizations. It's important for us to provide the interface with "permission toggles" to disable a few advanced features for better user control. This way, user's understanding and trust in the accuracy of AI solutions can be improved.  
话虽如此，并非所有用户都可能想放弃所有控件以进行完全自定义。对我们来说，为界面提供"权限切换"来禁用一些高级功能以实现更好的用户控制非常重要。这样，可以提高用户对人工智能解决方案准确性的理解和信任。

## Step 5: 第5步：

**Iterative testing and improving** with continuous user feedback have always been the key. Hopefully with modern AI, tracking user patterns and analytics should be easier to understand, as the data is pre-organized. Maybe in the future, we won't have to run any scripts to get user patterns 🤷🏽. Regular testing for proper task execution and output generation would bring out any loops that weren't tightened.  
**迭代测试和根据持续的用户反馈进行改进**始终是关键。希望借助现代人工智能，跟踪用户模式和分析应该更容易理解，因为数据是预先组织的。也许将来，我们不需要运行任何脚本来获取用户模式🤷🏽。定期测试正确的任务执行和输出生成会发现任何未收紧的循环。
