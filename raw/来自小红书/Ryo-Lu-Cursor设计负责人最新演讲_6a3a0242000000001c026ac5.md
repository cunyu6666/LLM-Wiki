## 核心论点（一段话）

> AI 把"想到"和"做出"之间的距离坍缩了——以前写 spec → 设计 → 实现 → 评审 → 测试 → 上线，每一步都是昂贵的循环；现在**循环本身变短、变便宜**，想法几乎立刻能变成可触摸的产物。但循环越短、Agent 越强，**"应该做什么"比"能不能做"更重要**——工具让你更接近真相还是更远离真相？

## 演讲骨架（基于 ASR 时间轴）

> 完整英文稿见同名 `.transcript.md`。下面是按时间轴的关键节点。

### [00:00] 开场：用 Ryo OS 自我介绍

- 演讲用他自己用 Cursor 写的 **"Ryo OS"**（一个复古 Mac OS 风的计算环境）来做
- 怀念早期电脑是**开放的游乐场**而非隔离应用

### [00:36–01:46] 循环变了

- 以前：想法 → spec → 设计 → 实现 → 评审 → QA → 下一轮（昂贵，所以事前论证）
- 现在：build → play → notice wrong → ask Cursor reshape → 试 → 再试（循环塌缩）
- "AI is collapsing the distance between ideas and reality"

### [02:06–04:44] 核心命题：循环变了，但"对/不对"问题更尖锐

- "When more things can be built faster by more agents, the question is no longer just, can we build it?"
- "What should exist? Does the tool bring me closer to the truth or push me further away?"
- 旧工作流变成"老虎机"（pulling a slot machine）——快速产出但不一定有 truth

### [05:52–08:31] 死掉的界面

- "Interfaces that are cracked but feels dead, products made by agents, carried by nobody"
- Cursor 内部代号 **"glass"** 的新界面——目标是让循环本身可被看见、被反思
- 关键点：循环不只是 build-inspect，还有 **before / after** 的思考环节

### [09:51–13:40] AI 越强，"前后"越重要

- Agent 工作几分钟、几小时、几天——你必须**信任**循环本身
- "Before the loop there is the question of what to build; after the loop there is the question of what you learned"
- Cursor 用 Cursor 自己的循环举例：Idea → Build → Test → Inspect → Design → 全部由 Cursor 跑

### [16:41–20:37] 收尾：demo 不是 PPT

- "What if the demo was not a slide about software becoming clay?"
- 真正的 demo 是**把演示本身变成可塑的材料**
- 工具悄悄把我们拉向某种未来——我们得意识到是哪一种
