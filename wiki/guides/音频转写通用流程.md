# 音频转写通用流程 (audio-to-text)

> 通用基础设施：把任何音频文件转成带时间戳的文字稿
> 适用：Twitter Spaces (x-audio)、YouTube STT 路线、播客、长访谈、抖音下载的 mp3
> 不适用：已有官方字幕的视频（走 YouTube SOP 优先方案）

---

## 流程总览

```
任意音频文件 (mp3/m4a/mp4/wav) → ffmpeg 统一转 m4a 16kHz mono → Groq Whisper API → 带时间戳 JSON → 写入 raw/<source>/.md
```

零本地 GPU，零模型下载。Groq 每天 1 小时免费额度（whisper-large-v3-turbo），足够日常使用。

## 1. 前置条件

```bash
# ffmpeg（macOS）
brew install ffmpeg

# Groq SDK
pip3 install groq

# API key
export GROQ_API_KEY="gsk_..."
```

> **key 持久化**：已写入 `~/.zshrc` + `~/.zprofile` + wiki `.env`（git 忽略）。新 shell 启动自动加载。

## 2. 音频预处理（统一格式）

Groq 接受 mp3/m4a/wav/webm/mp4，但**统一转 16kHz mono m4a** 可避免时长超限（>25MB 文件需先切片）。

```bash
# 单文件
ffmpeg -i input.mp3 -ar 16000 -ac 1 -c:a aac -b:a 64k output.m4a

# 批量（处理整个目录）
for f in /path/to/audios/*.{mp3,m4a,mp4}; do
  out="${f%.*}.m4a"
  ffmpeg -y -i "$f" -ar 16000 -ac 1 -c:a aac -b:a 64k "$out"
done
```

**参数说明**：
- `-ar 16000`：采样率 16kHz（Whisper 训练采样率）
- `-ac 1`：单声道
- `-c:a aac -b:a 64k`：AAC 64kbps 压缩
- 60 分钟访谈 → 约 30MB（Groq 25MB 限制下可能要切）

## 3. 长音频切片（>25MB 必切）

Groq API 限制单文件 25MB。>60 分钟的音频要切。

```bash
# 用 ffmpeg segment 切（每段 50 分钟，留 10 分钟重叠）
ffmpeg -i input.m4a -f segment -segment_time 3000 -c copy chunk_%03d.m4a
# 或按大小切（更精确）
ffmpeg -i input.m4a -f segment -segment_size 24000000 -c copy chunk_%03d.m4a
```

**重叠策略**：相邻段重叠 30 秒，转写后用 text 重叠去重。

**简单替代**：先尝试直接传，不超 25MB 就不切。

## 4. Groq Whisper API 调用

```python
import os, json
from groq import Groq

client = Groq(api_key=os.environ["GROQ_API_KEY"])

with open("input.m4a", "rb") as f:
    transcription = client.audio.transcriptions.create(
        file=("audio.m4a", f.read()),
        model="whisper-large-v3-turbo",  # 最佳速度/质量比
        language="en",                  # 显式指定可提速+提精度
        # language="zh"                 # 中文用这个
        response_format="verbose_json", # 必须用 verbose 才返回 segments
        timestamp_granularities=["segment"],
        # 可选参数：
        # temperature=0.0,              # 0=确定性强，1=发散
        # prompt="AI 设计, Claude Code, Anthropic"  # 上下文提示词
    )

# 输出 segments
for seg in transcription.segments:
    print(f"[{seg['start']:.1f}-{seg['end']:.1f}] {seg['text']}")
```

**模型选择**：
- `whisper-large-v3-turbo`（推荐）：速度快，精度高
- `whisper-large-v3`：质量更高但慢 ~2x
- `distil-whisper-large-v3-en`：英文最快，精度略低

**支持语言**：99 种，中英日韩法德西等全覆盖。

## 5. 时间戳对齐与段级合并

`verbose_json` 返回的 segments 通常每 5-10 秒一段（按静音切）。需合并成可读段落。

```python
def merge_segments(segments, gap=0.5, max_chars=200):
    """相邻段间隔 < gap 且累计字符 < max_chars 时合并"""
    merged = []
    for s in segments:
        text = s['text'].strip()
        if not text: continue
        if merged and (s['start'] - merged[-1]['end']) < gap and \
           len(merged[-1]['text']) + len(text) < max_chars and \
           not merged[-1]['text'].rstrip().endswith(('.', '?', '!')):
            merged[-1]['end'] = s['end']
            merged[-1]['text'] = (merged[-1]['text'] + ' ' + text).strip()
        else:
            merged.append({'start': s['start'], 'end': s['end'], 'text': text})
    return merged
```

**参数**：
- `gap=0.5`：相邻段间隔阈值（秒）
- `max_chars=200`：单段最大字符数（避免超长难读）

## 7. 错误处理速查

| 错误 | 原因 | 解决 |
|------|------|------|
| `413 Payload Too Large` | 文件 >25MB | 用 ffmpeg 切片 |
| `429 Too Many Requests` | RPM 限流 | 等待 60s 重试，或换 API key |
| `Invalid API key` | key 失效 | 重新设置 `GROQ_API_KEY` |
| 段级合并后仍断句 | gap 阈值太大 | 改 `gap=0.3` |
| 中文标点切碎 | Whisper 标点不规则 | 后处理 regex 规范化 |
| ASR 把"Claude Code" 听成"Clojure Code" | 英文模型对专名识别弱 | 加 `prompt="Claude Code, Anthropic, AI Agent"` |

| 音频时长 | 文件大小 | 转写耗时 | API 成本（免费额度内） |

| 5 分钟 | 5 MB | 3s | 免费 |
