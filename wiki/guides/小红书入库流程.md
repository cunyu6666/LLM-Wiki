# 小红书入库流程 (xhs ingest)

> 首次实战：2026-06-20 / note id `6a2bf71e000000002100a7d2`
> 完整流水线：未登录抓取 → 图片下载 → OCR → 归档 → 反向链接
> 适用：所有需要把小红书笔记沉淀到本 Wiki 的场景

---

## 流程总览

```
小红书 URL → 抓 HTML → 解析 __INITIAL_STATE__ → 下载图片 → OCR → raw/<id>.md → imgs/ → wiki/<主题>/ 索引 → 反向链接 → log.md
```

零依赖（无登录、无 Docker、无 API key），仅需：curl / Python / macOS Vision API。

## 1. 抓取 HTML

```bash
UA="Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1"
curl -sL -A "$UA" \
  -H "Referer: https://www.xiaohongshu.com/" \
  "https://www.xiaohongshu.com/explore/<NOTE_ID>?xsec_token=<TOKEN>&xsec_source=pc_collect" \
  -o /tmp/xhs/page.html
```

**关键**：
- 必须用**移动端 UA** + `Referer`，否则返回空 SPA 壳
- `__INITIAL_STATE__` 内嵌完整笔记数据（title/desc/images/user/interactInfo）
- **未登录也能拿到**，但 author.nickname = null，只有 author_id

## 2. 解析 JSON（小心 undefined）

小红书 HTML 里 `__INITIAL_STATE__` 混进了 `undefined`（非法 JSON），且正则非贪婪 `{...}` 会截断。用括号状态机：

```python
import re, json
clean = re.sub(r'\bundefined\b', 'null', h)
idx = clean.find('__INITIAL_STATE__=')
start = clean.find('{', idx)
depth, in_str, esc = 0, False, False
for i in range(start, len(clean)):
    c = clean[i]
    if esc: esc=False; continue
    if c=='\\': esc=True; continue
    if c=='"': in_str=not in_str; continue
    if in_str: continue
    if c=='{': depth+=1
    elif c=='}':
        depth-=1
        if depth==0: end=i+1; break
state = json.loads(clean[start:end])
```

## 3. 提取字段

```python
def find_note(d, path=''):
    if isinstance(d, dict):
        if 'noteId' in d and isinstance(d.get('noteId'), str) and ('desc' in d or 'title' in d):
            yield path, d
        for k, v in d.items():
            yield from find_note(v, path + '/' + k)
    elif isinstance(d, list):
        for i, x in enumerate(d):
            yield from find_note(x, path + f'[{i}]')

notes = list(find_note(state))  # 一般就 1 个
_, n = notes[0]
# 关键字段
title = n['title']
desc = n['desc']
note_id = n['noteId']
user_id = (n.get('user') or {}).get('userId')
imgs = n.get('imageList') or []
interact = n.get('interactInfo') or {}
# 图片 URL 在 img['infoList'][i]['url']，每个尺寸两套
```

## 4. 下载图片

```python
import urllib.request
hdrs = {'User-Agent': 'Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X)',
         'Referer': 'https://www.xiaohongshu.com/'}
for i, img in enumerate(imgs, 1):
    url = img['infoList'][0]['url']  # 取第一组（原图）
    req = urllib.request.Request(url, headers=hdrs)
    with urllib.request.urlopen(req, timeout=30) as r:
        data = r.read()
    open(f'/tmp/xhs/imgs/{i}.jpg', 'wb').write(data)
```

**坑**：URL 含 `!h5_1080jpg` 标识符，curl 直接传会有 shell 转义问题，**用 Python urllib 最稳**。

## 5. OCR（macOS Vision API，**强烈推荐 Tesseract**）

```bash
cat > /tmp/xhs/ocr.swift <<'SWIFT'
import Foundation, Vision, AppKit
let path = CommandLine.arguments[1]
let img = NSImage(contentsOfFile: path)!
let bmp = NSBitmapImageRep(data: img.tiffRepresentation!)!
let req = VNRecognizeTextRequest { (r, _) in
    for o in r.results as! [VNRecognizedTextObservation] {
        let b = o.boundingBox
        let s = o.topCandidates(1).first!.string
        print("[x=\(b.origin.x) y=\(b.origin.y) w=\(b.size.width) h=\(b.size.height)] \(s)")
    }
}
req.recognitionLevel = .accurate
req.recognitionLanguages = ["zh-Hans", "zh-Hant", "en"]
try VNImageRequestHandler(cgImage: bmp.cgImage!).perform([req])
SWIFT
swiftc /tmp/xhs/ocr.swift -o /tmp/xhs/ocr_bin
for i in 1 2 3 4 5 6 7 8 9; do /tmp/xhs/ocr_bin /tmp/xhs/imgs/$i.jpg; done
```

**为什么用 Vision API 而不是 Tesseract**：
- Tesseract macOS brew 默认**没中文包**（`tessdata` 只有 eng/osd/snum），要 `brew install tesseract-lang` 才能识别中文
- Vision 系统级集成，**零安装**，识别精度比 Tesseract 强
- 自带 boundingBox 坐标，重建阅读顺序极方便

## 6. 重建阅读顺序

Vision API 坐标系 `y=0` 为底，`y=1` 为顶——**y 降序 = 视觉从上到下**。

```python
import re
def parse(path):
    images, current = {}, None
    for line in open(path):
        m = re.match(r'=== IMAGE (\d+) ===', line)
        if m: current = int(m.group(1)); images[current] = []; continue
        m = re.match(r'\[x=([\d.]+) y=([\d.]+) w=([\d.]+) h=([\d.]+)\] (.*)', line)
        if m and current:
            x,y,w,h,s = m.groups()
            images[current].append((float(y), float(x), s))
    for k in images:
        images[k].sort(key=lambda t: (-t[0], t[1]))  # y 降序，同行 x 升序
    return images
```

## 批量入库（≥3 条时）

单条流程扩展为批量可节省 80% 时间。完整实现见 `xhs mega batch #1`（2026-06-20，10 条笔记 47 图）。

### 关键差异点

1. **避开 Python SSL 问题**：用 `xargs -P 6` + `curl` 并发抓 HTML，不要用 Python `urllib`（macOS 证书路径问题）
2. **OCR 重建自适应行高**：固定 `0.012` 阈值在小字段落失效，改用 `h * 1.3`（h 是 OCR 输出的行高）
3. **图片命名空间隔离**：每次入库必须 `imgs/<note_id>/` 子目录，避免串行覆盖
4. **文件名清洗**：`，。：！？` 等中文标点必须替换为 `_`，否则 markdown 渲染失败
5. **wiki 路径修正**：`wiki/design/` 不是 `/design/`

### 批量脚本骨架

```bash
# 并发抓 HTML
printf '%s\n' "${URLS[@]}" | xargs -P 6 -I {} bash -c 'fetch_one "$@"' _ {}

# 并发下载图片（用 curl 避开 SSL）
python3 + ThreadPoolExecutor + subprocess.run(['curl',...])  # 10 worker
```

## 11. 文件名清洗规范（2026-06-20 实战沉淀）

文件名 `<标题>_<note_id>.md` 决定了 raw 主文档路径、wiki 软链、Obsidian wikilink 三处一致性。**清洗失误 = 渲染失败 / 链接断裂**。

### 必须替换为下划线 `_` 的字符

| 类别 | 字符 | 原因 |
|------|------|------|
| 中文标点 | `，` `。` `：` `；` `？` `！` `、` `（` `）` `"` `"` `《` `》` | Markdown 渲染安全，但跨平台文件名易出问题 |
| 中文分隔符 | `｜` `—` `…` `·` | Obsidian wikilink 转义时丢字符 |
| 英文标点 | `?` `!` `:` `;` `,` `.` `(` `)` `[` `]` `"` | macOS 终端 shell 通配误伤 |
| 路径分隔符 | `/` `\` | 路径冲突 |
| 特殊符号 | `*` `?` `<` `>` `|` `&` `$` `#` `@` `!` `+` | shell glob / URL 编码冲突 |

### 空格 vs 下划线

- ✅ **允许空格**（Obsidian wikilink 完全支持空格）
- ✅ 推荐用空格作"自然分词"：`Loop Engineering 是什么 8 张图讲透_<id>.md`
- ❌ 不要全替成下划线（标题会变成难读的 `Loop_Engineering_是什么_8_张图讲透_<id>.md`）

### 实际清洗函数

```python
import re
def clean_title(s):
    # 1. 替换中英文标点为空格
    s = re.sub(r'[，。：；？、（）！""《》｜—…·,.;:?!()\[\]"\']', ' ', s)
    # 2. 替换特殊符号
    s = re.sub(r'[/\\*?<>|&$#@!+]', ' ', s)
    # 3. 多空格合一
    s = re.sub(r'\s+', ' ', s).strip()
    return s

# 例：
# "Loop Engineering 是什么｜8 张图讲透" → "Loop Engineering 是什么 8 张图讲透"
# "Kiro: 字节的"AI 工程师"，好用吗？" → "Kiro 字节的 AI 工程师  好用吗"
```

### 长度限制

- **主标题 ≤ 50 字符**（过长 Obsidian 标签页显示截断）
- 超出截断保留前半：`"一个非常非常长的小红书标题超过五十字符..."` → `"一个非常非常长的小红书标题..."`
- 完整标题放 frontmatter `title:` 字段，文件名用截断版

### 实战案例

| 原标题 | 清洗后 |
|--------|--------|
| `Loop Engineering 是什么｜8 张图讲透` | `Loop Engineering 是什么 8 张图讲透_6a2bdb14000000001503dc0e.md` |
| `Cursor 的 Ryo Lu 谈设计活工具与编程未来` | `Cursor的Ryo-Lu谈设计活工具与编程未来_8ncYSGbfeyY.md` |
| `Ryo-Lu谈AI将设计师变为开发者` | `Ryo-Lu谈AI将设计师变为开发者_PQhcHrCyU8M.md` |

## 12. 已知坑 & 解决方案

| 坑 | 表现 | 解决 |
|----|------|------|
| **imgs/ 同名覆盖** | 第 2 条入库覆盖第 1 条的 `1.jpg` | imgs 强制按 `<note_id>/` 子目录隔离 |
| **shell 历史扩展吞 `!`** | zsh 里 `!h5_1080jpg` 被解释成历史命令 | 用 Python `subprocess.run(['curl', url])` 避开 shell |
| **Python urllib SSL 错** | macOS 证书路径问题 | 用 curl 子进程，**不要**用 Python urllib |
| **未登录 author.nickname=null** | KeyError | `(n.get('user') or {}).get('userId')` |
| **HTML 混 `undefined`** | json.loads 失败 | regex 替换为 null + 括号状态机 |
| **OCR 行高阈值** | 固定 0.012 失效 | 改用 `h * 1.3`（行高的 1.3 倍） |
| **wikilink 渲染失败** | 文件名含特殊字符 | 走"第 11 节：文件名清洗规范" |
| **linter 过度清洗** | "Cursor的" 无空格（应该是 "Cursor 的"） | 接受 linter 输出，或手动 sed 修正 |

## 注意事项

1. **shell 历史扩展**：所有涉及 `![[...]]` 的 sed 替换用 Python 正则，**别用 sed in-place**
2. **空格文件名**：小红书标题常含空格，所有路径用 wikilink 别用 markdown link
3. **URL 含 `!`**：`!h5_1080jpg` 标识符，curl 会被 shell 截断，用 Python urllib
4. **未登录 author**：解析 JSON 时 `user` 可能是 `null`，用 `(n.get('user') or {})` 防 KeyError
5. **图片 URL 两套**：`infoList[0]` 是原图，`infoList[1]` 是缩略图；拿原图
