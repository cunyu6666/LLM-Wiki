## 2 · 认知锚（4 min）—— 囚徒 + 信徒 + Design Agent Loop

### 2.1 反直觉的事实 · 模型是个囚徒（90 秒）

【**PPT：白板。中间画一个圆形代表"模型"，周围画几个工具图标。**】

**"我讲一个反直觉的事实。"**

**"大模型本身——"**

**"它干不了活。"**

【**停顿 2 秒。**】

**"它能想，能说，能判断。"**

**"但它**不能动**。"**

**"它不能自己打开文件，不能自己跑命令，不能自己读数据，不能自己改代码。"**

**"它唯一能做的——是输出一段文字。"**

【**停顿 2 秒。**】

**"你可以把它理解成——"**

【**PPT：圆形中心标一个「嘴巴」icon。**】

**"一个只会发声的囚徒。"**

**"它有大脑，但它没有身体。"**

**"它能告诉世界'我想做什么'，但它做不了。"**

---

### 2.2 信徒 · 给囚徒造身体（90 秒）

**"那它怎么'做事'？"**

【**PPT：在圆形周围画工具 icon（read / write / bash / ls / Figma）。**】

**"工程师做了一件事——"**

**"给囚徒配了一群信徒。"**

**"什么是信徒？就是工具。"**

**"囚徒说'我要看这个文件'——信徒 `read` 去读。"**

**"囚徒说'我要写这段代码'——信徒 `write` 去写。"**

**"囚徒说'我要跑个测试'——信徒 `bash` 去跑。"**

**"跑完把结果喂回给囚徒，囚徒接着想下一步。"**

【**PPT：画一个循环箭头串起来。**】

**"这个循环——"**

**"就是 Agent Loop。"**

【**停顿 3 秒。**】

**"但这个发现让我意识到一件事——"**

**"Agent Loop **不是模型能力的胜利，是工程的胜利**。"**

**"真正让 AI '会做事'的，从来不是模型本身——"**

**"是有人替模型设计了一套工具和流程。"**

【**停顿 2 秒。**】

**"给囚徒造身体的工程。"**

---

### 2.3 关键问题 · 设计 Agent 谁来造（60 秒）

【**停顿 1 秒。**】

**"谁来给 Design Agent 配工具？"**

**"谁来决定——Figma 怎么调、什么时候读、什么设计模式适用、什么规范必须遵守？"**

【**停顿 3 秒。**】

**"是工程师吗？"**

**"不是。工程师不懂设计。"**

【**PPT：白板，画一个更大的循环图。中心是"设计意图"，周围是 4 个 Design Tool 卡片。**】
