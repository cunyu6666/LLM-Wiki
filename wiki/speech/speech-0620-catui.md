### 2.3 维护 monorepo 不是写组件（60 秒）

**"很多人以为设计师维护 monorepo 就是'写写组件、改改样式'。"**

**"不止。"**

**"我参与了 QoderWork Agent Loop 的设计决策。"**

【**停顿 2 秒。**】

**"什么是 Agent Loop？Claude Code 的核心就是一个 while 循环——"**

**"模型生成响应 → 如果有工具调用 → 执行工具 → 结果塞回上下文 → 模型生成下一步。"**

**"Claude Code 30 万行代码，核心就是这个。"**

【**停顿 1 秒。**】

**"在 QoderWork，这个 loop 长这样——"**

【**PPT：白板手绘 / 架构图。**】

**"用户输入 → Chat 入口。"**

**"意图分类 → 路由到 subagent。"**

**"subagent 调用——**包括调用我维护的 `@qoder/design` 组件来构造 UI**。"**

**"工具执行 → MCP、文件、SQL。"**

**"响应生成 → 回到用户。"**

【**停顿 2 秒。**】

**"我的 monorepo 不只是 UI 组件库——"**

**"它是 Agent Loop 的中间产物。"**

**"我的组件直接被 Agent 调用，用来构造 UI。"**

【**停顿 2 秒。**】

**"这件事——不是工程师做的。是我做的。"**

**"我参与了 Agent Loop 的设计决策。这不是工程师的事，是产品最终形态的事。"**

【**停顿 2 秒。**】

**"这条宪法意味着——"**

**"它想配色，对不起，看我的 token。"**

**"它想写按钮，对不起，用我的 Button。"**

【**停顿 1 秒。**】

**"我是设计师。"**

**"AI 在生成代码时，必须服从我。"**

【**停顿 3 秒。**】

**"这不是'保护专业分工'。"**
