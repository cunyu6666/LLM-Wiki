## 阶段 ⑤ 硬件原型：Agent + 硬件（并行线）

> 目标：做一个可以在演讲现场展示的 Agent + 硬件实物原型
> 不阻塞 6/25 PPT 交付，PPT 里先用概念图/渲染图占位

### 5.1 方案决策（6/15 - 6/16）

- [ ] **确定硬件方案**
  - 方案 A：**桌面 Agent 状态屏**（ESP32 + 小屏幕，显示 Agent 运行状态/配额）
    - 优点：简单、实用、wiki 已有参考（Clawdmeter）
    - 难度：⭐⭐
    - 周期：3-5 天
  - 方案 B：**桌面 AI 交互设备**（ESP32 + 屏幕 + 麦克风/扬声器，可跟 Agent 对话）
    - 优点：演示效果炸裂，直接呼应演讲主题
    - 难度：⭐⭐⭐⭐
    - 周期：7-10 天
  - 方案 C：**智能穿戴设备**（墨水屏手表/手环，显示 Agent 推送信息）
    - 优点：可穿戴 = 更前沿
    - 难度：⭐⭐⭐
    - 周期：5-7 天
  - 方案 D：**Agent 物理按钮/控制器**（一个实体按钮，按下触发 Agent 执行任务）
    - 优点：极简但概念强——"把 Agent 的控制权变成物理交互"
    - 难度：⭐
    - 周期：2-3 天

- [ ] **采购硬件清单**（根据选定方案）
  - ESP32 开发板（推荐 ESP32-S3，带 USB）
  - 屏幕模块（SPI 屏 / I2C OLED / 墨水屏，根据方案选）
  - 外壳材料（3D 打印 / 亚克力 / 现成盒子）
  - 杜邦线、面包板、USB 线等
  - 下单渠道：淘宝/拼多多（注意物流时间，预留 2-3 天到货）

### 5.2 环境搭建（6/16 - 6/17）

- [ ] **安装 Rhino**
  - 下载安装 Rhino 8（有 90 天试用）
  - 或考虑 Fusion 360（免费个人版）/ Blender（免费开源）
  - 跑通基础操作：画一个简单外壳 → 导出 STL

- [ ] **搭建 ESP32 开发环境**
  - 安装 PlatformIO（VS Code 插件）或 Arduino IDE
  - 跑通 blink 示例
  - 确认屏幕驱动库可用

- [ ] **参考项目研究**
  - Clawdmeter（ESP32 显示 Claude Code 配额）：https://github.com/HermannBjorgvin/Clawdmeter
  - qpaperOS（墨水屏手表）：https://github.com/qewer33/qpaperOS
  - M5Stack StackChan（ESP32 机器人）

### 5.3 外壳建模（6/17 - 6/19）

- [ ] **Rhino 建模**
  - 测量 PCB / 模块的实际尺寸
  - 设计外壳：考虑屏幕开孔、USB 口、散热、走线
  - 导出 STL

- [ ] **3D 打印**
  - 自有打印机 → 直接打
  - 没有 → 淘宝 3D 打印服务（上传 STL，选材料，2-3 天到货）
  - 材料建议：PLA（快速验证）/ PETG（更耐用）

### 5.4 固件开发（6/18 - 6/21）

- [ ] **基础固件**
  - 屏幕驱动：显示文字、图标、简单动画
  - WiFi 连接：接入网络
  - API 调用：对接 Agent 后端（可以是简单的 HTTP 轮询）

- [ ] **Agent 集成**
  - 显示 Agent 运行状态（idle / working / done）
  - 显示 Agent 输出摘要
  - 可选：物理按钮触发 Agent 任务
  - 可选：语音输入/输出（方案 B）

- [ ] **交互设计**（这是设计师的主场）
  - 信息层级：什么信息最重要、一瞥能看到什么
  - 动效设计：状态切换的过渡动画
  - 物理交互：按钮手感、屏幕角度、摆放位置

### 5.5 组装 & 调试（6/21 - 6/23）

- [ ] **组装**
  - PCB + 屏幕 + 外壳组装
  - 走线整理
  - 供电方案（USB 供电 / 电池）

- [ ] **调试**
  - 功能测试：所有功能正常运行
  - 稳定性测试：长时间运行不崩溃
  - 演示流程排练：模拟演讲现场的展示流程

### 5.6 演讲展示准备（6/24 - 演讲前）

- [ ] **展示方案**
  - 方案 A：实物带上台，演讲到案例 4 时拿出来展示
  - 方案 B：提前录好演示视频，PPT 里播放
  - 方案 C：A + B 结合——实物展示 + 视频演示详细流程
  - 建议：至少准备录屏备份，防止现场硬件翻车

- [ ] **展示脚本**
  - 30 秒快速展示：这是什么、能干什么
  - 2 分钟详细演示：Agent 下发任务 → 设备响应 → 物理反馈
  - Q&A 环节：观众可以近距离看/摸

## 风险 & 应对

| 风险 | 应对 |
|------|------|
| 5 篇原文抓取失败 | 只用标题和摘要，不影响核心论点 |
| 个人案例涉及保密 | 用脱敏数据或通用化描述 |
| PPT 制作时间不够 | 优先保证内容完整，视觉可以后续迭代 |
| Demo 现场翻车 | 准备录屏作为备选 |
| 大纲迟迟定不了 | Day 3 强制锁定，后续只改细节 |
| **硬件到货延迟** | **PPT 里先用概念图/渲染图占位，实物延期不影响 6/25** |
| **硬件开发难度超预期** | **降级方案：用现成开发板 + 面包板，不做外壳，先跑通功能** |
| **Rhino 学习曲线太陡** | **降级到 TinkerCAD（网页端，30 分钟上手）或直接用 Fusion 360** |
| **3D 打印质量不行** | **淘宝找专业打印服务，或改用亚克力激光切割** |
| **QoderWork 品牌露出审批** | **提前跟公司确认能否公开演讲，准备脱敏版方案** |
