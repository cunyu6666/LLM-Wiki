### 开场：设计即将经历同样的变革

今天，Framer 在进化。接下来要展示的东西，将从根本上改变你在 Framer 中设计和工作的方式。我们还会分享关于 Framer 社区和市场的重大更新。但先聊产品。

过去一年，工程师的工作方式发生了比过去几十年加起来还剧烈的变化。AI 和 Agent 彻底改变了软件的制造方式。我们相信设计即将经历同样的转型。但说实话，到目前为止，很多 AI 设计工具更像新奇玩具而非真正突破。一开始好玩，但很快跟真正的专业创意工作脱节。产出千篇一律，工作流僵硬，工具总是试图替代创意过程而不是支持它。

而且你也不能把一个普通营销人员或设计师丢到 Claude Code 前面然后指望一切顺利。所以我们决定做点更好的。

### 全新体验：围绕 Agent 构建

今天我们推出围绕 Agent 的全新 Framer 体验。我们相信画布优于代码——Agent 创建的每一样东西，都和你自己在画布上设计的一样可编辑、可控。整个过程变成完全协作的。

你可以从 Agent 获得超快、高质量的产出，跳进去自己改，再交回给 Agent 完成——它会尊重你做的修改。因为 Framer 深度聚焦网站，产出从一开始就是生产就绪的。没有东西要清理，没有代码要检查，直接发布。

我们也设计成在工作流的任何层级都能配合你。它当然能从零开始构建整个网站。但更重要的是——我们认为大多数人实际会这样用——它作为专家级 Framer 设计师、开发者、内容编辑、SEO 专家、分析专家，在你所有现有项目里工作。

无论你在数十个页面上做视觉刷新、为团队重组 CMS 集合、给全站加交互动画，还是全新生成页面区块，Framer Agent 都能搞定。

### 分支（Branching）：安全探索

我们都知道 AI 不完美。Framer 是人们做真实工作、管真实网站、承担真实风险的地方。所以伴随 Agent，我们还推出了分支。

分支给你一种安全的方式来探索想法、审查变更、对比版本，准备好了再发布——同时保持当前线上页面完全不变。

### 演示：Agent + 分支实战

**Joseph**：我在做一个设计公司的网站，还有很多工作要做。在新的 Framer 里，你会发现 UI 非常精简。右侧属性面板现在分为我们熟悉的"样式"标签和全新的"Agent"标签——和 Framer Agent 对话的地方。

可以从下拉菜单选模型——我个人喜欢 Opus 4.8——然后开始写第一条提示词。

我们的网站已经有了作品、案例研究、服务页面，甚至 CMS 驱动的博客。我从小处开始：让 Agent 在服务页面创建一个新的服务区块。我用新的上下文工具直接点选主页上的 wrapper 层，把它插入提示词。你可以想插多少层就插多少层。

"列出 13 项服务，每项一个标题和一个描述，底部有细线分隔，间距宽松，全宽。"

发送后，Canvas Agent 开始工作，实时显示它在操作哪些层。它使用整个项目的内容、结构和风格作为上下文。聊天里还能看到 Agent 在做什么的有用信息。我可以继续对话迭代，也可以直接在画布上改——有时候直接动手 rearrange 比打字解释更快。

这就是整个理念：Agent 和我共同编辑。

我让它给每个服务项加上最合适的图标。这些图标不是随机的——Agent 从矢量库里根据主题精心挑选，并且匹配了风格。点击撤销/重做可以快速对比前后效果。如果我想回退到发送提示词之前，点"回退"再确认就行。

一个小细节："给这个区块加标签，加一个'查看所有服务'的链接，跟其他区块保持一致。" 我甚至没告诉它链接到哪个页面，但基于对话上下文，Agent 理解应该链到完整的服务页面。

### 生成新页面

现在生成一个全新页面。"创建一个联系页面，三个办公地点：旧金山、纽约、阿姆斯特丹。包含城市、地址、邮箱，配风景城市照片并排。每个上面放我已经做好的时钟组件。匹配作品页面的头部布局。全站补上缺失的链接。"

因为操作在另一个页面发生，我可以点"跟随"跳到 Agent 正在工作的地方。新页面正在成型。它还在跑，但我知道想把图片放右边，就在它运行时直接挪过去。布局和内容看起来不错——它用占位信息填了我没给的地址和邮箱，还自己找了每个城市的图片。但这张不太像旧金山。"换一张更有标志性的 SF 图片。" "再多点雾。" 业余的雾。

### 从截图设计

我们还可以从视觉参考设计。Agent 非常擅长从图片重建功能性 Web UI。"添加一个匹配这个截图的联系表单。" 上传同事发来的截图——它构建出了一个完全可编辑的功能性表单。

### 动画与全局操作

回到主页做动画。"给 Hero 区块的每个元素加一个微妙的交错出现效果，从背景开始，然后从上到下引入前景内容。1 秒时长，每个延迟差 0.1 秒。"

预览效果很好。每层的时间微微错开。有些裁切问题——"确保父级 frame 不裁切任何内容。" 修好了。

这种繁琐的、重复性的工作，就是交给 Agent 的。"给所有页面上每个包含标题的文本层加微妙的文字动画。进入视口时触发，按字符动画，上移并淡入。"

Hero 看起来不错。不太喜欢模糊效果——"去掉模糊。" 长标题动画太慢了——"长标题太慢了，Hero 可以。" Agent 说它从按字符改成了按词——效果好多了。"再慢一点。" 搞定。

我不可能自己去找每一个动画标题逐个调整。

### 响应式 + 分支

现在桌面端看起来不错，但只有桌面端。让 Agent 让整个网站响应式——但这是大量页面的大量修改。正是使用分支的好时机。

画布顶部有个"main"标签。点它看所有分支。点加号创建新分支——一个安全的小沙盒，不影响主项目。右键重命名为"breakpoints"。

在新分支上："让整个网站所有页面响应式。给需要替代布局的组件创建变体，比如 footer。"

它在生成断点、做必要的布局调整。作品页看起来不错。服务页大部分不错，有个层的文本样式可能没绑对——等下修。Agent 做了我要的 footer 变体。但主页的 Hero 在手机上不好看——"适配到 H1，合理间距。" 好多了。

### 样式清理

网站有很多文字。让 Agent 自动从文本层创建文本样式？不——那会把所有错误的字体也固化为样式。更现实的做法：我们已经有该用的文本样式，但有些层没绑定。

"列出全站所有没有绑定样式的文本层。排除自适应尺寸的。" 找到了一堆。它们可点击，可以逐个查看。

Agent 主动提出自动匹配最近的样式并应用。输入"yes"。它在全站逐个文本层处理。这个段落肯定刚被修了——检查样式标签，body 样式如期应用。撤销/重做对比可以看到之前偏得有多远。

"用最近的大三度间隔优化标题尺寸的对比度。" 但 Hero 的 H1 被向下取整了——"把 H1 设到更大的间隔。" 好多了。忠于原始设计，但数学上和谐。

### 主题

网站只设计了暗色模式。切到亮色模式什么都没变。"用当前值做暗色模式，为亮色模式创建新值。静态颜色不动。" 整个网站得到了亮色模式的反转主题，保留了暗色模式的原始风格。Hero 区域的静态颜色没被动到，避免了黑上加黑。

"把所有颜色样式调成微冷的色调。降低对比度，不要纯黑或纯白。" 很微妙，但撤销/重做可以看到效果。去资产面板看颜色——全是蓝色色相，带一点饱和度。说实话不太喜欢这个网站的效果——回退。

但对最终结果满意，回到分支弹窗，点"应用到 main"合并分支。变更立刻集成到主设计里。

### 设计页面

所有这些都完美适用于设计页面。"在新的设计页面上创建所有文本和颜色样式的完整风格指南。" 设计页面被自动填充了视觉指南。

也可以创建所有组件的贴纸表。或者探索不同的设计方向——选中 Hero，"创建 6 个截然不同的布局探索，只用现有样式和颜色，尝试不同的液态渐变着色器，但颜色完全不变。" 几秒钟内，6 个带动画着色器的布局变体就出来了。太疯狂了。

### 制作层面的工作

**Justin**：Joseph 做了大量改动，布局和方向都有了。但通常到这里真正的生产工作才开始。Agent 怎么帮我们打磨和扩展已有的东西？

**图片替换**：三个占位图要换成审批过的图。直接把文件拖进聊天："用正确的图片替换联系页面上的占位图。" 搞定。

**文案审计**："全站文本中哪个形容词用得最多？" Agent 发现"bold"用太多了。"把'bold'的实例替换为匹配我们语气的同义词。" 它不是盲目查找替换——它看上下文，做仍然匹配网站语气的修改。"把所有被动语态的文案改成主动语态。" 以前需要专人过一遍文案的工作，现在几个提示词搞定。

**组件化**：服务列表每项结构一样——"把这些列表项变成带变量的单个组件，用每个卡片的内容填充变量。" 巨大的时间节省。

"审计全站，找更小的重复元素，看哪些适合做成可复用组件。" Agent 给出摘要，你决定哪些值得组件化。

**移动导航**：给导航组件的手机变体创建打开/关闭状态——"用菜单文字按钮显示导航链接，打开时变成关闭按钮。" 预览手机断点——移动导航工作正常，状态设好了，交互全接好了。Agent 处理了繁琐的设置，你仍然可以跳进去做最后的打磨。

**代码组件**：Agent 能从零创建炫酷的交互代码组件，但更好的是它能用画布上已有的东西作为起点。"把这个模拟时钟变成带时区属性的功能时钟。" Agent 从画布上的视觉方向出发，让它变得可运行。大到一个交互式 Hero，小到一个自动更新年份的版权组件。

**CMS**：Agent 在 CMS 上能省掉大量繁琐设置。"从服务页面的 13 个服务项目创建一个服务 CMS 集合，把主页的服务区块连上去，只显示 6 项。" Agent 不只是创建新集合——它还把主页上的静态服务替换成了从新 CMS 集合动态填充的列表。

结构性更新："团队集合的每项引用他们做过的项目。我想在工作集合里加一个多引用字段，引用对应的团队成员。" 现在每个工作项引用了正确的团队成员。

日常清理："批量重命名博客 CMS 条目，用清晰打磨的文章标题。" "简化一批 slug 并重定向旧 URL。" 这些事不难，只是重复。

用 CMS 内容设计新区块："在关于页面展示 CMS 中的团队成员，按部门分组，标签在左，卡片在右用网格排列。" 因为 Agent 有项目的完整上下文，它用已有的样式、间距、排版和设计模式来生成——看起来属于这里。

### 外部 Agent

**Andy**：这只是 Agent 的冰山一角。我们还知道人们喜欢以非常不同的方式使用 AI。有些人想要一切都内置在 Framer 里。但其他人已经围绕 Claude Code、Cursor、Codex 等工具建立了强大的工作流，带着自己的习惯、技能和配置。

所以我们同时推出外部 Agent——你可以直接从最爱的外部 AI 工具在 Framer 里构建，从 API 拉数据来设计，或者用自己的自定义技能来设计或审计网站。

**Joseph**：比如用 Claude Code 通过终端直接连到 Framer 项目。实际上可以合并两个项目、用 design.md 的规则设计、让 LLM 整理一堆文件然后构建有组织的关联 CMS。

演示：把项目时间倒回 CMS 完全为空的时候。画布上到处是空 frame 和组件。电脑上有一大堆内容——CSV 文件、markdown 文章、图片文件夹，各种格式互相引用。

复制项目链接，到终端：`claude`，输入 `/framer`，粘贴链接，回车。浏览器弹出授权——批准。Claude Code 正式连上了 Framer 项目。

粘贴一大段详细指令，把整个文件夹拖到终端添加路径。开始。它有一些问题——案例图片怎么映射到字段？要 4 个图片字段。客户比案例多怎么办？全导入。博客文件叫"blog"但设计里叫"articles"？好问题，就叫 articles。

提交答案，等它完成。回到 Framer——画布上看起来什么都没发生。但页面面板显示有页面在另一个分支上被编辑了。Agent 把那些巨大改动安全地放在一个叫"warm fjord"的分支里。切到 CMS 视图——五个集合、客户带所有字段、团队、照片映射正确、工作带所有图片和引用字段、文章——一切都在。

检查画布：主页有了工作和文章的集合列表。工作页填满了每个项目。详情页用了我提供的精确布局。关于页有团队、照片、头衔。最新文章的列表和详情页都搞定了。

Claude 没删那些不再需要的布局页面——我手动删掉。分支准备好了，点"应用到 main"。

外部 Agent 打开了很多有趣的可能性：把 Framer 项目的设计模式变成可复用的 design.md；导出带 SEO 优化建议和决策理由的文案审计。"并行跑性能审计、SEO 审计和无障碍审计。" 它会为每个审计启动独立的子 Agent，输出 .md 文件。

因为 Framer 只是你能用外部 LLM 接入的众多工具之一，你可以把各种 API 和 MCP 服务器拉进来。已经连了 Notion MCP——"把它们加到 Notion"，粘贴链接。所有 markdown 内容整整齐齐，可以分享给团队。

反过来也行：让 LLM 爬 Notion 页面，解析需要的内容导入 CMS。或者从 FigJam 抓取内容、从 Figma 文件获取样式和 token。

来个有趣的：用 X API 找推文，做成卡片滚动条。Agent 决定把它做成 CMS 驱动的集合列表——共享一个设计精良的卡片组件，变量绑定到 CMS。按 Cmd+P 预览——一个 CMS 驱动的推文动画滚动条，真实数据，一条提示词搞定。

### 模型与未来

模型在飞速进化，新能力几乎每天都有。有了外部 Agent，你可以随时利用这些进步。无论你想要原生 Framer 体验，还是连接已经在中心工作流中的其他工具，Framer 都准备好了。

### 社区与市场

今天花了大量时间聊 AI。但我们也在改善 Framer 生态系统中最"人"的部分——社区。因为当创作变快，社区更重要。当更多人能构建、发布、交付好作品，发现和反馈更重要。

今天我们把更多东西整合到一个屋檐下——全新的 Framer 社区。包括 Framer 市场、画廊，以及新增的创作者体验：发布最新作品的 feed、浏览和发现人才的成员页面、以及竞赛。

市场物品现在支持评论、点赞，创作者和客户之间有更丰富的反馈循环。审核和发布体验完全改版——创作者可以在准备好后立即发布。

这就是 Framer 的方向：用 Agent 加速工作、用分支让团队掌控、用社区帮创作者分享作品、被发现、找资源、参加挑战、保持连接。
