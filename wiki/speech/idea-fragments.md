## 🟡 F-001 · Figma 查找和替换：一个"看似简单"功能背后的工程与设计挑战

**来源**
- Moonvy 设计素材周刊 #034（2022/10/09）：https://moonvy.com/blog/post/%E8%AE%BE%E8%AE%A1%E7%B4%A0%E6%9D%90%E5%91%A8%E5%88%8A/034/
- Figma 原文：https://www.figma.com/blog/behind-the-feature-find-and-replace/

**核心摘录**

> 我们第一次开始进行查找和替换时，我认为它会很简单，一个看似很小的功能可以揭示一个错综复杂的工程和设计挑战。
>
> 画布不像网页或文本文档那样从上到下流动；它有一个用户定义的结构，我们需要能够预测它，以便无论您使用哪种类型的文件，排序都是有意义的。
>
> 我们围绕部分的形状如何影响排序方向（以及许多其他细微差别）形成了启发式方法。

**为什么可能值得放进演讲**

- 可作为"**设计师本来就该是工程参与者**"的反例论据：Figma 这个查找替换功能的真正难点不在 UI 而在数据结构（画布的 user-defined order），如果设计师不参与到工程层级的判断，做出来的就是"看似简单实则错的"功能
- 跟③ 干货段 3.3 的 `select_design_skill` / `recommend_design_context` 相通：**好的设计 SOP 必须包含对底层结构的理解**，不只是排版与视觉
- 也可以拿来反衬"设计师 = 只画 Figma"的旧观念——讲一个"连 Figma 自己做查找替换都要设计师下到工程层"的故事，特别有共鸣

**候选用法**

- **A · 故事引子**：放在 ② Day 1（"我以为我要画 Figma"）的引子里，30 秒一带，建立"设计师早就该懂工程"的预期
- **B · ③ 干货段 3.3 注脚**：在 4 个工具表后追加一句"连 Figma 团队都承认：一个查找替换功能就要设计师介入工程判断"
- **C · Q&A 弹药**：被问"那你之前到底怎么算设计师"时掏出来——"设计师本来就不是只画稿的人"

**风险**

- 2022 年的素材，对 D20（2026 年）来说**有点旧**，可能被听众觉得"老黄历"
- 跟主线"AI Agent 时代"关系是间接的——需要桥接句把"设计师下到工程"和"设计师写 Agent Loop"接上
- 如果引用，需要 1 张 Figma 原 blog 截图作为 PPT 配图

**🟡 待你决定**：A / B / C / 弃用

## 🟡 F-002 · 编码是设计师被低估的工具（2022 · UX Design Bootcamp）

**来源**
- Moonvy 设计素材周刊 #014（2022/05/22）：[[设计素材周刊-014]]
- 原文：https://bootcamp.uxdesign.cc/coding-is-an-underrated-tool-for-designers-bda15fee44eb
- 案例应用：https://www.sinwaver.com/?ref=onepagelove （生成基于参数的曲线，GIF demo）

**核心摘录**

> 如果您是设计师，编码是您可以尝试的最好的事情（违反直觉），因为**事物的外观与事物的工作方式息息相关**。编码并不可怕，它给桌面带来了真正的魔力。

**为什么可能值得放进演讲**

- **正中标题命脉**：标题"从 Design 到 Build"——这篇文章 4 年前就提了一句几乎一模一样的话："**外观和工作方式息息相关**"。可作为"这条路不是 AI 才让设计师走的，是设计本来就该走的"的历史佐证
- 跟①开场"墙正在被推倒"的题解形成**时间对位**：4 年前是少数人喊，2026 年是 Cursor / Anthropic / QoderWork 都在做——**趋势比 AI 早**
- `sinwaver.com` 那个参数曲线案例是个**视觉锚点**——讲"编码给设计带来真正的魔力"时配 GIF，比纯口号更有说服力

**候选用法**

- **A · ① 开场段后接半句**：在 Ryo Lu 引文之后追加一句"——其实这个判断 4 年前就有人写过，[uxdesign 引文]"，加深"行业共识"分量。但要小心多塞一句会破坏开场节奏
- **B · ② Day 3 引子**："我的设计交付物从 .fig 变成 .tsx"那段之前，引这句话当心理准备
- **C · Q&A 弹药**：被问"设计师怎么开始学写代码"时调用 + sinwaver 案例 → 直接给一个"从画面到代码"的具体可触摸案例
- **D · 弃用**：跟 F-001 一样是 2022 年素材，可能"老了"，且跟 v2 主线 Agent Loop 关系不直接

**风险**

- **2022 年素材**，对 D20（2026 年）听众有点旧——但因为"编码 + 设计"这个判断本身**经历过时间**反而成了优点（"4 年前就有人这么说，今天才被 AI 兑现"）
- 跟 F-001 风险相同：跟"AI Agent / 造身体"主线**间接**，需要桥接句
- 如果跟 F-001（Figma 查找替换）同时引用，**会显得"老料堆叠"**，需要二选一或合并表述

**与 F-001 关系**

- 都讲"设计师早就该懂工程"，**主张相同**
- 区别：F-001 讲的是"做产品功能时设计师要懂数据结构"，F-002 讲的是"设计师亲手写代码会更好"
- 两条**择一即可**——若选 F-002 留着，F-001 优先级降为 Q&A 弹药

**🟡 待你决定**：A / B / C / D；以及与 F-001 的取舍
