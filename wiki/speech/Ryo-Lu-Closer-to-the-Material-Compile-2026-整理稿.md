# Ryo Lu（Cursor）：Closer to the Material

> **整理自 STT 原始稿**（Groq Whisper）。砍掉时间戳、ASR 错误、英文夹杂；保留金句与论证。
> **D20 用途**：可作为"AI 时代工具设计哲学" / "Agent 与判断力" / "Cursor 产品观"三块主题的源材料。

---

## 一、开场：用 Ryo OS 自我介绍

这场演讲本身是**用 Cursor 写的**，并且演示在 Ryo Lu 自己用 Cursor 搭的 **Ryo OS** 上跑——一个复古 Mac OS 风的桌面计算环境。

> 「我怀念一种感觉——电脑是一个开放的游乐场，不是孤岛化的应用、不是命令行提示符，是个能让你摆弄和思考的地方。」

Ryo OS 从一个 vibe soundboard 小工具开始，逐渐变成一个「能在里面思考的地方」。Ryo Lu 说：**It has my soul.** 这就是他觉得此刻不一样的根本原因。

---

## 二、核心命题：循环变了，"对不对"的问题更尖锐

### 1. 想法 ↔ 现实的距离被 AI 坍缩

> 「AI 在坍缩想法和现实之间的距离——从'想要它存在'到'让一片它成为真的'之间的距离。」

具体到开发：

- 让 Agent 读整个代码库
- 制定计划
- 编排更多 Agent
- 验证正确性
- 继续循环

**但这种加速让老问题变得更尖锐**：当更多事情能被更多 Agent 更快做出——

> 「问题不再是'我们能不能做出来'。问题变成了'**什么应该存在**'。更个人一点说：**这个工具让你离真相更近，还是更远**？」

### 2. 旧工作流的代价：循环昂贵，所以我们事前论证

| 阶段 | 角色 | 备注 |
|------|------|------|
| 想法 → spec | 你 | |
| 设计 | 设计师 | |
| 实现 | 工程师 | |
| 评审 | PM | |
| 测试 | QA | |
| 进入下一件事 | 所有人 | 循环到此为止 |

> 「循环很贵，所以我们守在循环外——动工前先吵一仗、动手前先规划、提前就要做对，因为做错很贵。」

### 3. 新工作流：循环变快、变短、变便宜

> 「现在你能有一个想法 → 要一个版本 → 检查结果 → 改 → 跑 → 继续。有时几分钟，有时几秒。」

这不只是生产力的改变——**它改变了判断力的形成方式**：

- 第一版几乎从来都不对
- 概念太模糊
- 过渡显得 jank
- 间距技术上对，但视觉上错
- 文案说的是对的，但说的不是真的

**「你只有当它存在之后才会注意到这些，然后你改它。这个小修正就是判断力形成的地方。」**

### 4. AI 的深层承诺：让更多人进入循环

> 「你试，世界回答；你注意到哪儿不对；你调整；你建立直觉。**这个循环就是品味形成的地方，就是手艺发展的地方，就是你变好的地方。**」

所以 AI 的承诺不是"帮我们更快做更多东西"。**更深的承诺**是：让更多人能**按自己想要进入多深就多深**进入这个循环——

- 设计师离实现更近
- 工程师能探索不同的架构方向
- 从没把自己当程序员的人，能做出他们想象中东西的第一版

## 三、危险面：循环被隐藏，人变成批准者

### 1. 黑箱化

> 「随着循环变快，它也可能被隐藏起来。当循环被隐藏，人们停止从中学东西。工具变成黑箱——你许个愿、等一下、拿到输出。**接受或拒绝**。如果行得通，你不知道为什么；如果失败了，你不知道哪儿失败；如果它改变了你的工作方式，你甚至可能没意识到你自己身上发生了什么改变。」

> 「**这就不是 build，这是在拉老虎机。**」

### 2. 这一刻的吊诡之处

> 「两件事同时为真——更多人能 build，更多平庸的东西能被 build。更多代码被写、更多原型出现、更多功能上线，但投入到决策上的注意力可能更少。」

真正的风险——

> 「不是机器变得太有创造力，而是人类在它们周围变得太被动。」

### 3. Mediocre things 不可怕，可怕的是无限复制

> 「当事情做起来几乎不要钱，平庸的东西不一定看起来坏。它们看起来还行、它们能用、它们上线、它们占据空间。**而现在可以有无限个它们**——能跑但毫无意义的软件。」

> 「裂了但死气沉沉的界面，Agent 做的产品，没人维护，扔进虚空。」

### 4. Output vs Material（核心对立）

| 维度 | Output（输出） | Material（材料） |
|------|----------------|------------------|
| 终点 | 「这是答案」 | 「摸摸它、塑它、让它成你的」 |
| 人的位置 | 批准者 | 作者 |
| 循环 | 结束 | 邀请回来 |
| 后果 | 人变弱 | 人变强 |

> 「**Making is not just producing an artifact. Making is a way of thinking.**（制作不只是产生产物，制作是一种思考方式。）」

> 「如果 AI 把太多过程藏起来，它仍然能给你 output，但它拿走了那些你变强的挣扎。」

### 5. Renaissance 还是 slop？

> 「如果 AI 让执行变便宜但让判断力变弱，我们不会迎来文艺复兴。我们得到的是**能工作的按钮裹着泔水**（slop with working buttons）。」

## 四、为什么去 Cursor：玻璃界面原则

### 1. 个人动机

> 「我加入 Cursor 是因为我个人很强烈地感受到这个问题——Cursor 和 Code 能帮到更多人，**不是通过把每个人都变成程序员**，而是通过让工具足够灵活，从最简单的需求一直 scale 到最专业的需求。」

### 2. 从 VS Code 继承开始，重新问"今天该长什么样"

Cursor 继承了 VS Code 的成熟编辑器 / 强原语 / 快速移动的能力。但世界变了——模型变强、Agent 更能干、交互形态开始偏移。

> 「问题突然不再是'我们怎么把 AI 加到一个代码编辑器里'。问题是——**如果我们今天在一个 Agent 的世界里设计一个软件环境，它应该长什么样**？」

不是聊天面板挂在 IDE 上，不只是 autocomplete，不只是 AI 给你答案的地方——

> 「而是人和 Agent 能在同一份材料里协作的地方。」

### 3. Glass（玻璃）作为界面原则

> 「我说的 glass 不是视觉风格——不是 backdrop filter blur、不是 liquid glass。我说的是**一个界面原则**。系统应该让你**看穿它的工作**——不是说你想控制一切，而是有时那一件小事会告诉你整件事对不对。」

| 维度 | 黑箱（Black Box） | 玻璃（Glass） |
|------|-------------------|--------------|
| AI 行为 | 消失后返回裁定 | 按你想看到的方式工作 |
| 你能看见 | 没有 | 计划 / 思考 / 工具流 / 改动 / 命令 |
| 你的控制 | 接受或拒绝 | 停 / 塑 / 深检 / 接管 |
| 学习机会 | 零 | 按需披露，无限深度 |
| 优化目标 | 干净的输出 | **人的能动性（human agency）** |
| 软件变成 | 一个愿望和裁定 | **一块黏土（clay）** |

**Progressive disclosure, infinite depth.** 是用户的选择，不是我们（设计师）的强加。

### 3. Agent 越长越需要可读性

> 「这在 AI 更强时更重要——Agent 工作几分钟，你需要 steer；工作几小时，你需要几小时可读性；工作几天，你需要离开整个系统。你需要知道发生了什么、查看状态、设计护栏、表达你的意图。**当它被隐藏时，你只是在许愿，许愿不是一个好界面。没有可见性的权力不是工具，是风险。**」

答案不是拒绝自动化（那很傻），也不是永远手动参与每一步（那很累）。**关键是选择**：

> 「你应该能 step out of the machine and carry the work，也应该能 step back in when the closest matters。因为有些决策不只是任务——那是你的品味进入材料的地方，你的价值观变得可见的地方，**那个东西开始感觉像是来自某个人**的地方。」

## 五、BabyCursor → BabyGlass：Cursor 自己怎么 build with Cursor

### 1. 探索阶段（1 月起）

团队从「更激进的 Cursor 界面重置」开始探索。内部代号：

- **BabyCursor 3** = Electron 原型，基于 Cursor CLI
- **BabyGlass** = 新原型环境（变成 web app → 跨团队共享更容易）

### 2. 用 Cursor build Cursor，循环自己变紧

> 「因为我们在用 Cursor build Cursor，那个循环自己变得紧得多——Idea, prototype, reaction, edit, repeat。」

关键不是速度本身，而是**更快的原型帮我们抵达 conviction（确信）**：

> 「取代用文档和观点对齐人，**我们能通过 demo 看穿产品**。取代远观预测产品，**我们能通过 build 发现它**。」

### 3. BabyGlass 的属性

| 属性 | 内容 |
|------|------|
| 类型 | Web app（不是本地 IDE） |
| 关键问题 | "n agents × n projects 时，界面应该什么感觉？" |
| 状态可见性 | 默认该显示多少状态？ |
| 导航 | 跨越不同概念和 scope 是什么感觉？ |
| 简单性 vs 深度 | 既要够简单，又要给关心的人显示深路径 |

> 「这些问题你不能只在文档或 Figma mock 里回答——你必须 feel 它们，你必须 build 一个东西、用它、注意哪儿不对、改它。」

## 六、循环的前与后：留给人的是什么

> 「**如果 Agent 能执行更多循环，留给我们的是什么**？我认为答案在循环的前与后。」

### 1. 上游（Before）：是什么让你开始

> 「循环之前是那个让你开始的东西——一个你不能忽视的 gap、一个让你烦的感觉、一个**某物应该存在**的 feeling，即使你还没法清楚解释。」

**Mac 时代的回忆**作为 Ryo Lu 的源头：

- Dock 弹跳
- Genie effect 横扫
- Exposé 把窗口像桌上的卡片一样散开

> 「没有一件是必要的，但**所有这些都让你感觉有人在乎**。软件有质感、有意见、你能感觉到它背后那个人。」

后来发生了什么：

> 「一路上，很多软件变得更平滑、更快、更一致，**然后莫名其妙变得更无趣**。quirk 被去掉因为测试不好。warmth 被砍掉因为不可测。我们优化到了一个完美工作但感觉空无一物的世界。」

> 「而现在 AI 把速度推到非人——一个下午能生成整个产品、午饭前能 ship 一个功能、喝咖啡前能 spin 起 10 个变体。**疯了，但陷阱是同一个：做东西几乎不要钱时，slop 也变得免费。**」

### 2. 下游（After）：责任

> 「循环之后是责任——**我们向世界释放了什么？它改变了什么？它服务了谁？它让什么变得更容易变成？**」

> 「Agent 能执行循环、能 multiply 循环、能做 10 版、100 版、1000 版，**但它们不知道什么值得保护**。它们不知道技术上对但感觉死。它们不知道这个东西悄悄把我们拉向哪种未来。**那仍然是我们的责任。**」

### 3. 手艺（craft）的迁移

> 「手艺不会消失，它迁移。当执行昂贵时，手艺活在执行本身——一行小心的代码、一处精确的布局、一句重写 12 次的文案，那些东西仍然重要。但当生成变便宜，手艺移到上游和下游。」

| 阶段 | 手艺的位置 | 具体内容 |
|------|------------|----------|
| 上游 | 判断 | 问什么、留什么、拒什么、哪里慢下来、什么不做 |
| 中游（执行） | （被 Agent 接管） | |
| 下游 | 责任 | 释放什么、改变什么、服务谁 |

> 「**Making is not only producing an artifact, making is a bet on what kind of the world should exist.**（制作不只是产生产物，制作是一个关于**世界应该存在哪种形态**的下注。）」

> 「Craft does not disappear, it moves to the edges of judgment.（手艺不会消失，它移到判断的边缘。）」

## 七、收尾：让 demo 本身成为黏土

> 「这场演讲从一个 Ryo OS 里的小笔记开始——一个'某物应该存在'的 feeling。到结尾时，那个 note 变了。不是因为想法变完美了，是因为**它变真实了**。我能 build 它的一片、用它、feel 哪儿不对、改那一处、再试、再试、再试。」

> 「某个时刻，这个东西不再只是一个想法。它变成一个**我能在里面思考的地方**。我能用这个东西思考这个东西。**那是我 build 的时刻**——不是生产力，不是 disruption，不是为了某个大词。」

> 「**那是一样只存在于你头脑里的东西，变得真实到能被另一个人看见、摸到的时刻**。这部分没变、不会变。工具会变、模型会变、经济会变，但这一段留着。」

> 「你感觉哪儿不对，你 mark 一下；世界回答；你让它 real；最终这东西变得 real 到能 touch 到别人。**那是我想要的未来**——不是人类旁观、软件自建；我想做让人**感觉离材料更近**的工具。**让想象与现实之间的循环更短、更清晰、更诚实**的工具。」

---

## 八、一句话总结

> **AI 让循环变快、变便宜；但循环变快也意味着它能被藏起来；藏起来意味着人变成批准者而非作者；**所以 Cursor 做「Glass」——让循环本身可读、可塑、可学——让 software 重新变成 clay 而不是 verdict。

**[Para 1]** 大家好，我是 Ryo，我在 Cursor 做设计。这是今天的演讲：「Closer to the Material」——AI 如何改变我们 build 的方式，以及它**不能抹掉**的东西。我希望这个东西存在。这场演讲是在 **Ryo OS** 里写的，一个我用 Cursor 搭的小计算环境。

**[Para 2]** 我做它是因为我怀念一种感觉——电脑像一个开放的游乐场。不是孤岛化的应用、不是命令行提示符，是能让我**摆弄和思考**的地方。

**[Para 3]** 很长时间里，这种东西只能是一个卡在我脑子里的想法。一张便条、一张草图。有时候我会跟朋友描述它们，但它们从来没真实过。但现在循环不一样了。

**[Para 4]** 我能 build 它的一片、玩它、注意哪儿不对。我可以让 Cursor 重塑它、让整个东西演化、改那一处细节、再试、再试、再试。

**[Para 5]** 最后 Ryo OS 不再只是一个 vibe 播放器。它变成了一个**我能在里面思考的地方**。它装着我的灵魂。这就是为什么此刻感觉真的很不一样。

**[Para 6]** AI 正在**坍缩**想法与现实之间的距离——从"想要它存在"到"让一片它真实"之间的距离。

**[Para 7]** 你可以打一行字，看着它活过来。你可以让 Agent 读整个代码库、做计划、编排更多 Agent、验证正确性、然后继续循环。这太不可思议了。但它也让一个老问题更尖锐了。

**[Para 8]** 因为当更多事情能被更多 Agent 更快地 build，问题不再只是——**我们能 build 它吗？**

**[Para 9]** 问题变成了——**什么应该存在？** 更个人一点说——这个工具让你离真相**更近**，还是**更远**？

**[Para 10]** 这就是我想讲的事——AI 如何改变我们 build 的方式，以及它不能抹掉什么。

**[Para 11]** AI 改变的最重要的事是**循环**。以前，做软件是一个慢循环。

**[Para 12]** 你有一个想法 → 你把它翻译成 spec → 设计师设计 → 工程师实现 → PM 评审 → QA 测试 → 然后大家转到下一件事。循环很贵，所以我们守着它。

**[Para 13]** 我们在 build 之前先吵一仗、在碰材料之前先规划、提前就要做对，因为做错很贵。

**[Para 14]** 但现在循环快得多了。

**[Para 15]** 你可以有一个想法、要一个版本、检查结果、改、跑、继续。

**[Para 16]** 有时几分钟，有时几秒。**这不只是生产力的改变，它改变了判断力形成的方式。**

**[Para 17]** 因为第一版几乎从来都不对。概念太模糊、过渡显得 jank、间距技术上对但视觉上错。

**[Para 18]** 文案说的是对的，但说的不是真的。你只有当它**存在**之后才会注意到这些，然后你改它。

**[Para 19]** 这个小修正，就是**判断力形成的地方**。你试一下，世界回答；你注意到哪儿不对；你调整；你建立直觉。

**[Para 20]** 这个循环是**品味形成的地方**。是**手艺发展的地方**。是**你变好的地方**。所以 AI 的承诺不是"帮我们更快做更多东西"。**更深的承诺**是：它能让**更多人按自己想要进入多深就多深**地进入这个循环——设计师能离实现更近；工程师能探索不同的架构方向；那些从没把自己当程序员的人，能做出他们想象中东西的第一版。

**[Para 21]** 这是好的部分，这是我在乎的部分。但循环变快，它也可能被**隐藏**起来。当循环被隐藏，人们停止从中学东西。工具变成黑箱——你许个愿、等一下、拿到输出。

**[Para 22]** 接受或拒绝。如果行得通，你不知道为什么；如果失败了，你不知道哪儿失败；如果它改变了你的工作方式，你可能甚至没意识到你自己身上发生了什么改变。

**[Para 23]** **这就不是 build，这是在拉老虎机。**

**[Para 24]** 循环之所以重要，是因为它**让你保持清醒**。

**[Para 25]** 它逼你去注意，它要求你在乎。

**[Para 26]** 如果 AI 让执行变便宜、但让判断力变弱，我们不会迎来文艺复兴。

**[Para 27]** 我们得到的是**能工作的按钮裹着泔水**。

**[Para 28]** 这就是此刻这个时刻很吊诡的地方。

**[Para 29]** 两件事同时为真。

**[Para 30]** 更多人能 build，更多平庸的东西能被 build。

**[Para 31]** 更多代码被写、更多原型出现、更多功能上线，但投入到决策上的注意力可能更少。

**[Para 32]** 什么应该存在？为什么应该存在？

**[Para 33]** 什么应该被拒绝？我们应该在哪里慢下来？

**[Para 34]** 当事情做起来几乎不要钱，平庸的东西不一定看起来坏。它们看起来还行、它们能用、它们上线、它们占据空间。**而现在可以有无限个它们**——能跑但毫无意义的软件。

**[Para 35]** **裂了但死气沉沉的界面**，Agent 做的产品，没人维护，扔进虚空。对我而言，那才是真正的风险。不是机器变得太有创造力，而是**人类在它们周围变得太被动**。

**[Para 36]** 危险的版本很舒服——你许个愿，机器消失，它带着一个看起来合理的东西回来。按钮能用，页面能渲染，demo 能过——但你从来不在决策里。你只是在批准表面。

**[Para 37]** 危险的不是更多人能做软件——这部分是好的。危险的是**界面把人变成批准者**。危险的是我们把 **output（输出）** 和 **material（材料）** 混为一谈。

**[Para 38]** **Output 结束循环。Material 邀请你回来。** Output 说"这是答案"。Material 说"摸摸它、塑它、让它成你的"。差别真的重要。因为制作不只是产生产物。制作是一种**思考方式**。如果 AI 把太多过程藏起来，它仍然能给你 output——但它拿走了那些你变强的挣扎。

**[Para 39]** 我不喜欢那个未来。我不想要那种**软件自己 build 自己、而人类只是看着**的未来。

**[Para 40]** 那是一个无聊的未来。我想要**让人感觉离材料更近**的工具。

**[Para 41]** 我加入 Cursor 是因为我个人很强烈地感受到这个问题——我觉得 Cursor 和 Code 能帮到更多人，**不是通过把每个人都变成程序员**，而是通过让工具**足够灵活**，从最简单的需求一直 scale 到最专业的需求。

**[Para 42]** Cursor 从 VS Code 继承了很多东西。那是个巨大的优势。它给了我们一个成熟的编辑器、强的原语、以及快速移动的能力。

**[Para 43]** 但围绕那个基础的世界变化很快——模型变强、Agent 更能干、交互形态开始偏移。突然问题不再是"我们怎么把 AI 加到一个代码编辑器里"。

**[Para 44]** 问题变成了——**如果今天我们在一个 Agent 的世界里设计一个软件环境，它应该长什么样？**

**[Para 45]** 不是挂在 IDE 上的聊天面板、不只是 autocomplete、不只是 AI 给你答案的地方——而是**人和 Agent 能在同一份材料里协作的地方**。

**[Para 46]** 那就是我们的新界面背后的动机——我们内部叫它 **glass（玻璃）**。

**[Para 47]** 我说 glass 的时候，我不是说一种视觉风格——不是 backdrop filter blur、不是 liquid glass。

**[Para 48]** 我说的是一个**界面原则**。系统应该让你**看穿它的工作**——不是说你想控制一切，而是因为有时那一件小事会告诉你整件事对不对。

**[Para 49]** 在黑箱里，AI 消失后返回裁定。

**[Para 50]** 在 Glass 里，AI 按你想看到的方式工作。

**[Para 51]** 你看到计划。

**[Para 52]** 你看到思考和工具流进来。

**[Para 53]** 你看到改动。

**[Para 54]** 你看到命令在跑。

**[Para 55]** 你能停它、塑它、深检、接管。你不必读每一行，但**你一直可以**。这个选择很重要。

**[Para 56]** 对一个资深程序员，这意味着他们能让 Agent 流起来，在需要时跳进去。对设计师，这意味着他们能接受结构、调整那一处让它感觉对的细节。

**[Para 57]** 对新程序员，这意味着他们能慢下来、读计划、通过**看见**来学习。同一个系统应该支持这三种人——**渐进披露、无限深度**。这是用户的选择，不是我们（设计师）的。

**[Para 58]** 这在 AI 更强时更重要——Agent 工作几分钟，你需要 steer；工作几小时，你需要几小时可读性。

**[Para 59]** 工作几天，你需要离开整个系统。你需要知道发生了什么、查看状态、设计护栏、表达你的意图。**当它被隐藏时，你只是在许愿，而许愿不是一个好界面。没有可见性的权力不是工具，是风险。** 所以答案不是拒绝自动化——那很傻。答案也不是永远手动参与每一步。

**[Para 60]** 那会让人筋疲力尽。重点是**选择**。你应该能 step out of the machine and carry the work，也应该能 step back in when the closest matters。

**[Para 61]** 因为有些决策不只是任务——那是**你的品味进入材料**的地方，你的**价值观变得可见**的地方，**那个东西开始感觉像是来自某个人**的地方。

**[Para 62]** **黑箱为干净的 output 优化。Glass 为人的能动性优化。** 黑箱把软件简化成一个愿望和一个裁定。Glass 把软件**变成一块黏土**。Glass 有权利**离材料更近**。

**[Para 63]** 一月份我们开始探索这个想法——一个更激进的 Cursor 界面重置。Glass 内部从 BabyCursor3 开始——一个基于 Cursor CLI 搭的 Electron 原型。

**[Para 64]** 重点不是重建整个生产架构。重点是创造一个**我们能亲自动手、不被 VS Code 的包袱拖累**的环境。

**[Para 65]** 一个快速、灵活的游乐场，让我们能同时探索大的结构变动和小机制。当 1-to-n Agent 和 0-to-1-to-n 项目协作时，界面应该感觉怎样？

**[Para 66]** 默认该显示多少状态？在不同概念和 scope 之间导航是什么感觉？

**[Para 67]** 它感觉够简单吗，但同时为那些在意的人显示更深的路径？

**[Para 68]** 这些问题你不能只在文档或 Figma mock 里回答——你必须**感受**它们，你必须 build 一个东西、用它、注意哪儿不对、改它。

**[Para 69]** 因为我们在用 Cursor build Cursor，那个循环自己变得更紧——Idea、prototype、reaction、edit、repeat。速度本身不是有趣的部分。

**[Para 70]** 有趣的是**更快的原型帮我们抵达 conviction（确信）**。取代用文档和观点把人拉齐，**我们能通过 demo 看穿产品**。取代远观预测产品，**我们能通过 build 发现它**。

**[Para 71]** 在某个点，原型不再是实验。

**[Para 72]** 我们把它做真了。

**[Para 73]** 然后 BabyCursor 演化成了 BabyGlass。

**[Para 74]** BabyGlass 是我们的新原型环境。

**[Para 75]** 它变成了 web app。

**[Para 76]** 这真的重要，因为分享变容易了。团队里更多人能在同一个游乐场工作。

**[Para 77]** 原型变得更不像私人沙盒，更像一个**共享的思考未来的环境**。

**[Para 78]** 这就是我想让 AI 工具创造的循环——不只是更快的 output，而是**更快地接触材料、更快地学习、更快地分歧、更快地协作、更快地抵达确信**。

**[Para 79]** 因为最好的想法不是完整成形才出现的——它们是当你保持足够近、注意你想让它们变成什么时，才变清晰的。所以——**如果 Agent 能执行更多循环，留给我们的是什么？**

**[Para 80]** 我认为答案在循环的**前与后**。循环之前，是那个让你开始的东西——一个你**不能忽视的 gap**、一个让你烦的感觉、一个"**某物应该存在**"的 feeling，即使你还没法清楚解释。

**[Para 81]** 这个 feeling 我有很久了。有一个用 Mac 的时刻。

**[Para 82]** 我们想给专业客户一个更强大的用户界面——但与此同时，我们想让它是**一个从未碰过电脑的人的梦想界面**。

**[Para 83]** 那个用 Mac 的时刻感觉像是在摸一样活着的东西——Dock 弹跳、Genie effect 划过、Exposé 把窗口像桌面上的卡片一样散开。**没有一件是必要的，但所有这些都让你感觉有人在乎。**

**[Para 84]** 软件有质感，有意见，你能感觉到它背后那个人。一路上，很多软件变得更平滑、更快、更一致，**然后莫名其妙变得更无趣**。quirk 被去掉因为测试不好。warmth 被砍掉因为不可测。我们——

**[Para 85]** 把我们优化到了一个**完美工作但感觉空无一物**的世界。而现在 AI 把速度推到非人——你能一个下午生成整个产品、午饭前 ship 一个功能。

**[Para 86]** 喝咖啡前 spin 起 10 个变体。疯了，但陷阱也是同一个——**做东西几乎不要钱时，泔水也变得免费**。所以人类更重要，不是更不重要。

**[Para 87]** **品味不是 prompt，在乎不是参数。** 你放进某件东西里的那种奇怪、具体、个人的东西，不能被平均成存在。在 Notion，我以某种方式在乎这件事——人们不需要另一个单点应用。

**[Para 88]** 他们需要**能变成他们工作形状的简单原语**。在 Cursor，我以另一种方式在乎这件事。人们不只是需要能产出代码的 AI。他们需要一个**他们能保持作为自己作品的作者**的环境。

**[Para 89]** Ryo OS 来自同样的本能，但更个人化。我怀念电脑**感觉像地方**的时刻。我怀念**有质感、有历史、有玩心、有点奇怪**的软件。

**[Para 90]** 我想看看当一个 AI 系统**不必住在聊天框里**而是住在**环境里**，会发生什么。

**[Para 91]** 如果文件、应用、音乐、笔记、地图、设置、记忆、Agent ——**全都就是一件事**，会怎样？

**[Para 92]** 如果 demo 不是一张关于"软件变成黏土"的幻灯片呢？

**[Para 93]** 如果**demo 本身就是那块黏土**呢？

**[Para 94]** 那个最初的痒不是 Agent 能替你生成的。

**[Para 95]** 那个东西必须来自**活在世界里、注意哪儿不对**。

**[Para 96]** 然后循环之后，是**责任**。

**[Para 97]** 我们向世界释放了什么？

**[Para 98]** 它改变了什么？

**[Para 99]** 它服务了谁？

**[Para 100]** 它让什么**变得更容易变成什么**？

**[Para 101]** Agent 能执行循环、能 multiply 循环、能做 10 版、100 版、1000 版——**但它们不知道什么值得保护**。

**[Para 102]** 它们不知道某样东西技术上对但感觉**死**。

**[Para 103]** 它们不知道这个东西悄悄把我们拉向**哪种未来**。那仍然是我们的责任。手艺不会消失——它会**移动**。当执行昂贵时，手艺活在执行本身。

**[Para 104]** 小心的代码行、精确的布局、改了 12 次的小段文案——这些仍然重要。

**[Para 105]** 但当生成变便宜，手艺**向上下游迁移**。

**[Para 106]** 上游到判断——问什么、留什么、拒什么、哪里慢下来、什么不做。下游到责任——释放什么、改变什么、服务谁。

**[Para 107]** **制作不只是产生产物，制作是一个关于"世界应该存在哪种形态"的下注。** 手艺不会消失，它移到判断的边缘。

**[Para 108]** 这场演讲从 Ryo OS 里的一张小便条开始——一个"某物应该存在"的 feeling。现在到结尾，那张便条变了。

**[Para 109]** 不是因为想法变完美了，是因为**它变真实了**。我能 build 它的一片、用它、感受哪儿不对、改那一处、再试、再试、再试。

**[Para 110]** 某个时刻，这个东西不再只是一个想法。它变成一个**我能在里面思考的地方**。我**能用这个东西来思考这个东西**。

**[Para 111]** 那就是我 build 的**时刻**——不是生产力，不是 disruption，不是为了某个大词。

**[Para 112]** **那样东西只存在于你头脑里，变得真实到能被另一个人看见和摸到的时刻**。这部分没变、不会变。工具会变、模型会变、经济会变——但这一段留着。

**[Para 113]** 你感觉哪儿不对、你 mark 一下；世界回答；你让它 real；最后这东西变得 real 到能 touch 到别人。**那是我想要的未来。**

**[Para 114]** 不是那种人类旁观、软件自建的未来。我想做**让人感觉离材料更近**的工具。**让想象与现实之间的循环更短、更清晰、更诚实**的工具。
