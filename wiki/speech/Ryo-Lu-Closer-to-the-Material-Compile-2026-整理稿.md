# Ryo Lu（Cursor）：Closer to the Material

> **整理自 STT 原始稿**（Groq Whisper）。砍掉时间戳、ASR 错误、英文夹杂；保留金句与论证。
> **D20 用途**：可作为"AI 时代工具设计哲学" / "Agent 与判断力" / "Cursor 产品观"三块主题的源材料。

---

## 一、开场：用 Ryo OS 自我介绍

这场演讲本身是**用 Cursor 写的**，并且演示在 Ryo Lu 自己用 Cursor 搭的 **Ryo OS** 上跑——一个复古 Mac OS 风的桌面计算环境。

> 「我怀念一种感觉——电脑是一个开放的游乐场，不是孤岛化的应用、不是命令行提示符，是个能让你摆弄和思考的地方。」

Ryo OS 从一个 vibe soundboard 小工具开始，逐渐变成一个「能在里面思考的地方」。Ryo Lu 说：**It has my soul.** 这就是他觉得此刻不一样的根本原因。

---

## 二、核心命题：循环变了，"对不对"的问题更尖锐

### 1. 想法 ↔ 现实的距离被 AI 坍缩

> 「AI 在坍缩想法和现实之间的距离——从'想要它存在'到'让一片它成为真的'之间的距离。」

具体到开发：

- 让 Agent 读整个代码库
- 制定计划
- 编排更多 Agent
- 验证正确性
- 继续循环

**但这种加速让老问题变得更尖锐**：当更多事情能被更多 Agent 更快做出——

> 「问题不再是'我们能不能做出来'。问题变成了'**什么应该存在**'。更个人一点说：**这个工具让你离真相更近，还是更远**？」

### 2. 旧工作流的代价：循环昂贵，所以我们事前论证

| 阶段 | 角色 | 备注 |
|------|------|------|
| 想法 → spec | 你 | |
| 设计 | 设计师 | |
| 实现 | 工程师 | |
| 评审 | PM | |
| 测试 | QA | |
| 进入下一件事 | 所有人 | 循环到此为止 |

> 「循环很贵，所以我们守在循环外——动工前先吵一仗、动手前先规划、提前就要做对，因为做错很贵。」

### 3. 新工作流：循环变快、变短、变便宜

> 「现在你能有一个想法 → 要一个版本 → 检查结果 → 改 → 跑 → 继续。有时几分钟，有时几秒。」

这不只是生产力的改变——**它改变了判断力的形成方式**：

- 第一版几乎从来都不对
- 概念太模糊
- 过渡显得 jank
- 间距技术上对，但视觉上错
- 文案说的是对的，但说的不是真的

**「你只有当它存在之后才会注意到这些，然后你改它。这个小修正就是判断力形成的地方。」**

### 4. AI 的深层承诺：让更多人进入循环

> 「你试，世界回答；你注意到哪儿不对；你调整；你建立直觉。**这个循环就是品味形成的地方，就是手艺发展的地方，就是你变好的地方。**」

所以 AI 的承诺不是"帮我们更快做更多东西"。**更深的承诺**是：让更多人能**按自己想要进入多深就多深**进入这个循环——

- 设计师离实现更近
- 工程师能探索不同的架构方向
- 从没把自己当程序员的人，能做出他们想象中东西的第一版

## 三、危险面：循环被隐藏，人变成批准者

### 1. 黑箱化

> 「随着循环变快，它也可能被隐藏起来。当循环被隐藏，人们停止从中学东西。工具变成黑箱——你许个愿、等一下、拿到输出。**接受或拒绝**。如果行得通，你不知道为什么；如果失败了，你不知道哪儿失败；如果它改变了你的工作方式，你甚至可能没意识到你自己身上发生了什么改变。」

> 「**这就不是 build，这是在拉老虎机。**」

### 2. 这一刻的吊诡之处

> 「两件事同时为真——更多人能 build，更多平庸的东西能被 build。更多代码被写、更多原型出现、更多功能上线，但投入到决策上的注意力可能更少。」

真正的风险——

> 「不是机器变得太有创造力，而是人类在它们周围变得太被动。」

### 3. Mediocre things 不可怕，可怕的是无限复制

> 「当事情做起来几乎不要钱，平庸的东西不一定看起来坏。它们看起来还行、它们能用、它们上线、它们占据空间。**而现在可以有无限个它们**——能跑但毫无意义的软件。」

> 「裂了但死气沉沉的界面，Agent 做的产品，没人维护，扔进虚空。」

### 4. Output vs Material（核心对立）

| 维度 | Output（输出） | Material（材料） |
|------|----------------|------------------|
| 终点 | 「这是答案」 | 「摸摸它、塑它、让它成你的」 |
| 人的位置 | 批准者 | 作者 |
| 循环 | 结束 | 邀请回来 |
| 后果 | 人变弱 | 人变强 |

> 「**Making is not just producing an artifact. Making is a way of thinking.**（制作不只是产生产物，制作是一种思考方式。）」

> 「如果 AI 把太多过程藏起来，它仍然能给你 output，但它拿走了那些你变强的挣扎。」

### 5. Renaissance 还是 slop？

> 「如果 AI 让执行变便宜但让判断力变弱，我们不会迎来文艺复兴。我们得到的是**能工作的按钮裹着泔水**（slop with working buttons）。」

## 四、为什么去 Cursor：玻璃界面原则

### 1. 个人动机

> 「我加入 Cursor 是因为我个人很强烈地感受到这个问题——Cursor 和 Code 能帮到更多人，**不是通过把每个人都变成程序员**，而是通过让工具足够灵活，从最简单的需求一直 scale 到最专业的需求。」

### 2. 从 VS Code 继承开始，重新问"今天该长什么样"

Cursor 继承了 VS Code 的成熟编辑器 / 强原语 / 快速移动的能力。但世界变了——模型变强、Agent 更能干、交互形态开始偏移。

> 「问题突然不再是'我们怎么把 AI 加到一个代码编辑器里'。问题是——**如果我们今天在一个 Agent 的世界里设计一个软件环境，它应该长什么样**？」

不是聊天面板挂在 IDE 上，不只是 autocomplete，不只是 AI 给你答案的地方——

> 「而是人和 Agent 能在同一份材料里协作的地方。」

### 3. Glass（玻璃）作为界面原则

> 「我说的 glass 不是视觉风格——不是 backdrop filter blur、不是 liquid glass。我说的是**一个界面原则**。系统应该让你**看穿它的工作**——不是说你想控制一切，而是有时那一件小事会告诉你整件事对不对。」

| 维度 | 黑箱（Black Box） | 玻璃（Glass） |
|------|-------------------|--------------|
| AI 行为 | 消失后返回裁定 | 按你想看到的方式工作 |
| 你能看见 | 没有 | 计划 / 思考 / 工具流 / 改动 / 命令 |
| 你的控制 | 接受或拒绝 | 停 / 塑 / 深检 / 接管 |
| 学习机会 | 零 | 按需披露，无限深度 |
| 优化目标 | 干净的输出 | **人的能动性（human agency）** |
| 软件变成 | 一个愿望和裁定 | **一块黏土（clay）** |

**Progressive disclosure, infinite depth.** 是用户的选择，不是我们（设计师）的强加。

### 3. Agent 越长越需要可读性

> 「这在 AI 更强时更重要——Agent 工作几分钟，你需要 steer；工作几小时，你需要几小时可读性；工作几天，你需要离开整个系统。你需要知道发生了什么、查看状态、设计护栏、表达你的意图。**当它被隐藏时，你只是在许愿，许愿不是一个好界面。没有可见性的权力不是工具，是风险。**」

答案不是拒绝自动化（那很傻），也不是永远手动参与每一步（那很累）。**关键是选择**：

> 「你应该能 step out of the machine and carry the work，也应该能 step back in when the closest matters。因为有些决策不只是任务——那是你的品味进入材料的地方，你的价值观变得可见的地方，**那个东西开始感觉像是来自某个人**的地方。」

## 五、BabyCursor → BabyGlass：Cursor 自己怎么 build with Cursor

### 1. 探索阶段（1 月起）

团队从「更激进的 Cursor 界面重置」开始探索。内部代号：

- **BabyCursor 3** = Electron 原型，基于 Cursor CLI
- **BabyGlass** = 新原型环境（变成 web app → 跨团队共享更容易）

### 2. 用 Cursor build Cursor，循环自己变紧

> 「因为我们在用 Cursor build Cursor，那个循环自己变得紧得多——Idea, prototype, reaction, edit, repeat。」

关键不是速度本身，而是**更快的原型帮我们抵达 conviction（确信）**：

> 「取代用文档和观点对齐人，**我们能通过 demo 看穿产品**。取代远观预测产品，**我们能通过 build 发现它**。」

### 3. BabyGlass 的属性

| 属性 | 内容 |
|------|------|
| 类型 | Web app（不是本地 IDE） |
| 关键问题 | "n agents × n projects 时，界面应该什么感觉？" |
| 状态可见性 | 默认该显示多少状态？ |
| 导航 | 跨越不同概念和 scope 是什么感觉？ |
| 简单性 vs 深度 | 既要够简单，又要给关心的人显示深路径 |

> 「这些问题你不能只在文档或 Figma mock 里回答——你必须 feel 它们，你必须 build 一个东西、用它、注意哪儿不对、改它。」

## 六、循环的前与后：留给人的是什么

> 「**如果 Agent 能执行更多循环，留给我们的是什么**？我认为答案在循环的前与后。」

### 1. 上游（Before）：是什么让你开始

> 「循环之前是那个让你开始的东西——一个你不能忽视的 gap、一个让你烦的感觉、一个**某物应该存在**的 feeling，即使你还没法清楚解释。」

**Mac 时代的回忆**作为 Ryo Lu 的源头：

- Dock 弹跳
- Genie effect 横扫
- Exposé 把窗口像桌上的卡片一样散开

> 「没有一件是必要的，但**所有这些都让你感觉有人在乎**。软件有质感、有意见、你能感觉到它背后那个人。」

后来发生了什么：

> 「一路上，很多软件变得更平滑、更快、更一致，**然后莫名其妙变得更无趣**。quirk 被去掉因为测试不好。warmth 被砍掉因为不可测。我们优化到了一个完美工作但感觉空无一物的世界。」

> 「而现在 AI 把速度推到非人——一个下午能生成整个产品、午饭前能 ship 一个功能、喝咖啡前能 spin 起 10 个变体。**疯了，但陷阱是同一个：做东西几乎不要钱时，slop 也变得免费。**」

### 2. 下游（After）：责任

> 「循环之后是责任——**我们向世界释放了什么？它改变了什么？它服务了谁？它让什么变得更容易变成？**」

> 「Agent 能执行循环、能 multiply 循环、能做 10 版、100 版、1000 版，**但它们不知道什么值得保护**。它们不知道技术上对但感觉死。它们不知道这个东西悄悄把我们拉向哪种未来。**那仍然是我们的责任。**」

### 3. 手艺（craft）的迁移

> 「手艺不会消失，它迁移。当执行昂贵时，手艺活在执行本身——一行小心的代码、一处精确的布局、一句重写 12 次的文案，那些东西仍然重要。但当生成变便宜，手艺移到上游和下游。」

| 阶段 | 手艺的位置 | 具体内容 |
|------|------------|----------|
| 上游 | 判断 | 问什么、留什么、拒什么、哪里慢下来、什么不做 |
| 中游（执行） | （被 Agent 接管） | |
| 下游 | 责任 | 释放什么、改变什么、服务谁 |

> 「**Making is not only producing an artifact, making is a bet on what kind of the world should exist.**（制作不只是产生产物，制作是一个关于**世界应该存在哪种形态**的下注。）」

> 「Craft does not disappear, it moves to the edges of judgment.（手艺不会消失，它移到判断的边缘。）」

## 七、收尾：让 demo 本身成为黏土

> 「这场演讲从一个 Ryo OS 里的小笔记开始——一个'某物应该存在'的 feeling。到结尾时，那个 note 变了。不是因为想法变完美了，是因为**它变真实了**。我能 build 它的一片、用它、feel 哪儿不对、改那一处、再试、再试、再试。」

> 「某个时刻，这个东西不再只是一个想法。它变成一个**我能在里面思考的地方**。我能用这个东西思考这个东西。**那是我 build 的时刻**——不是生产力，不是 disruption，不是为了某个大词。」

> 「**那是一样只存在于你头脑里的东西，变得真实到能被另一个人看见、摸到的时刻**。这部分没变、不会变。工具会变、模型会变、经济会变，但这一段留着。」

> 「你感觉哪儿不对，你 mark 一下；世界回答；你让它 real；最终这东西变得 real 到能 touch 到别人。**那是我想要的未来**——不是人类旁观、软件自建；我想做让人**感觉离材料更近**的工具。**让想象与现实之间的循环更短、更清晰、更诚实**的工具。」

---

## 八、一句话总结

> **AI 让循环变快、变便宜；但循环变快也意味着它能被藏起来；藏起来意味着人变成批准者而非作者；**所以 Cursor 做「Glass」——让循环本身可读、可塑、可学——让 software 重新变成 clay 而不是 verdict。
