# 证据表 · 三个金句 + 三个证据 (v3 · TED 级)

> **TED 标准**：
> - 每个证据必须有一个**具体画面**（观众脑里能浮现）
> - 每个金句必须有**一个能让观众点头的语义支点**（不是孤立的漂亮话）
> - 每张 PPT 必须有一个**观众会主动拍照的金句 / 数据 / 对照**
> - 现场留 1-2 处**反方空间**（智性诚实 > 一边倒）
>
> **演讲骨架已钉死**：
> - 听众：一线设计师（执行层）
> - 散场动作：反思"coding 是不是该追的浪潮"
> - 情感发动机：买椟还珠反讽
> - 叙事骨架：三金句 + 三证据
> - 可信度配比：内部 60% + 外部 40%
> - 核心独家证据：你自己的 monorepo / QoderWork 案例
> - QoderWork 公开深度：身份 + 组件库 + 客户端 UI + QoderIDE 升级 + Agent Loop 设计决策
> - 金句语气：冷 + 反讽
> - 证据 1 锁定：A + C + D (Boris + Ryo Lu + Jenny Wen)
> - 可贴图：可以

## 一、三个金句（最终版）

### 金句 1：开篇钩子（2 min 砸下）

> **"设计师不缺审美，缺的是转身。"**

- 语义支点：**审美是设计师的存量优势，转身是增量选择**
- 为什么能点头：因为台下 90% 设计师都觉得自己"审美很好"，但从没想过"不转身 = 浪费审美"
- 反讽底层：Mark Pincus 的 "Proven 博士学位" —— 设计师得先成为"亲手 build 的博士"，才有资格谈"设计创新"

### 金句 2：中段反转（在证据 2 之后）

> **"你以为你在保护专业分工，其实你在错过时代窗口。"**

- 语义支点：**分工是手段，话语权是目的**
- 为什么能点头：台下大部分设计师被"我会审美就够了"这套叙事养了 10 年，听到"你在错过时代窗口"会下意识抗拒，但无法反驳
- 反讽底层：George Hotz "AI Coding Agent 可能是最昂贵的错误之一" (684赞/553收藏) —— 我承认反方存在，但**你的转身不是因为我无视风险，是因为我赌了**
- 现场反方空间（高级反讽）："如果你不同意，欢迎听完最后再说"

### 金句 3：结尾落锤

> **"设计师不是去学代码，设计师是去重新拿回产品最终形态的话语权。"**

- 语义支点：**学习 coding 是手段，重新拥有产品是目的**
- 为什么能点头：台下设计师听完 monorepo 故事，自然会把"学代码"重新定义为"拿话语权"
- 反讽底层：买椟还珠 —— 把"会审美"当椟，把"亲手 build"当珠。设计师以为自己是椟的守护者，其实是珠的让渡者

### 证据 2：你的 monorepo 案例（7 min · 独家重量级）

**这是全演讲的独家重量级证据，必须讲透。**

#### 主线叙事（3 min）

> "我是 QoderWork 的主设计师，过去一年，我跟工程师的协作模式变了。
>
> 不是我去学业务代码——**我不碰业务代码**。
> 而是**我维护一套 monorepo 库**：组件、样式、风格、设计规范、人机交互 UI/UX、Agent Loop 的设计决策。
>
> 工程师**消费**我维护的组件库，他们只管业务逻辑。
> 我们互相遵守边界，**共同为 Agent 而构建各自的专业代码**。
>
> 我也参与了 QoderIDE 的设计升级、整个 QoderWork 客户端 UI 的开发和维护。"

#### 三个证据支点（4 min · 必讲满）

**支点 A · 库维护者身份 (1 min)**
- TED 级画面：你打开 monorepo 仓库，展示组件 / token / 规范 / Agent Loop 设计决策的文件树
- 关键金句：**"我不是去学代码，我是去拥有我设计的产品的最终形态。"**
- 数据：组件数 / token 数 / 消费业务数（你能给具体数字吗？）

**支点 B · 跟工程师的边界 (1 min)**
- TED 级画面：左半屏是设计师的 monorepo，右半屏是工程师的业务代码
- 关键金句：**"业务归工程师，形态归我。"**
- 反向论据：Lee Robinson (Cursor 工程师) "全栈通是伪命题" —— 我不是全栈通，我是 monorepo 自治

**支点 C · Agent Loop 设计决策 (2 min)**
- TED 级画面：你在白板上画 monorepo 协调者 + 业务代码 spoke，跟 Claude Architect (Hub-and-Spoke) 架构图并排
- 关键金句：**"我参与了 Agent Loop 的设计决策。这不是工程师的事，是产品最终形态的事。"**
- 技术注脚（给少数技术向院长）：
  - **Harness = monorepo**（包裹模型的环境）
  - **Skill = 组件库**（设计师的技能资产）
  - **Loop = Agent Loop**（设计师参与的核心架构）
  - **数据锚**：Anthropic Multi-Agent 系统比 Single-Agent 强 90.2%（Deep Research 博客）
- 业界同盟：**Framer "画布优于代码 / Agent 创建的每一样东西，都和你自己在画布上设计的一样可编辑、可控"** —— Framer 用产品哲学说"画布不被代码替代"，你用 monorepo 实践说"设计师不被代码替代"
- 反讽底层：Meaghan Choi (Claude Code 首席设计师) 每天的工作流 —— **Anthropic 自家设计师也在 build，你的转身不是异类，是主流**

#### 证据 2 反方空间（备用）
