## 摘要

大家都觉得像 Anthropic 这种估值逼近一万亿的超级独角兽，内部肯定有着极其严密的研发流水线，产品经理负责写需求，设计师负责做原型，程序员负责敲代码，各司其职互不干扰。
	
其实完全不是这样。
	
这家公司的设计负责人 Meaghan Choi 每天都在直接往生产环境里推代码。不是在画高保真原型图，不是在验收前端视觉效果，是真刀真枪地把写好的代码直接推上线。
	
Claude Code 在过去一年里创造了 25 亿美元的营收，甚至拿下了一半的编程市场份额。打下这片江山的团队，并不是几百人的庞大技术军团。一年半前他们只有 12 个人挤在一个房间里，现在依然保持着 3 到 5 人的敏捷小分队。
	
坦率的讲，看到人家真正的运作模式，比看多少份行业分析报告都来得实在。我们很多人还在死守自己的专业边界，最头部的 AI 公司工位上已经完全换了一套生存法则。
	
「职场上的头衔其实是最大的障眼法」
	
回想一下日常工作中最常见的开会拉扯场景。开发人员抱怨需求文档不清晰，设计吐槽前端代码还原度太低，产品经理焦头烂额地催着两边赶紧交付。每个人都死死守着自己的岗位边界，生怕多干了一点活儿，或者背了不该背的锅。
	
我们总以为这种厚重的部门墙是大公司管理的宿命，其实只是因为以前敲代码的试错成本太高了。
	
我是真的觉得，以前懂代码的人极其稀缺，所以必须把软件工程拆解得极其细碎。但在 Anthropic 的那个几人小分队里，大家全都在带着一堆 Claude 往前冲。
	
Meaghan 在访谈里有一句话，「职位头衔真的已经完全不重要了。我自己都在把代码推送到生产环境，而我的工程师们也在做设计。」连他们的财务团队都在熟练使用 Claude Code 处理业务数据。
	
在这个环境里，职位头衔仅仅变成了一个人带进团队的某种专长，绝不是限制干活的紧箍咒。工程师在做设计，设计师在推代码，大家唯一的共同目标就是把东西做出来跑通。
	
撕掉给自己贴的那张专业标签，试着一个人用 Claude Code 从头到尾把手里的活儿完整跑一遍。
