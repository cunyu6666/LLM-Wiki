## 摘要

🔬 背景痛点
现在的 Agent 框架虽然给大模型配上了工具、记忆、规划能力，但部署后能力就固定了。面对长序列决策和部分可观测环境，Agent 需要不断从交互中学习，而不是靠人类反复迭代 prompt 和工具。现有的 prompt 优化方法还有个致命限制——需要环境重置，每次都得从头开始，这在真实的持续交互场景中根本不现实。
-
💡 核心方案
Continual Harness 的思路是让 Agent 自己改自己。它是一个无重置的自我改进框架，从一个最简环境接口起步——没有预设知识、没有手工工具、没有领域脚手架。Agent 在运行过程中交替执行两件事：在环境中行动，以及根据历史轨迹数据来优化自身的 prompt、子 Agent、技能库和记忆。整个过程发生在一次连续运行中，不需要 episode reset。
这个框架的前身是 Gemini Plays Pokemon（GPP）项目。该项目通过人机协作的方式迭代优化 harness，GPP 成为第一个通关宝可梦蓝版、黄版（困难模式）以及水晶版（零阵亡）的 AI 系统。Continual Harness 做的事情就是把人类从这个循环中完全移除。
-
✨ 关键创新
整个方案的核心理念是：Harness不该是静态的工程产物，而应该是一个跟着经验一起进化的动态系统。
论文提出「在线过程奖励协同学习」机制。具体做法是：开源 Agent 在自改进框架中生成 rollout，然后由前沿教师模型对这些轨迹做过程级重标注，再用标注数据更新 Agent 自身的模型参数。Agent 的策略改进和奖励信号学习是同时在线进行的，不需要停下来单独训练。
-
📊 实验结论
作者在宝可梦红版和绿宝石版上做了评估，测试了多个前沿模型。Continual Harness 大幅降低了按键操作成本，并且恢复了与手工精调专家 harness 之间的大部分性能差距。
协同学习机制的效果也很明显：在不重置环境的条件下，Agent 在宝可梦红版中持续达成新的游戏里程碑。这说明在线自我改进确实能带来稳定的能力增长，而不只是早期阶段的一次性提升。
