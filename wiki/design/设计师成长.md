# 设计师成长

> 88 sources

## Overview

AI 时代设计师的角色正在发生根本性转变。"设计师的圣诞树"（技能树不断膨胀但收入不涨）引发了广泛共鸣，但更深层的变化是：设计师的核心能力从"执行"转向"判断"。Notion 日历创始人分享的"AI 时代设计师通关秘籍"强调，设计师需要掌握 Vibe Coding 等新技能，Vercel 设计团队已经不按传统设计流程干活了。

DeepSeek 的逆袭之路提供了一个反直觉的案例："UI 无关紧要"——当产品核心能力足够强时，界面设计退居次要位置。这不意味着设计不重要，而是说设计师需要理解**什么阶段设计最重要**。离开苹果设计团队的设计师分享的感悟也印证了这一点：大厂设计的光环背后，是对商业和技术的深度理解。

设计奖项（iF、红点等）依然是职业发展的重要助力，但"来拿个设计奖给履历添添光"的务实态度说明，设计师越来越清楚奖项的本质是营销工具而非能力证明。

来自抖音的设计师成长视频则提供了更接地气的实战经验：作品集制作（从 Notion 轻量方案到百万年薪设计师的叙事方式）、面试策略（设计素养的差异化表达、B 端 vs C 端的设计逻辑差异）、以及设计思维（"设计的本质是解决问题"、"好设计师拷贝，伟大设计师偷"、打破常规追求有趣而非好看）。

## Ryo Lu 谈设计师与开发者的融合

Ryo Lu（Cursor 设计负责人，前 Notion 设计主管）在 a16z Deep Dives 播客中系统阐述了**AI 时代设计师角色的根本性转变**。两期访谈（50 分钟完整版 + 设计工具专题）的核心观点：

### 角色的融合与协作的坍缩

过去十五年，软件制作高度碎片化——设计师困在 Figma 里，PM 在 Google Docs 里，数据人员在别的工具里。Cursor 试图统一这些产出物：**同一段代码可以有不同视图**——工程师看代码编辑器，设计师看视觉界面，PM 看文档式界面。"共享的真相就是代码"。

> 未来在 Cursor 里，PM 写 PRD 也是在改代码，只是通过一种更被动的、组织化的方式。设计师更多影响视觉层面。

### 品味（Taste）的本质

Ryo 对"品味"的定义很具体：**品味 = 在所有选项中做选择的能力**。要做选择，你得先见过一切——挖掘过过去的设计、联系过自然和人造物、随时间建立了偏好边界（"这是对的、这是美的"）。

> LLM 的问题是：它其实见过一切，却没有自己的观点——或者说它把自己搞糊涂了，以为人们到处都想要紫色渐变。

### 设计师变为开发者

Cursor 让设计师第一次可以直接操作功能性产出物。"你只需要有一个想法——哪怕有点模糊——告诉 Agent，它可能第一枪就给你 60%、70% 的结果。你跳过了大量复杂性。"

关键洞察：**AI 是通用接口**。不区分"设计工具"和"开发工具"，只区分"想法"和"实现"。设计师不需要学会所有软件概念，但需要学会**描述意图 + 评估结果**。

> 设计的本质，就是为所有人找到最佳配置和最简状态。美，在于把一切整合到一起。

### 面试与求职策略

- **设计素养的差异化**：水球泡强调，与顶级设计师的差距不在软件技能，而在设计素养。例如在面试中，能从字体历史（意大利体、罗马体、哥特体）、设计理论（如逆反差字体在宋代的应用）等角度阐述设计决策，会让面试官感受到专业深度。"你和顶级设计师差得不仅仅是几款软件"——差距在于接触和学习的东西不同，要主动去学顶级设计师关注的内容。（
