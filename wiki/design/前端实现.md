# 前端实现

![](https://sns-webpic-qc.xhscdn.com/202409021238/e51158d2741eb63f9ee6518ab0c85e20/1040g008316eb87er1c005oqbfop65cgnvon1n8o!nd_dft_wlteh_webp_3)
## Overview

前端实现是设计师将创意落地的最后一步。2024-2025 年间，技术栈发生了显著迁移：Tailwind CSS + shadcn/ui 的组合正在取代 ElementUI/Ant Design 成为新主流，React 生态持续壮大（Lynx 作为抖音团队推出的 React Native 替代方案引发关注），Webflow/Framer 等无代码工具让设计师可以直接构建网站。

AI 正在以前所未有的速度入侵前端领域。"AI 入侵 Frontend，前端程序员药丸"的标题虽然夸张，但 HeroUI Chat（AI 一键生成 React UI）、Tempo Labs（AI 驱动的 React 可视化编辑器）等工具确实在改变前端开发的方式。Chrome 甚至开始清理"最丑的 UI 组件"，说明浏览器厂商也在推动前端设计标准的提升。

## CSS 文字方向属性

三个与 `writing-mode` 相关的属性，让宽高和间距**跟随文字方向**而非屏幕方向变化，在竖排文字、多语言混排场景中很实用：

| 属性 | 作用 |
|------|------|
| `writing-mode` | 改变文字排列方向（如 `vertical-rl`），元素的 width/height 语义随之旋转 |
| `inline-size` | 对应文字方向的「宽度」——竖排时控制高度方向上的尺寸 |
| `block-size` | 对应文字方向的「高度」——竖排时控制宽度方向上的尺寸 |
| `padding-block` | 始终相对于文字的上下内边距，不随 `writing-mode` 旋转而变成左右 |

典型场景：竖排日文/中文标题、RTL 语言适配、以及需要保持 padding 语义一致的多方向排版。
