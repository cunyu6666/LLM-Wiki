## CSS 技巧

### 逻辑属性：跟随文字方向的宽高

CSS 提供了一组与书写方向相关的逻辑属性，可以替代传统的 `width`/`height` 和 `padding`：

- **`inline-size` / `block-size`**：替代 `width` / `height`。当文字方向通过 `writing-mode` 旋转后，它们会自动跟随文字方向切换——`inline-size` 始终是文字流方向的尺寸，`block-size` 是垂直方向的尺寸。
- **`padding-block` / `padding-inline`**：替代 `padding-top/bottom` 和 `padding-left/right`。当元素旋转后，`padding-block` 始终是相对于文字的上下方向，不会随元素旋转而变化。

> 实际效果：一个 `width:60px; height:30px` 的元素，当 `writing-mode: vertical-rl` 后，`inline-size: 60px` 会变成垂直方向 60px，而 `width: 60px` 仍然是水平方向 60px。

## 浏览器机制

### Chrome 自动播放策略

Chrome 禁止网页自动播放音视频。要获得自动播放权限，需满足以下三种情况之一：

1. **静音媒体**：`muted` 的 `<video>`/`<audio>` 标签始终允许自动播放
2. **用户参与度**：用户经常在该网站手动点击播放，Chrome 会判定"媒体参与度"高，允许自动播放（可通过 `chrome://media-engagement` 查看）
3. **iframe 继承**：顶级页面已获得自动播放权限后，可以通过 `allow="autoplay"` 将权限下放给同域 iframe 子页面

> 关键发现：即使是抖音这样的网站，在全新浏览器配置文件中也无法自动播放——因为它依赖已有的媒体参与度数据。

### Chrome 多用户配置文件

Chrome 的"其他个人资料"功能可以实现同一网站同时登录多个账号：

- 每个配置文件的 Cookie、浏览记录、LocalStorage 完全隔离
- 不同于无痕模式——关闭后数据保留
- 适合前后台联调时同时登录管理员和普通用户账号

**Mac 快捷键技巧**：系统设置 > 键盘 > 快捷键 > App 快捷键，可以为任意应用的任意菜单项自定义快捷键。

## JavaScript 调试

### Console.log 的"不准"问题

`console.log` 打印对象或数组时，展开看到的不是打印那一刻的值，而是**展开那一刻**的最新值：

```js
let arr = [];
console.log(arr);  // 打印时为空
arr.push(1);       // 打印后添加元素
// 展开 console 时看到 [1]，但打印时确实是空的
```

**原理**：Chrome DevTools 在打印引用类型（对象/数组）时存储的是引用，展开时才实时读取该引用的当前状态。

**正确调试方法**：需要查看打印那一刻的值，使用 `JSON.stringify(arr)` 或展开运算符 `console.log(...arr)` 将值转为快照。

### Promise.resolve() 的实用场景

`Promise.resolve()` 的价值不在于它本身，而在于**保持类型一致性**：

```js
// 不用 Promise.resolve()：需要判断 null
async function submit() {
  const result = validate ? validate() : null;
  if (result) await result;  // 需要判空
}

// 用 Promise.resolve()：统一为 Promise 类型
async function submit() {
  const result = validate ? validate() : Promise.resolve();
  await result;  // 无需判空，类型一致
}
```

适合用作"占位 Promise"，避免在异步链路中对可能为空的 Promise 做额外判空。

## Vue / Element UI 实战

### el-collapse-transition 卡顿问题

Element UI 的 `el-collapse-transition` 组件展开时会卡顿，原因是它的实现原理：

- 不是用 CSS `transition`，而是用 JS 不断修改 `div` 的 `height`
- 动画截止高度取决于**内部元素的实际高度**，而非 CSS class 设定的高度
- 如果 class 设定 `height: 100px` 但内部元素只需 `20px`，动画到 20px 就结束了，剩余 80px 是瞬间跳变

**解决方案**：在折叠组件外层再包一个 `div`，让 JS 修改外层高度，内层的 class 高度就成为"实际需要的高度"。

### el-table max-height 百分比滚动问题

当 `el-table` 的 `max-height` 设置为比例值（如 `80%`）时，可能出现"滚动不到底"的问题。根本原因是初始化顺序：

- `BetterScroll` 等滚动组件在初始化时会计算容器高度
- 如果此时底部的占位元素还未插入 DOM，计算出的高度会偏小

**解决方案**：调换初始化顺序——先设置好容器高度和占位元素，再初始化滚动组件。

### Vue 货币计算技巧

推荐使用 **Dinero.js**（仅 1.14KB）处理货币计算，避免 JavaScript 浮点精度问题：

```js
// 浮点累加的问题
0.1 + 0.2  // 0.30000000000000004

// 使用 Dinero.js
Dinero({ amount: 100 }).add(Dinero({ amount: 200 }))  // 精确
```

**Vue 全局注入技巧**：

```js
// main.js
import Dinero from 'dinero.js';
Vue.prototype.$money = Dinero;

// Vue.prototype 上封装 formatMoney 方法
Vue.prototype.formatMoney = (amount) => {
  return Dinero({ amount }).setCurrency('CNY').toFormat();
};

// 组件中直接使用
this.$money({ amount: 1000 }).toFormat()  // "¥10.00"
this.formatMoney(1500)  // "¥15.00"
```

## 开发工具

### Fork：原生 Git 客户端

Fork 是一款高质量的 Git 客户端（Mac + Windows），特点：

- Mac 版用 Swift 原生开发，Windows 版用 .NET WPF 原生开发——非 Electron，体验流畅
- 界面简洁，支持 tag、stash、merge conflict 等常用功能
- 可免费试用（无功能限制），正式版 $50
- 开发者是一对夫妻——丈夫开发 Mac 版，妻子开发 Windows 版

官网：https://git-fork.com

### GitHub Copilot 代码提示体验

GitHub Copilot（VS Code / IntelliJ 插件）的代码生成能力：

- **注释驱动生成**：写注释描述功能，Copilot 自动补全实现代码
- **方法名推断**：输入 `calculateDaysBetween` 这样的方法名，自动补全日期计算逻辑
- **上下文感知**：能理解项目中的变量命名和代码风格
- **VS Code 体验优于 IntelliJ**：IntelliJ 版本响应较慢，VS Code 版本更流畅

> 实测：写一个"快速排序"的注释，Copilot 能自动生成完整的 partition + 递归实现，包括正确的边界处理。

### PyScript：在 HTML 中运行 Python

Anaconda 推出的 PyScript 项目，可以在浏览器中直接运行 Python：

```html
<link rel="stylesheet" href="https://pyscript.net/releases/2022.12.1/pyscript.css" />
<script defer src="https://pyscript.net/releases/2022.12.1/pyscript.js"></script>

<py-script>
print("Hello from Python!")
</py-script>
```

- 通过 `<py-script>` 标签嵌入 Python 代码
- 支持标准库（如 `time` 模块）和列表推导等语法
- 代码缩进必须顶到最左边，否则会报缩进错误
- 项目处于 Alpha 阶段，官网：https://pyscript.net
